# MVP 기능 범위 및 평가 기준

정리일: 2026-06-30
기준 문서: `content/06_참고자료/2026-06-30_KX_인턴쉽_프로젝트_명세서.md`

## 1. 문서 목적

이 문서는 A2팀이 2주차 중간점검과 4주차 최종평가에서 어떤 기능을 어디까지 보여줘야 하는지 정리한다.

PRD는 제품 요구사항을 설명하고, 이 문서는 평가 기준과 기능 우선순위를 관리한다.

## 2. MVP 원칙

| 원칙 | 설명 |
| --- | --- |
| 핵심 기능 완주 우선 | 완성품보다 작동하는 최소 기능을 우선한다. |
| 추가 기능은 선택 | 역프롬프트와 고급 관리 기능은 A등급 흐름이 안정화된 뒤 진행한다. |
| 비용 통제 | 영상 생성은 짧은 길이와 제한된 호출 수로 테스트한다. |
| 데모 가능성 | 최종 산출물은 배포 데모 URL, GitHub Public Repo, 3분 시연 영상, README를 포함한다. |

## 3. 2주차 중간점검 Pass 기준

프로젝트 명세서 기준 중간점검은 Pass / Fail 평가다. A2팀은 이미지 생성과 라이브러리 흐름을 먼저 완성한다.

| 항목 | Pass 기준 | A2팀 해석 |
| --- | --- | --- |
| 기획 완성도 | 핵심 기능과 화면 설계 포함 | PRD, MVP 범위, 화면 흐름이 정리되어 있다. |
| API 연동 | 이미지 생성 호출 성공 | GPT Image 2.0 호출로 이미지가 생성된다. |
| 핵심 기능 | 프롬프트 → 이미지 → 저장 동작 | 이미지 생성 결과가 라이브러리에 저장되고 다시 조회된다. |

### 3.1 2주차 필수 흐름

```text
프롬프트 입력
→ GPT Image 2.0 이미지 생성
→ 생성 결과 표시
→ 라이브러리 저장
→ 라이브러리 조회
```

### 3.2 2주차 제외 범위

| 제외 기능 | 이유 |
| --- | --- |
| 영상 생성 | 4주차 A등급 목표로 관리한다. |
| 역프롬프트 | S등급 선택 기능으로 관리한다. |
| 검색 / 필터 / 다운로드 | 핵심 저장 흐름 이후 보조 기능으로 진행한다. |
| 고급 편집 | 4주 MVP 범위를 벗어난다. |

## 4. 4주차 최종평가 기준

| 등급 | 프로젝트 명세서 기준 | A2팀 구현 기준 |
| --- | --- | --- |
| S | 이미지+영상 통합 + 역프롬프트 재생성 루프 + 라이브러리 + 배포 | A등급 기능에 역프롬프트 추출, 수정, 이미지 재생성, 저장 루프를 추가한다. |
| A | 이미지+영상 생성 + 라이브러리 + 배포 | 이미지 생성, 3x3 그리드 기반 영상 생성, 라이브러리, 배포를 연결한다. |
| B | 이미지 생성 + 라이브러리 + 배포 | 2주차 Pass 흐름을 배포 환경에서 안정적으로 보여준다. |
| C | 이미지 생성만 동작 | 저장/조회나 배포 흐름이 부족하다. |
| Fail | API 연동 실패 | 핵심 생성 API가 동작하지 않는다. |

## 5. A등급 목표 기능

| ID | 기능 | 완료 기준 |
| --- | --- | --- |
| A001 | 이미지 생성 | 배경 시트, 캐릭터 시트, 키이미지를 생성한다. |
| A002 | 영상 생성 | 3x3 스토리보드 그리드 이미지 1장과 9개 씬 프롬프트로 3~15초 영상을 생성한다. |
| A003 | 생성 상태 표시 | 영상 생성 대기, 진행, 완료, 실패 상태를 사용자에게 보여준다. |
| A004 | 라이브러리 | 이미지와 영상을 저장하고 다시 조회한다. |
| A005 | 배포 | Vercel Hobby와 AWS EC2/S3 기준으로 데모 가능한 상태를 만든다. |

## 6. S등급 선택 기능

역프롬프트는 필수가 아니라 선택 기능이다.

| ID | 기능 | 착수 조건 |
| --- | --- | --- |
| S001 | 레퍼런스 이미지 분석 | A등급 핵심 흐름 안정화 이후 |
| S002 | 프롬프트 추출 | 분석 결과를 재현용 프롬프트로 변환할 수 있을 때 |
| S003 | 프롬프트 수정 | 사용자가 추출 결과를 편집할 수 있을 때 |
| S004 | 이미지 재생성 및 저장 | 수정 프롬프트로 이미지를 다시 생성하고 라이브러리에 저장할 수 있을 때 |

## 7. 클라이언트 관점 평가 포인트

| 평가 포인트 | 확인 내용 |
| --- | --- |
| 안정성 | 생성 실패율과 에러 안내가 관리되는가 |
| 비동기 경험 | 영상 생성 상태가 사용자에게 자연스럽게 전달되는가 |
| 통합도 | 이미지, 영상, 라이브러리 흐름이 하나의 서비스처럼 이어지는가 |
| 직관성 | 방송 제작자가 기능을 설명 없이 이해할 수 있는가 |
| 재사용성 | 생성 결과를 다시 찾고 발표나 회의에 활용할 수 있는가 |
