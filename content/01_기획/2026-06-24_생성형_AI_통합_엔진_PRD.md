1. 프로젝트 개요
1.1 프로젝트명
프로젝트 A: 생성형 AI 통합 엔진
1.2 한 줄 정의
방송 제작자를 위한 GPT Image 2.0 기반 이미지·영상 생성 통합 워크플로우 서비스
1.3 서비스 소개
본 서비스는 방송 제작자가 여러 생성형 AI 사이트를 오가지 않고, 한 화면에서 이미지와 영상을 생성하고 결과물을 저장·관리할 수 있도록 돕는 제작 보조형 서비스이다.
사용자는 프롬프트를 입력해 GPT Image 2.0 기반 이미지를 생성할 수 있으며, 별도의 영상 생성 모델을 통해 영상 초안을 만들 수 있다. 생성된 이미지와 영상은 미디어 라이브러리에 저장되어 다시 확인하거나 활용할 수 있다.
역프롬프트 기능은 핵심 MVP 이후 여유가 있을 때 구현하는 선택 기능으로 둔다. 해당 기능은 레퍼런스 이미지를 분석해 재현용 프롬프트를 추출하고, 추후 이미지 재생성 기능과 연결하는 방향으로 확장한다.
2. 기획 배경 및 목표
2.1 문제 정의
방송 제작 과정에서는 기획안, 콘셉트 회의, 레퍼런스 공유, 영상 방향성 설명을 위해 빠르게 시각 자료가 필요하다. 하지만 현재는 이미지 생성, 영상 생성, 결과물 저장을 각각 다른 서비스에서 처리해야 하므로 작업 흐름이 분산된다.
이로 인해 다음과 같은 문제가 발생한다.

5. 핵심 기능 요구사항
5.1 이미지 생성
사용자는 텍스트 프롬프트를 입력하고 이미지 생성 버튼을 눌러 이미지를 생성할 수 있다.
항목
 |
내용

입력값
 |
프롬프트, 비율, 품질 옵션

사용 모델
 |
GPT Image 2.0 / gpt-image-2

결과
 |
생성 이미지 표시

후속 행동
 |
라이브러리 저장, 다시 생성
완료 기준
프롬프트 입력 후 이미지 생성 요청이 가능해야 한다.
생성 결과가 화면에 표시되어야 한다.
생성된 이미지를 라이브러리에 저장할 수 있어야 한다.
실패 시 에러 메시지를 표시해야 한다.
5.2 영상 생성
사용자는 영상용 프롬프트를 입력하고 영상 생성 요청을 보낼 수 있다. 영상 생성은 시간이 오래 걸릴 수 있으므로 비동기 방식으로 처리한다.
항목
 |
내용

입력값
 |
프롬프트, 영상 모델, 길이 옵션

사용 모델
 |
fal.ai 기반 Seedance 2.0 또는 Kling 3.0

처리 방식
 |
submit → polling → result

결과
 |
생성 완료된 영상 URL 또는 미리보기 표시
완료 기준
영상 생성 요청 후 request id를 받을 수 있어야 한다.
생성 상태를 queued, processing, completed, failed 등으로 표시해야 한다.
completed 상태가 되면 영상 결과를 보여줘야 한다.
생성된 영상을 라이브러리에 저장할 수 있어야 한다.
5.3 미디어 라이브러리
이미지와 영상 생성 결과를 저장하고 다시 조회할 수 있는 공간을 제공한다.
항목
 |
내용

저장 대상
 |
이미지, 영상

저장 정보
 |
결과물 URL, 타입, 프롬프트, 모델, 생성일

조회 방식
 |
카드 또는 그리드 형태

상세 보기
 |
결과물 미리보기, 프롬프트, 생성 정보 확인
완료 기준
이미지와 영상을 라이브러리에 저장할 수 있어야 한다.
라이브러리에서 저장된 결과물을 다시 조회할 수 있어야 한다.
이미지와 영상을 구분해서 표시할 수 있어야 한다.
결과물이 없을 때 빈 상태 UI를 표시해야 한다.

8. 주요 워크플로우
8.1 이미지 생성 흐름
사용자 프롬프트 입력
→ 이미지 옵션 선택
→ GPT Image 2.0용 프롬프트 보강

사용자 영상 프롬프트 입력

→ fal.ai queue submit
→ request id 반환
→ 클라이언트에서 상태 polling
→ completed 상태 확인
→ 영상 결과 표시
→ 라이브러리 저장

8.3 라이브러리 저장 흐름
이미지 또는 영상 생성 완료
→ 저장 버튼 클릭
→ Supabase Storage에 파일 저장
→ DB에 결과물 정보 저장
→ 라이브러리 화면에서 조회

11. 최종 MVP 성공 기준
본 프로젝트의 MVP 성공 기준은 다음과 같다.
프롬프트 입력
→ GPT Image 2.0 이미지 생성
→ 결과 확인
→ 라이브러리 저장
→ 영상 생성 요청
→ 영상 생성 상태 확인
→ 영상 결과 저장
→ 라이브러리에서 이미지·영상 조회

위 흐름이 실제 서비스 화면에서 동작하면 MVP 완료로 본다.
선택 기능인 역프롬프트, 프롬프트 재생성, 다운로드, 검색·필터는 핵심 MVP 완료 이후 여유가 있을 때 추가한다.
