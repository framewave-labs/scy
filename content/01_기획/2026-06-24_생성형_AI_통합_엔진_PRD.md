## 2. 제품 개요

### 2.1 프로젝트명

프로젝트 A: 생성형 AI 통합 엔진

### 2.2 서비스명

**GeNova** (제노바)

**KX 엔터테인먼트(본사) 지정 서비스명. 2026-07-20(월) 수령.** 프로젝트 착수 시 최종 서비스명은 본사가 직접 정해 전달하기로 되어 있었다. 그때까지 쓸 임시명은 팀 브레인스토밍으로 `KX GenStudio`로 정해 사용해 왔고(`lsg` 서비스기획서 기재), 본사 지정명을 수령한 이 시점에 `GeNova`로 교체한다.

수령 사실이 2026-07-20 회의록과 같은 날 제출한 3주차 중간보고서에 기재되지 않아, 이 절이 최초 기록이다. 최종 발표 자료와 대외 표기는 모두 `GeNova`를 기준으로 한다.

### 2.3 한 줄 정의

아이디어를 이미지와 짧은 영상으로 구체화할 수 있도록, 이미지 생성, 3x3 스토리보드 기반 영상 생성, 미디어 라이브러리를 연결하는 웹 기반 생성형 AI 제작 서비스

### 2.4 서비스 설명

GeNova는 사용자가 복잡한 편집 프로그램이나 고급 프롬프트 작성법을 몰라도 단계별 흐름을 따라 이미지와 짧은 영상 결과물을 만들 수 있도록 돕는 서비스다.

초기 MVP는 이미지 생성과 라이브러리 저장/조회 흐름을 먼저 안정화한다. 이후 B서비스에서 받은 `3x3그리드로 구성된 1장의 이미지`와 영상 프롬프트를 입력해 영상 생성 흐름으로 확장한다.

```text
홈 진입
→ 이미지 생성
→ 생성 결과 확인
→ 라이브러리 저장
→ 라이브러리 조회
→ 3x3 스토리보드 이미지와 영상 프롬프트 입력
→ 영상 생성 요청
→ 진행상태 조회
→ 영상 결과 확인
→ 라이브러리 저장 및 재조회
```

## 3. 문제 정의

| 문제 | 설명 |
| --- | --- |
| 생성 흐름 분산 | 이미지 생성, 영상 생성, 프롬프트 정리, 결과물 저장이 서로 다른 도구에 흩어질 수 있다. |
| 프롬프트 작성 부담 | 사용자는 장면 의도는 알고 있어도 AI 프롬프트 구조를 익숙하게 다루기 어렵다. |
| 결과물 관리 어려움 | 생성한 이미지와 영상이 누적되면 다시 찾고 비교하기 어렵다. |
| 시각 일관성 부족 | 캐릭터, 배경, 키이미지, 영상 결과가 같은 기준으로 이어지지 않을 수 있다. |
| 영상 생성 비용 리스크 | 영상 생성은 실패 호출과 긴 영상 옵션에서 비용 부담이 빠르게 커진다. |
| 비동기 상태 이해 어려움 | 영상 생성처럼 시간이 걸리는 작업은 대기, 처리, 성공, 실패 상태를 명확히 보여줘야 한다. |

## 5. 제품 목표

### 5.1 2주차 Pass 목표

2주차 중간점검은 이미지 생성, 결과 표시, 저장, 라이브러리 조회가 실제 화면에서 끊기지 않고 이어지는 것을 목표로 한다.

```text
프롬프트 입력
→ 이미지 생성 요청
→ 생성 결과 표시
→ 저장
→ 라이브러리 조회
```

### 5.2 4주차 최소 완주 목표

4주차에는 이미지 생성 흐름 위에 영상 생성과 미디어 라이브러리를 연결한다.

```text
이미지 생성 및 저장
→ 3x3 스토리보드 이미지 업로드
→ 영상 프롬프트 입력
→ 영상 생성 요청
→ 상태 조회
→ 영상 결과 표시
→ 이미지/영상 통합 라이브러리 조회
→ Vercel FE + AWS BE 기준 배포 시연
```

### 5.3 A등급 목표

최소 완주 흐름에 더해 캐릭터 시트, 배경 시트 등 방향별 레퍼런스 확보 흐름을 완성한다.

### 5.4 S등급 도전

A등급 핵심 기능이 안정화된 뒤 역프롬프트, 프롬프트 수정, 이미지 재생성, 재생성 결과 저장 흐름을 도전 범위로 진행한다.

## 7. 주요 사용자 흐름

### 7.1 이미지 생성 흐름

```text
이미지 생성 화면 진입
→ 생성 목적 선택: 캐릭터 / 배경 / 키이미지
→ 프롬프트 입력
→ 비율 또는 품질 옵션 선택
→ 생성 요청
→ 로딩 상태 확인
→ 생성 결과 확인
→ 저장
→ 라이브러리에서 재조회
```

### 7.2 방향별 레퍼런스 흐름

```text
이미지 생성 화면 진입
→ 기준 이미지 선택
→ 캐릭터 시트 또는 배경 시트 생성
→ 필요 시 키이미지 생성
→ 결과 저장
```

방향별 레퍼런스는 별도 화면으로 분리하지 않고 이미지 생성 화면 안의 탭 또는 섹션으로 포함하는 방향을 1차 기준으로 둔다.

### 7.3 영상 생성 흐름

```text
영상 생성 화면 진입
→ B서비스에서 받은 3x3 스토리보드 이미지 1장 업로드
→ 영상 프롬프트 입력
→ 모델 또는 길이 옵션 선택
→ 영상 생성 요청
→ 진행상태 확인
→ 영상 결과 미리보기
→ 저장
→ 라이브러리에서 재조회
```

영상 생성 상태는 사용자 화면에서 `processing`, `completed`, `failed`처럼 이해하기 쉬운 상태로 표시한다. DB 또는 백엔드 작업 상태값은 2026-07-07 결정에 따라 `PENDING`을 제거하고 `RUNNING`, `SUCCESS`, `FAILED`를 사용한다. 사용자 화면의 `queued` 상태도 이에 맞춰 제외한다.

### 7.4 라이브러리 흐름

```text
라이브러리 진입
→ 전체 / 이미지 / 영상 필터 선택
→ 최신순 작업물 확인
→ 작업물 상세 보기
→ 프롬프트 확인
→ 다운로드 / 재생성 / 삭제 등 후속 액션
```

생성 결과물은 `게시글`이 아니라 `작업물` 또는 `라이브러리 항목`으로 표현한다.

### 7.5 역프롬프트 흐름

```text
역프롬프트 화면 진입
→ 이미지 업로드
→ 이미지 분석
→ 프롬프트 추출
→ 사용자가 프롬프트 수정
→ 이미지 재생성
→ 저장
```

역프롬프트는 S등급 도전 범위다. 2주차 Pass와 4주차 최소 완주 기능이 안정화된 뒤 착수한다.

## 10. API 및 DB 기준

API와 DB는 2026-07-03 사용자가 업로드한 명세 자료를 기준으로 한다. 이 PRD에서는 제품 흐름에 필요한 수준만 요약하고, 상세 요청/응답 스키마는 엔지니어링 문서에서 관리한다.

### 10.1 API 목록

| 구분 | 기능 | Method | URI |
| --- | --- | --- | --- |
| 생성 | 이미지 생성 | `POST` | `/api/generate-image` |
| 생성 | 역프롬프트 생성 | `POST` | `/api/reverse-prompt` |
| 생성 | 영상 생성 | `POST` | `/api/generate-video` |
| 작업 상태 | 영상 진행상태 조회 | `GET` | `/api/video-status` |
| 파일 | 이미지 업로드 | `POST` | `/api/file` |
| 파일 | 파일 삭제 | `DELETE` | `/api/file/{id}` |
| 파일 | 파일 다운로드 | `GET` | `/api/file/{id}/download` |
| 파일 | 파일 목록 조회 | `GET` | `/api/file` |
| 파일 | 파일 상세 조회 | `GET` | `/api/file/{id}` |

### 10.2 DB 기준

| 테이블 | 역할 |
| --- | --- |
| `MediaFile` | 이미지와 영상 파일 자산 저장 |
| `GenerateJob` | 이미지, 영상, 프롬프트 생성 작업 상태 저장 |

| 항목 | 현재 기준 |
| --- | --- |
| 파일 타입 | `IMAGE`, `VIDEO` |
| 파일 경로 | AWS S3 또는 로컬 전체 경로 |
| 원본 프롬프트 | `MediaFile.origin_prompt` |
| 역프롬프트 | `MediaFile.reversed_prompt` |
| 작업 유형 | `IMAGE`, `VIDEO`, `PROMPT` |
| 작업 상태 | `RUNNING`, `SUCCESS`, `FAILED` (2026-07-07: PENDING 삭제) |

### 10.3 프론트엔드 연동 전 확인 필요

| 확인 항목 | 목적 |
| --- | --- |
| request body / response body / error response | FE-BE 연동 기준 확정 |
| `/api/video-status` 조회 키 | 영상 상태 조회 방식 확정 |
| 파일 업로드 필드명 | 업로드 UI 연결 |
| 파일 삭제 정책 | DB row와 실제 파일 삭제 범위 확정 |
| 다운로드 응답 방식 | redirect URL, stream, presigned URL 중 선택 |
| 역프롬프트 저장 범위 | 반환만 할지, `MediaFile.reversed_prompt`에 저장할지 확정 |
| 실제 물리 테이블명 | 문서 표기와 구현 표기 정렬 |
| 인증 / 권한 | MVP 인증 범위 확정 |
