## 2. 제품 개요

### 2.1 프로젝트명

프로젝트 A: 생성형 AI 통합 엔진

### 2.2 서비스명

KX GenStudio

### 2.3 한 줄 정의

방송 제작자를 위한 GPT Image 2.0 기반 이미지 생성, 3x3 스토리보드 그리드 기반 영상 생성, 미디어 라이브러리 통합 워크플로우 서비스

### 2.4 서비스 소개

KX GenStudio는 방송 제작자가 기획 단계에서 필요한 시각 자료를 빠르게 만들고, 저장하고, 다시 활용할 수 있도록 돕는 제작 보조 서비스다.

사용자는 배경 시트와 캐릭터 시트로 시각 기준을 잡고, 이를 바탕으로 키이미지를 만든다. 이후 9개 장면이 3x3으로 배치된 스토리보드 그리드 이미지 1장과 9개 씬 프롬프트를 입력해 3~15초 범위의 짧은 영상을 생성한다.

## 3. 문제 정의

| 문제 | 설명 |
| --- | --- |
| 작업 흐름 분산 | 이미지 생성, 영상 생성, 프롬프트 정리, 저장을 서로 다른 도구에서 처리해야 한다. |
| 프롬프트 작성 부담 | 방송 제작자는 AI 프롬프트 문법보다 장면 의도와 연출에 익숙하다. |
| 결과물 관리 어려움 | 생성한 이미지와 영상이 흩어져 다시 찾거나 비교하기 어렵다. |
| 시각 일관성 부족 | 배경, 캐릭터, 장면 분위기가 생성 결과마다 달라질 수 있다. |
| 비용 관리 어려움 | 영상 생성은 실패 호출과 긴 영상 옵션에서 비용이 빠르게 늘어난다. |

## 4. 제품 목표

| 목표 | 설명 |
| --- | --- |
| 2주차 Pass | 프롬프트 입력, 이미지 생성, 결과 표시, 저장, 라이브러리 조회 흐름을 실제 화면에서 동작시킨다. |
| 4주차 A등급 | 이미지 생성, 3x3 그리드 기반 영상 생성, 라이브러리, 배포를 연결한다. |
| S등급 도전 | 역프롬프트 재생성 루프를 선택 기능으로 추가한다. |

역프롬프트는 필수 기능이 아니라 선택 기능이다. A등급 핵심 흐름이 안정화된 뒤 여유가 있을 때 S등급 도전 범위로 진행한다.

## 6. 핵심 사용자 흐름

### 6.1 이미지 생성 흐름

```text
이미지 생성 화면 진입
→ 배경 시트 또는 캐릭터 시트 프롬프트 입력
→ GPT Image 2.0 이미지 생성
→ 결과 확인
→ 라이브러리 저장
→ 라이브러리에서 다시 조회
```

### 6.2 영상 생성 흐름

```text
영상 생성 화면 진입
→ 3x3 스토리보드 그리드 이미지 1장 업로드
→ scene 1~9 프롬프트 입력
→ 영상 길이 선택
→ 영상 생성 요청
→ 생성 상태 확인
→ 3~15초 영상 결과 확인
→ 라이브러리 저장
```

### 6.3 선택 기능: 역프롬프트 흐름

```text
레퍼런스 이미지 업로드
→ 프롬프트 추출
→ 사용자가 프롬프트 수정
→ 수정 프롬프트로 이미지 재생성
→ 라이브러리 저장
```

## 9. 평가 기준

프로젝트 명세서의 평가 기준을 A2팀 최신 방향에 맞춰 해석한다.

### 9.1 2주차 중간점검 Pass

| 항목 | Pass 기준 |
| --- | --- |
| 기획 완성도 | 핵심 기능과 화면 흐름이 문서로 정리되어 있다. |
| API 연동 | 이미지 생성 호출이 성공한다. |
| 핵심 기능 | 프롬프트 → 이미지 생성 → 저장 → 라이브러리 조회 흐름이 동작한다. |

### 9.2 4주차 최종평가

| 등급 | 기준 |
| --- | --- |
| S | 이미지 생성 + 영상 생성 + 역프롬프트 재생성 루프 + 라이브러리 + 배포 |
| A | 이미지 생성 + 영상 생성 + 라이브러리 + 배포 |
| B | 이미지 생성 + 라이브러리 + 배포 |
| C | 이미지 생성만 동작 |
| Fail | 핵심 API 연동 실패 |

클라이언트 관점 평가 포인트는 생성 실패율, 영상 처리 흐름의 매끄러움, 생성 결과 관리 편의성, 방송 제작자 관점의 직관성이다.
