3. 기존 클라이언트 서비스 참고
3.1 참고 목적
클라이언트가 기존에 제작한 프롬프트 추출 사이트는 본 프로젝트의 경쟁 서비스로 비교하기보다는, 역프롬프트 기능의 출력 구조를 참고하기 위한 내부 자료로 보는 것이 적합하다.
즉, 기존 프롬프트 추출 사이트와 우리 서비스를 비교해 차별성을 찾는 것이 아니라, 해당 서비스에서 사용한 이미지 분석 방식과 프롬프트 출력 형식을 참고하여 우리 서비스의 역프롬프트 기능을 개선하는 방향으로 활용한다.

3.4 정리
기존 프롬프트 추출 사이트는 경쟁 서비스 비교 대상이 아니라, 역프롬프트 기능 설계에 참고할 내부 레퍼런스이다.
우리 서비스의 차별성은 Higgsfield, Magnific, Krea 같은 외부 생성형 AI 서비스들과 비교해 정리한다.

7.2 영상 생성 방식
영상 생성은 이미지 생성보다 시간이 오래 걸리므로 동기 호출 방식은 적합하지 않다.
따라서 fal.ai의 큐 기반 흐름을 사용하는 것이 적합하다.
submit
→ request_id 반환
→ status polling
→ completed 확인
→ result 조회

8.1 역프롬프트의 역할
역프롬프트는 사용자가 마음에 드는 이미지를 업로드했을 때, 해당 이미지를 재현하거나 변형 생성할 수 있도록 프롬프트를 추출하는 기능이다.

12.2 최종 정리
우리 서비스의 차별성은 특정 프롬프트 추출 사이트와 비교해서 찾는 것이 아니라, 이미지 생성 중심 서비스, 영상 생성 중심 서비스, 품질 개선 중심 서비스가 각각 분리되어 있다는 점에서 출발한다.
Krea는 이미지 생성에 강점이 있고, Magnific은 이미지 품질 개선에 강점이 있으며, Higgsfield는 영상 생성과 카메라 무브먼트에 강점이 있다.
반면 우리 서비스는 방송 제작자가 실제 작업 흐름에서 필요한 GPT Image 2.0 이미지 생성, 영상 생성, 라이브러리 저장을 하나의 흐름으로 묶는 것을 목표로 한다.
따라서 핵심 포지셔닝은 다음과 같다.
단일 기능형 AI 생성 서비스가 아니라, 방송 제작자를 위한 GPT Image 2.0 기반 이미지·영상 생성 통합 워크플로우 서비스
