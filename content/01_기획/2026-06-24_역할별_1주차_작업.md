# 역할별 1주차 작업

정리일: 2026-06-30
목적: 2주차 Pass를 준비하기 위한 파트별 1주차 작업 범위를 정리한다.

## 1. 1주차 목표

1주차는 기능을 많이 만드는 기간이 아니라, 2주차 Pass에 필요한 기반을 맞추는 기간이다.

| 목표 | 설명 |
| --- | --- |
| PRD 공유 | 제품 목표, 기능 범위, 평가 기준을 팀원에게 공유한다. |
| 이미지 생성 검증 | GPT Image 2.0 호출 가능 여부를 확인한다. |
| 저장 흐름 준비 | 생성 결과를 저장하고 다시 조회할 구조를 정한다. |
| 화면 구조 정리 | 이미지 생성, 결과, 라이브러리 화면의 기본 흐름을 정한다. |
| 비용 기준 준비 | 2026-07-01까지 API와 배포 비용 기준안을 준비한다. |

## 2. PM 작업

| 작업 | 산출물 |
| --- | --- |
| PRD 정리 | 생성형 AI 통합 엔진 PRD |
| MVP 범위 정리 | MVP 기능 범위 및 평가 기준 |
| API 비용 기준 정리 | API 비용 산정 문서 |
| 회의록 관리 | 킥오프 회의록, 결정 사항, 액션 아이템 |
| 파트별 질문 수집 | Frontend, Backend, UX/UI 고민사항 목록 |

## 3. Frontend 작업

| 작업 | 산출물 |
| --- | --- |
| 프로젝트 초기 구조 확인 | Next.js 화면 구조 초안 |
| 이미지 생성 화면 골격 | 프롬프트 입력, 옵션, 생성 버튼 |
| 결과 표시 영역 | 생성 이미지 카드 또는 결과 패널 |
| 라이브러리 화면 골격 | 저장 결과 목록 UI |
| 상태 UI 초안 | 빈 상태, 로딩, 실패, 완료 상태 |

## 4. Backend 작업

| 작업 | 산출물 |
| --- | --- |
| 이미지 생성 API 검토 | GPT Image 2.0 호출 테스트 |
| 저장 방식 검토 | S3, DB, Supabase/AWS 비교 |
| 영상 API 사전 조사 | Higgsfield 3~15초 호출 방식과 단가 확인 |
| 상태 조회 방식 검토 | request id 기반 polling 구조 후보 |
| 배포 구조 검토 | AWS EC2/S3 예상 구성과 비용 |

## 5. UX/UI 작업

| 작업 | 산출물 |
| --- | --- |
| 레퍼런스 확인 | 전달받은 레퍼런스 정리 |
| IA 초안 | 이미지 생성, 영상 생성, 라이브러리 흐름 |
| 디자인 시스템 논의 | 컬러칩, 타이포, 버튼, 상태 UI 기준 |
| 생성 화면 와이어프레임 | 프롬프트 입력과 결과 확인 흐름 |
| 라이브러리 화면 와이어프레임 | 이미지/영상 카드와 상세 보기 |

## 6. 1주차 일자별 초안

| 일차 | PM | Frontend | Backend | UX/UI |
| --- | --- | --- | --- | --- |
| 1일 | PRD와 평가 기준 공유 | 프로젝트 구조 확인 | API 키와 호출 방식 확인 | 레퍼런스 정리 |
| 2일 | Pass 흐름 확정 | 생성 화면 골격 | 이미지 생성 테스트 | IA 초안 |
| 3일 | 비용 산정 기준 정리 | 결과 표시 UI | 저장 구조 검토 | 디자인 시스템 논의 |
| 4일 | 파트별 이슈 정리 | 라이브러리 UI | 영상 API 사전 조사 | 상태 UI 설계 |
| 5일 | 2주차 작업 계획 정리 | 통합 연결 준비 | 배포 구조 검토 | 중간점검 화면 정리 |

## 7. 1주차 완료 기준

- 팀원이 2주차 Pass 흐름을 같은 기준으로 이해한다.
- GPT Image 2.0 호출 가능 여부가 확인된다.
- 저장과 라이브러리 구현 방향이 정리된다.
- UX/UI는 생성 화면과 라이브러리 화면의 기본 흐름을 제시한다.
- 수요일까지 API, 배포, 저장 비용 산정 기준안이 준비된다.
