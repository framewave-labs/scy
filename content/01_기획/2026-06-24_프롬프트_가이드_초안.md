1. 문서 목적
본 문서는 프로젝트 A에서 사용할 GPT Image 2.0 이미지 생성과 영상 생성 프롬프트의 작성 기준을 정리하기 위한 문서이다.
프로젝트 A는 사용자가 입력한 프롬프트를 그대로 API에 전달하는 단순 생성 서비스가 아니라, 생성 목적에 맞게 프롬프트를 정리하고 보강하여 결과물 품질을 높이는 것을 목표로 한다.
현재 프로젝트 기준 이미지 생성은 GPT Image 2.0을 사용하며, 실제 API 호출 모델명은 gpt-image-2로 통일한다.
역프롬프트는 핵심 MVP 이후 여유가 있을 때 구현하는 선택 기능으로 두며, 본 문서에서는 선택 기능으로 구현할 경우의 출력 방향만 정리한다.

5. 공통 Structured Metadata
5.1 Metadata 역할
Structured Metadata는 사용자의 입력 또는 이미지 분석 결과를 내부적으로 정리하기 위한 구조이다.
기존 프롬프트 추출 사이트처럼 SUBJECT, CLOTHING, SCENE 등을 나누는 방식은 최종 생성 프롬프트로 바로 쓰기보다는 내부 데이터로 활용하는 것이 적합하다.
이 구조를 사용하면 이미지 생성, 영상 생성, 선택 기능인 역프롬프트와 재생성 기능에서 동일한 정보를 재사용할 수 있다.
