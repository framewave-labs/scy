# 프롬프트 가이드 초안

정리일: 2026-06-30
목적: 이미지 생성과 3x3 그리드 기반 영상 생성에 필요한 프롬프트 작성 기준을 정리한다.

## 1. 공통 원칙

| 원칙 | 설명 |
| --- | --- |
| 장면 목적 먼저 | 무엇을 보여주려는 장면인지 먼저 쓴다. |
| 방송 제작 맥락 포함 | 프로그램 장르, 분위기, 촬영 의도를 함께 적는다. |
| 시각 요소 분리 | 장소, 인물, 구도, 조명, 색감, 분위기를 나눠 적는다. |
| 과도한 옵션 제한 | 모델별 세부 파라미터보다 장면 설명을 우선한다. |
| 재사용 가능성 | 생성 결과를 라이브러리에서 다시 찾을 수 있게 핵심 키워드를 남긴다. |

## 2. 이미지 프롬프트 구조

이미지 생성 화면은 메뉴 A(초기 레퍼런스)와 메뉴 B(키이미지) 두 단계로 나뉜다.

- **메뉴 A**: 간단한 프롬프트로 배경 1장 또는 캐릭터 다각도(정면·측면·후면) 1장을 빠르게 생성. 캐릭터/배경 복수 선택 가능(2.1·2.2절)
- **메뉴 B**: 완성도 높은 캐릭터 키이미지(=캐릭터 시트) 생성. 속성 선택 사이드바 또는 자유 텍스트 프롬프트, 둘 다 지원(2.3절)

두 메뉴 모두 "AI 프롬프트 교정"(가칭, 기능명 재검토 중) 토글을 켜면 한국어 입력을 Claude Sonnet이 GPT Image 2 최적화 영어 프롬프트로 변환해 화면에 보여준다.

### 2.1 배경 시트 (메뉴 A)

360도(다각도) 커버리지 불필요 — 특징적인 요소만 담으면 나머지는 자연스럽게 생성된다.

```text
[장소/공간]
[시간대]
[조명]
[색감]
[분위기]
[방송 장르 또는 촬영 톤]
```

예시:

```text
quiet old bookstore interior, late afternoon, warm sunlight through dusty windows,
soft shadows, muted brown and green palette, calm Korean drama mood,
wide establishing shot, photorealistic
```

### 2.2 캐릭터 다각도 초기 레퍼런스 (메뉴 A)

정면·측면·후면을 1장에 담아 빠르게 출력. 간단형 한 줄 프롬프트로 충분하며, 다각도 구성은 프롬프트 엔지니어링으로 처리한다(BE 별도 작업 불필요).

```text
[인물 정보]
[의상]
[표정]
[헤어/소품]
[기준 포즈]
[시각 스타일]
```

예시:

```text
a woman in her early 30s, simple beige trench coat, short black hair,
calm but tired expression, holding a notebook, neutral standing pose,
character design sheet showing front view, side view, and back view
arranged side by side, white background, photorealistic, high detail
```

### 2.3 캐릭터 키이미지 완성형 (메뉴 B)

한 이미지 안에 턴어라운드(정면/측면/후면), 표정 배리에이션, 액세서리·소품 디테일, 스타일·키컬러를 전부 담는다. 이후 생성 요청에 이 이미지를 계속 참조로 넣어 일관성을 유지한다. 장면 연출(포즈, 액션, 장면 맥락)은 이 이미지가 아니라 생성 시점 프롬프트 텍스트로 직접 지시한다.

메뉴 B는 아래 표준형 프롬프트를 자유 텍스트로 직접 입력하거나, 속성 선택 사이드바(파트1 캐릭터 기본/파트2 아트 스타일·세계관/파트3 헤어/파트4 얼굴/파트5 의상·악세서리)에서 값을 골라 자동 생성할 수 있다. 컬러 항목(헤어·눈·의상)은 프리셋이 아니라 컬러 팔레트 + 헥스코드 직접 입력으로 받는다.

**PD 제공 표준형 (2단 구성)**:

Part 1은 이미지 전체를 설명하는 자연어 문단, Part 2는 아래 필드로 구성된 구조화 블록이다.

| 필드 | 내용 |
| --- | --- |
| SUBJECT | 인물 기본 정보, 헤어/외형, 표정 배리에이션, 포즈(턴어라운드 각도) |
| CLOTHING & STYLING | 의상 아이템별 소재·색상(헥스코드 포함) |
| ACCESSORIES | 액세서리별 소재·색상 |
| SCENE | 장면/레이아웃 유형, 핵심 요소 |
| BACKGROUND | 배경 요소 (그리드, 텍스트 블록, 컬러 스와치 등) |
| LIGHTING | 조명 방식 |
| TECHNICAL | 촬영/렌더링 기술 정보 (해당 없으면 N/A) |
| COMPOSITION | 구도 방식 |
| STYLE | 아트 스타일, 주요 색상 |
| ASPECT RATIO | 출력 비율 |
| CONSTRAINTS - Must keep | 반드시 유지해야 할 요소 |
| CONSTRAINTS - Avoid | 피해야 할 것 |

전체 예시(원문)는 `2026-07-07_A2팀_키이미지_정의_PD확인.md`를 참고한다.

## 3. 영상 프롬프트 구조

영상 생성 입력은 3x3 스토리보드 그리드 이미지 1장과 scene 1~9 프롬프트다.

9개 프롬프트는 각각 독립 장면 설명이지만, 최종 영상 요청에서는 하나의 영상 맥락으로 함께 전달한다.

### 3.1 scene 프롬프트 형식

```text
Scene 1: [장면 설명], [카메라 움직임], [인물/대상 움직임], [감정/분위기]
Scene 2: ...
Scene 3: ...
Scene 4: ...
Scene 5: ...
Scene 6: ...
Scene 7: ...
Scene 8: ...
Scene 9: ...
```

### 3.2 작성 기준

| 항목 | 기준 |
| --- | --- |
| 순서 | 3x3 그리드의 좌상단부터 우하단 순서로 scene 1~9를 작성한다. |
| 길이 | 각 scene은 1~2문장으로 짧게 쓴다. |
| 카메라 | pan, dolly, zoom, handheld 등 움직임을 한 가지만 우선 쓴다. |
| 움직임 | 인물 또는 대상의 핵심 움직임을 명확히 쓴다. |
| 일관성 | 인물, 장소, 시간대, 색감이 갑자기 바뀌지 않게 한다. |
| 길이 옵션 | Higgsfield 기준 3~15초 범위에서 선택한다. |

## 4. 영상 프롬프트 예시

```text
Scene 1: A quiet bookstore exterior at dusk, slow push-in toward the entrance, warm lights turning on inside.
Scene 2: The woman enters the bookstore, handheld medium shot, she looks around cautiously.
Scene 3: Close-up of her hand touching an old notebook on a shelf, shallow depth of field.
Scene 4: A man appears at the end of the aisle, slow dolly shot, tense silence.
Scene 5: The woman freezes and looks up, close-up, soft warm light on her face.
Scene 6: The man steps closer but stops, medium shot, restrained emotion.
Scene 7: The notebook falls slightly open, insert shot, old photo visible for a moment.
Scene 8: Both characters stand apart in the aisle, wide shot, emotional distance emphasized.
Scene 9: The camera slowly pulls back through the shelves, quiet unresolved mood.
```

## 5. 역프롬프트 작성 기준

역프롬프트는 선택 기능이다. 레퍼런스 이미지를 분석해 아래 항목을 추출하는 것을 목표로 한다.

| 항목 | 설명 |
| --- | --- |
| style | 전체 시각 스타일 |
| lighting | 조명 방향과 강도 |
| composition | 구도와 프레이밍 |
| lens | 렌즈 느낌과 심도 |
| color_palette | 주요 색감 |
| mood | 감정과 분위기 |
| prompt_en | 재생성에 사용할 영문 프롬프트 |

## 6. 피해야 할 작성 방식

- 한 프롬프트에 서로 충돌하는 분위기를 많이 넣지 않는다.
- 영상 프롬프트에서 장면마다 인물 설정을 새로 바꾸지 않는다.
- 9개 scene을 9개의 독립 영상처럼 쓰지 않는다.
- 모델명과 옵션만 나열하고 장면 설명을 생략하지 않는다.
- 최종 후보가 아닌 테스트에서 긴 영상 길이를 기본값으로 쓰지 않는다.
