# A2팀 생성형 AI 통합 엔진 작업범위 정리

작성일: 2026-06-26
업데이트: 2026-06-30
작성자: 손찬용 PM

## 1. 문서 목적

이 문서는 A2팀이 먼저 만들 기능과 뒤로 미룰 기능을 구분한다.

평가 기준의 자세한 내용은 `MVP 기능 범위 및 평가 기준` 문서에서 관리하고, 이 문서는 작업 우선순위만 다룬다.

## 2. 작업 우선순위

| 우선순위 | 범위 | 설명 |
| --- | --- | --- |
| 1순위 | 2주차 Pass | 이미지 생성, 저장, 라이브러리 조회 |
| 2순위 | 4주차 A등급 | 3x3 그리드 기반 영상 생성, 영상 저장, 배포 |
| 3순위 | S등급 선택 | 역프롬프트, 프롬프트 수정, 이미지 재생성 |
| 4순위 | 보조 기능 | 다운로드, 검색, 필터 |

## 3. 2주차 작업범위

| 기능 | 포함 여부 | 메모 |
| --- | --- | --- |
| 이미지 프롬프트 입력 | 포함 | 배경 시트 또는 캐릭터 시트 기준 |
| GPT Image 2.0 이미지 생성 | 포함 | `gpt-image-2` 기준 |
| 이미지 결과 표시 | 포함 | 생성 결과를 화면에서 확인 |
| 이미지 저장 | 포함 | 라이브러리 저장 흐름 |
| 라이브러리 조회 | 포함 | 저장한 이미지를 다시 확인 |
| 영상 생성 | 제외 | 4주차 A등급 목표 |
| 역프롬프트 | 제외 | S등급 선택 기능 |

## 4. 4주차 작업범위

| 기능 | 포함 여부 | 메모 |
| --- | --- | --- |
| 키이미지 생성 | 포함 | 배경/캐릭터 기준 활용 |
| 3x3 그리드 기반 영상 생성 | 포함 | 그리드 이미지 1장 + 9개 씬 프롬프트 |
| 영상 상태 표시 | 포함 | 대기, 진행, 완료, 실패 |
| 영상 저장 및 조회 | 포함 | 라이브러리에서 이미지/영상 구분 |
| 배포 | 포함 | 프론트 Vercel Hobby, 백엔드 AWS EC2/S3 |
| 역프롬프트 | 선택 | A등급 기능 이후 |

## 5. 제외 범위

| 제외 항목 | 이유 |
| --- | --- |
| 스토리보드 그리드 생성 | B팀 범위로 보고 A2팀은 결과물을 입력으로 받는다. |
| 9개 개별 영상 생성 | 최신 기준은 단일 영상 생성 요청이다. |
| 영상 편집 타임라인 | 생성 이후 편집 도구는 MVP 범위가 아니다. |
| 실시간 이미지 캔버스 | 구현 난이도가 높아 4주 MVP에 맞지 않는다. |
| 팀 권한 관리 | 배포와 저장 구조가 안정된 뒤 검토한다. |

## 6. 작업범위 한 줄 정리

먼저 이미지 생성과 라이브러리를 완성하고, 그다음 3x3 그리드 기반 영상 생성과 배포로 확장한다. 역프롬프트는 선택 기능으로 관리한다.
