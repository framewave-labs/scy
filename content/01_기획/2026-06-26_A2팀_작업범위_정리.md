# A2팀 생성형 AI 통합 엔진 작업범위 정리

작성일: 2026-06-26  
작성자: 손찬용 PM  
공유 대상: A2팀  
문서 목적: 6월 30일 팀 작업 시작 전, 2주차 중간점검과 4주차 최종평가 기준에 맞춰 작업범위를 명확히 정리합니다.

---

## 1. 작업범위 정리 목적

이번 문서는 팀원들이 실제 개발에 들어가기 전, **무엇을 먼저 만들고 무엇을 나중으로 미룰지**를 명확히 하기 위한 기준 문서입니다.

현재 A2팀의 핵심 목표는 많은 기능을 한 번에 구현하는 것이 아니라, 평가 기준에 맞춰 **반드시 동작해야 하는 핵심 흐름을 먼저 완성하는 것**입니다.

따라서 작업범위는 아래 3단계로 나눕니다.

| 구분 | 의미 | 우선순위 |
| --- | --- | --- |
| 2주차 Pass 범위 | 중간점검을 통과하기 위해 반드시 완성해야 하는 범위 | 최우선 |
| 4주차 A등급 범위 | 최종평가에서 기본 목표로 삼는 범위 | 필수 목표 |
| S등급 도전 범위 | 핵심 기능이 안정화된 후 추가로 도전할 범위 | 선택 |

## 2. 현재 확정된 기준

### 2.1 이미지 생성 목적

이미지 생성 기능의 목적은 단순한 단일 이미지 생성이 아니라, 방송/영상 제작에 활용할 수 있는 **배경 시트와 캐릭터 시트를 먼저 만들고, 이를 기준으로 키이미지를 생성하는 것**입니다.

| 구분 | 의미 |
| --- | --- |
| 배경 시트 | 하나의 공간을 여러 구도에서 활용할 수 있도록 먼저 정리하는 배경 기준 이미지 |
| 캐릭터 시트 | 캐릭터의 정면, 측면, 후면, 3/4 구도, 표정, 의상 디테일 등을 먼저 정리하는 캐릭터 기준 이미지 |
| 키이미지 | 배경 시트와 캐릭터 시트를 기준 레퍼런스로 활용해 만드는 장면 대표 이미지 |
| 360도 활용 | 실제 3D 이미지를 만든다는 의미가 아니라, 어느 구도에서도 같은 공간/캐릭터처럼 보이도록 여러 시점의 레퍼런스를 확보하는 것 |

MVP에서는 완전한 3D 일관성을 보장하기보다, 프롬프트 구조화와 시트 이미지 생성 흐름을 통해 **배경과 캐릭터의 구도별 시각적 일관성을 높이고, 이를 바탕으로 키이미지를 생성하는 것**을 목표로 합니다.

### 2.2 이미지 생성 API

이미지 생성 API는 **GPT Image 2.0** 기준으로 진행합니다.

| 항목 | 결정 내용 |
| --- | --- |
| 이미지 생성 모델 | GPT Image 2.0 |
| API 기준 모델명 | `gpt-image-2` |
| 2주차 핵심 기능 | 프롬프트 기반 시트 이미지 생성 |
| 우선 구현 흐름 | 프롬프트 입력 → 배경/캐릭터 시트 이미지 생성 → 결과 표시 → 저장 → 라이브러리 조회 |

### 2.3 배포 기준

내부 테스트 배포는 **Railway** 기준으로 진행합니다.

초기 프로젝트 명세와 일부 기존 문서에는 Vercel 배포가 언급되어 있으나, 현재 운영 방향은 내부 테스트 단계에서 Railway를 사용하는 것으로 정리합니다. 최종 제출용 배포 플랫폼은 프로젝트 책임자와 확인 후 확정합니다.

| 항목 | 현재 기준 |
| --- | --- |
| 내부 테스트 배포 | Railway |
| 최종 제출 배포 | 프로젝트 책임자 확인 후 확정 |
| 기존 Vercel 언급 | 초기 명세 기준 항목으로 보고 재검토 |

### 2.4 영상 생성 방식

영상 생성은 하나의 긴 통짜 영상을 한 번에 만드는 방식이 아니라, **스토리보드 이미지 9컷과 각 컷에 해당하는 프롬프트를 기반으로 컷별 영상을 생성하는 방식**으로 정리합니다.

각 컷은 독립적인 영상 생성 단위로 관리합니다. 마음에 드는 컷은 유지하고, 마음에 들지 않는 컷만 프롬프트를 수정해 다시 생성할 수 있도록 하는 것이 목표입니다.

| 항목 | 기준 |
| --- | --- |
| 입력 | 스토리보드 이미지 9컷 + 각 컷별 프롬프트 |
| 생성 방식 | 9컷을 각각 개별 영상으로 생성 |
| 수정 방식 | 마음에 들지 않는 컷만 프롬프트 수정 후 재생성 |
| 유지 방식 | 마음에 드는 컷은 재생성하지 않고 그대로 유지 |
| 핵심 가치 | 전체 영상을 다시 만들지 않고 필요한 컷만 수정해 제작 효율을 높임 |

따라서 A2팀의 영상 생성 기능은 **9컷 스토리보드 기반 컷별 영상 생성/재생성 워크플로우**로 이해합니다.

### 2.5 작업 우선순위 선언

A2팀은 2주차 중간점검 Pass를 위해 아래 흐름을 최우선 작업범위로 합니다.

```text
프롬프트 입력
→ 배경/캐릭터 시트 이미지 생성
→ 결과 표시
→ 저장
→ 라이브러리 조회
```

영상 생성은 4주차 A등급 목표로 진행하되, 2주차 Pass 범위에는 포함하지 않습니다.  
역프롬프트와 이미지 재생성은 S등급 도전 기능으로 분리합니다.

## 3. 2주차 중간점검 Pass 범위

2주차 중간점검은 Pass / Fail 기준이므로, 아래 기능을 가장 먼저 완성해야 합니다.

### 3.1 필수 기능 범위

| ID | 기능 | 설명 | 완료 기준 | 평가 연결 |
| --- | --- | --- | --- | --- |
| M001 | 시트 이미지 프롬프트 입력 | 사용자가 배경 시트 또는 캐릭터 시트 생성용 프롬프트를 입력합니다. | 텍스트 입력 가능 | 핵심 기능 |
| M002 | 이미지 생성 API 호출 | `gpt-image-2` 기반 이미지 생성 API를 호출합니다. | 테스트 프롬프트로 이미지 생성 성공 | API 연동 |
| M003 | 이미지 결과 표시 | 생성된 시트 이미지를 화면에 표시합니다. | 생성 결과가 결과 영역에 표시 | 핵심 기능 |
| M004 | 이미지 저장 | 생성 이미지를 라이브러리에 저장합니다. | 저장 버튼 클릭 후 저장 상태 반영 | 프롬프트 → 이미지 → 저장 |
| M005 | 이미지 라이브러리 조회 | 저장된 이미지를 다시 조회합니다. | 라이브러리 화면에서 저장 이미지 확인 | 핵심 기능 |
| M006 | 기본 화면 설계 | 생성 화면, 결과 화면, 라이브러리 화면을 구성합니다. | 생성 흐름을 이해할 수 있는 UI 완성 | 기획 완성도 |

### 3.2 2주차 Pass 판단 기준

아래 흐름이 실제 화면에서 끊기지 않고 동작하면 2주차 중간점검 Pass 기준을 충족한 것으로 봅니다.

```text
1. 사용자가 배경 시트 또는 캐릭터 시트 생성용 프롬프트를 입력합니다.
2. 이미지 생성 버튼을 클릭합니다.
3. GPT Image 2.0 API를 호출합니다.
4. 생성된 시트 이미지가 결과 영역에 표시됩니다.
5. 사용자가 저장 버튼을 클릭합니다.
6. 저장 완료 상태가 표시됩니다.
7. 라이브러리 화면에서 저장된 이미지를 다시 확인합니다.
```

### 3.3 2주차에서 제외하는 기능

아래 기능은 중요하지만 2주차 Pass 기준에는 포함하지 않습니다.

| 기능 | 처리 기준 |
| --- | --- |
| 영상 생성 | 4주차 A등급 목표로 관리 |
| 영상 상태 조회 | 영상 생성 기능과 함께 3주차 이후 구현 |
| 영상 라이브러리 저장 | 이미지 라이브러리 완성 후 확장 |
| 역프롬프트 | S등급 도전 기능으로 분리 |
| 프롬프트 재생성 | 역프롬프트 이후 확장 |
| 다운로드 | 핵심 흐름 완료 후 선택 기능 |
| 검색·필터 | 라이브러리 기본 조회 완료 후 선택 기능 |

## 4. 4주차 A등급 목표 범위

4주차 최종평가에서는 A등급을 기본 목표로 합니다.

### 4.1 A등급 필수 기능

| ID | 기능 | 설명 | 완료 기준 | 평가 연결 |
| --- | --- | --- | --- | --- |
| A001 | 시트 이미지 및 키이미지 생성 | 배경 시트와 캐릭터 시트를 먼저 생성하고, 이를 기준으로 키이미지 생성 | 이미지 생성 및 화면 표시 | A / B / C |
| A002 | 영상 생성 | 스토리보드 9컷 기반으로 컷별 영상 생성 요청 | 컷별 영상 생성 request id 발급 | A |
| A003 | 영상 상태 조회 | 컷별 request id 기반으로 영상 생성 상태 확인 | 각 컷의 queued, processing, completed, failed 상태 표시 | A |
| A004 | 영상 결과 표시 및 재생성 | completed 상태에서 컷별 영상 미리보기 표시, 필요한 컷만 재생성 | 컷별 영상 URL 또는 미리보기 확인, 컷별 재생성 가능 | A |
| A005 | 미디어 라이브러리 | 이미지와 영상 저장 및 조회 | 이미지와 영상 카드 구분 표시 | A / B |
| A006 | 배포 | Railway 또는 최종 확정 플랫폼에 배포해 시연 가능한 상태 구성 | 데모 URL 접속 및 주요 흐름 확인 | A / B / S |

### 4.2 A등급 기준 사용자 흐름

```text
배경/캐릭터 시트 프롬프트 입력
→ 시트 이미지 생성
→ 시트 이미지를 기준으로 키이미지 생성
→ 이미지 저장
→ 라이브러리 조회
→ 스토리보드 이미지 9컷과 컷별 프롬프트 입력
→ 컷별 영상 생성 요청
→ 컷별 영상 상태 조회
→ 컷별 영상 결과 표시
→ 마음에 들지 않는 컷만 프롬프트 수정 후 재생성
→ 영상 저장
→ 라이브러리에서 이미지와 영상 조회
```

## 5. S등급 도전 범위

S등급 기능은 2주차 Pass 범위와 4주차 A등급 핵심 기능이 안정적으로 완료된 후 진행합니다.

| ID | 기능 | 설명 | 완료 기준 | 평가 연결 |
| --- | --- | --- | --- | --- |
| S001 | 역프롬프트 | 레퍼런스 이미지를 분석해 재현용 프롬프트를 추출합니다. | 이미지 업로드 후 프롬프트 추출 | S |
| S002 | 프롬프트 수정 | 추출된 프롬프트를 사용자가 수정합니다. | 수정 가능한 텍스트 영역 제공 | S |
| S003 | 이미지 재생성 | 수정된 프롬프트로 이미지를 다시 생성합니다. | 재생성 이미지 표시 | S |
| S004 | 재생성 결과 저장 | 재생성 이미지를 라이브러리에 저장합니다. | 라이브러리에서 재생성 결과 확인 | S |

S등급 기능은 프로젝트 완성도를 높이는 기능이지만, 2주차 Pass와 4주차 A등급 범위를 방해하지 않는 선에서 진행합니다.

## 9. 최종 정리

A2팀의 현재 작업범위는 다음과 같이 정리합니다.

| 단계 | 목표 | 핵심 범위 |
| --- | --- | --- |
| 2주차 Pass | 중간점검 통과 | 프롬프트 입력 → 배경/캐릭터 시트 이미지 생성 → 결과 표시 → 저장 → 라이브러리 조회 |
| 4주차 A등급 | 최종평가 기본 목표 | 시트 이미지 기반 키이미지 생성 + 9컷 컷별 영상 생성/재생성 + 미디어 라이브러리 + 배포 |
| S등급 도전 | 추가 완성도 확보 | 역프롬프트 → 프롬프트 수정 → 이미지 재생성 → 저장 |

팀 전체는 먼저 2주차 Pass 범위를 완성한 뒤, 영상 생성과 S등급 기능으로 확장합니다.
