## 기술·배포 기준

| 항목 | 기준 |
|------|------|
| 프론트엔드 | Vercel 기준 배포 |
| 백엔드 | AWS 기준 배포 |
| API 표현 | 문서의 `/api/...`는 기능 단위 엔드포인트 명칭이며, Next.js API Route로 단정하지 않음 |
| 백엔드 세부 구성 | AWS 내 세부 서비스 조합, 저장 구조, 데이터 모델은 BE 담당자가 확정 후 반영 |
| 배포 일정 | 3주차 중반에 FE+BE 공동 1차 배포 후, 4주차~최종주에는 기능 추가·안정화·핫픽스 중심으로 운영 |
