# A2팀 1주차 중간보고서
> 보고 기간: 2026.06.30(화) ~ 2026.07.03(금)
> 제출일: 2026.07.06(월)
> 작성자: 손찬용 (PM)

---

## 작성 기준

이 문서는 A2팀이 1주차 동안 진행한 팀 단위 업무를 정리하기 위한 중간보고서입니다.

v0.1(07.02까지 확인된 회의록, PRD, 마일스톤, IA 준비 문서 기준)에 이어, v0.2에서는 07.03 업무 종료 후 확인된 IA 협의 결과와 BE API·DB 명세서 정리 내용을 반영했습니다. 07.03 업무 중 실제 산출물로 확인되지 않은 항목(와이어프레임 등)은 추정으로 채우지 않고 "확인 필요"로 남겨두었습니다.

### 07.02(목) - 마일스톤 확정 범위 정리 및 IA 협의 준비

**PM**

| 업무 | 결과 |
|------|------|
| 마일스톤 v3.1~v3.2 업데이트 | 완료 - BE 2인 분담, AWS 기준, 3주차 중반 FE+BE 1차 배포 반영 |
| PRD v0.1 업데이트 | 완료 - PM/Backend 마일스톤 확정사항 반영 |
| 백엔드 일정 및 작업 범위 확인 | 완료 - 이미지 담당 / 영상 담당 분담 기준 정리 |
| UX/UI 협의 준비 | 완료 - 07.03 IA 및 와이어프레임 협의 항목 정리 |
| 화면 구성 개인 메모 작성 | 완료 - MVP 화면 구조, 라이브러리 중심 흐름, 커뮤니티 제외 범위 정리 |

**주요 결정 사항**
- 프론트엔드 배포는 Vercel, 백엔드 배포는 AWS 기준으로 정리
- 문서의 API 명칭은 기능 단위 엔드포인트로 표현하고, 특정 프레임워크 Route 구조로 단정하지 않음
- BE 작업은 이미지 담당 / 영상 담당으로 분리
- 세부 AWS 서비스 조합, 데이터 모델, API 요청·응답 형식, 저장 구조는 BE 명세서 확정 후 반영
- 커뮤니티, 게시글, 댓글, 추천 기능은 MVP에서 제외하고 추후 논의 항목으로 분리
- 생성 결과물은 게시글이 아니라 작업물 또는 라이브러리 항목으로 표현

**BE**
- 팀 레포 생성 및 브랜치 전략 확정
- API 명세서 작업 진행 중
- DB 명세서 및 저장 구조 초안 작업 진행 중
- AWS 백엔드 프로젝트 구조 세팅 착수

**UX/UI**
- 각 팀 디자이너 소통 및 와이어프레임 방향 논의
- 디자인 시스템 방향과 주요 화면 구조 논의 준비
- vivago.ai 등 생성형 AI 서비스 레퍼런스 리서치 진행

### 07.03(금) - IA 협의 결과 및 BE API·DB 명세서 정리

**PM**

| 업무 | 결과 |
|------|------|
| PM·UX/UI IA 협의 | 완료 - `A2팀_IA_초안_v0.1_20260703.md` 가안 기준으로 협의 진행, IA 1차 확정본 엑셀(`A2_IA_1차_확정본_20260703.xlsx`)로 정리 |
| IA 공유용 자료 제작 | 완료 - 팀 공유용 IA 다이어그램(html/svg/png) 정리 |
| BE API·DB 명세서 반영 | 완료 - 백엔드팀 Notion API/DB 명세서 캡처를 `scy`에 문서화 (`A2팀_API_DB_명세서`, v0.1, review 상태) |

**주요 결정 사항 (IA 협의)**
- 홈/이미지 생성/영상 생성/라이브러리/역프롬프트/내 프로필·설정으로 화면 구조 1차 확정
- 방향별 레퍼런스(캐릭터 시트·배경 시트)는 별도 화면이 아니라 이미지 생성 화면 안의 탭/섹션으로 포함
- 생성 결과물은 `게시글`이 아니라 `작업물`/`라이브러리 항목`으로 통일, 커뮤니티(게시글·댓글·추천)는 MVP 제외
- 라이브러리는 현재 히스토리 방향으로 정리하되, 좋아요/스크랩형 확장은 후속 안건으로 분리
- 역프롬프트는 S001~S004 S등급 도전 범위로 화면 구조에는 남겨둠

**미확정 / 후속 안건 (IA 기준)**
- 좌측 사이드바 고정 내비게이션 포함 여부
- 라이브러리를 히스토리 방향으로 유지할지, 좋아요/스크랩형으로 전환할지 및 백엔드 저장 정책
- 사전 프롬프트 가이드(네거티브 키워드 등)의 전역 기본값 여부
- 결과 공유(다운로드가 아닌 링크 공유) 방식의 공개 범위·만료·삭제 권한

**BE**
- API 명세서: 생성 3종(이미지/역프롬프트/영상), 상태 조회 1종, 파일 4종 총 9개 엔드포인트 정리
- DB 명세서: `MediaFile`(자산), `GenerateJob`(작업 상태: `PENDING`/`RUNNING`/`SUCCESS`/`FAILED`) 2개 테이블 정리
- 프론트엔드 연동 전 확인 필요 항목으로 요청/응답 스키마, 진행상태 조회 키, 업로드 필드명, 삭제 정책, 다운로드 응답 방식, 인증 여부 등 8개 항목을 문서에 명시

**2주차(07.06~) 시작 전 확인 필요 항목** (`A2팀_날짜별_확인질문_v0.2_20260701.md` 기준)
- 결제 완료 및 API 키 실사용 가능 여부
- 라이브러리 화면에서 이미지/영상 구분 및 상태 관리 방식(FE)
- Vercel↔AWS 연결 환경변수 기준(FE/BE)
- 생성 실패·저장 실패·조회 실패 에러 응답 형식 통일(BE)
