## 1. 이어받는 사람을 위한 핵심 요약

A2팀은 `KX GenStudio`라는 웹 기반 생성형 AI 제작 서비스를 기획하고 있다. 핵심 흐름은 이미지 생성, 결과 저장, 라이브러리 조회를 먼저 완성한 뒤, B서비스에서 받은 `3x3그리드로 구성된 1장의 이미지`와 영상 프롬프트를 입력해 짧은 영상 생성으로 확장하는 것이다.

현재 기준 문서는 `scy/content/01_기획/2026-06-24_생성형_AI_통합_엔진_PRD.md`이며, 2026-07-05 기준 v1.2까지 업데이트되어 있다. 1주차에는 PRD, 마일스톤, 회의록, IA 협의, API/DB 명세, 1주차 중간보고서가 정리되었다. 2주차 시작일인 2026-07-06에는 FE 복귀, API 키/결제 확인, UX/UI 와이어프레임 확인, FE-BE 연동 세부 스키마 확인을 먼저 처리해야 한다.

| 구분 | 현재 기준 |
| --- | --- |
| 서비스명 | KX GenStudio |
| 서비스 형태 | 웹 기반 생성형 AI 제작 서비스 |
| 2주차 Pass | 이미지 생성 → 결과 표시 → 저장 → 라이브러리 조회 |

## 5. 확정된 제품/기획 기준

### 5.1 제품 흐름

```text
홈
→ 이미지 생성
→ 생성 결과 확인
→ 라이브러리 저장
→ 라이브러리 조회
→ 3x3 스토리보드 이미지와 영상 프롬프트 입력
→ 영상 생성 요청
→ 진행상태 조회
→ 영상 결과 확인
→ 라이브러리 저장 및 재조회
```

| 단계 | 기능 |

### 6.2 현재 DB 기준

현재 상태값은 `PENDING`, `RUNNING`, `SUCCESS`, `FAILED` 기준이다.

### 6.3 FE 연동 전 반드시 확인할 것

| 확인 항목 | 이유 |
| --- | --- |
| 각 API의 request body, response body, error response | FE 타입과 에러 UI 설계에 필요 |
| `/api/video-status` 조회 키 | `job id`, `file id`, query parameter 여부 결정 필요 |
| 파일 업로드 필드명 | `multipart/form-data` 연결 기준 필요 |
| 파일 삭제 정책 | DB row만 삭제하는지, 실제 S3/로컬 파일까지 삭제하는지 확인 필요 |
| 다운로드 응답 방식 | redirect URL, stream, presigned URL 중 선택 필요 |
| 역프롬프트 저장 범위 | 반환만 할지 `MediaFile.reversed_prompt`에 저장할지 결정 필요 |
| 실제 물리 테이블명 | `MediaFile` 표기와 `media_file(id)` 표기 정렬 필요 |
| 인증/권한 | MVP에서 인증 없이 파일 접근을 허용할지 결정 필요 |
