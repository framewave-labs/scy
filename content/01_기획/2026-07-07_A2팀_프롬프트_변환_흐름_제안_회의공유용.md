# 프롬프트 변환 흐름 제안 (회의 공유용)
> 상세 버전: `2026-07-07_A2팀_프롬프트_변환_흐름_제안.md`
> 이 문서는 회의에서 바로 읽고 설명할 수 있게 쉬운 말로 요약한 버전입니다.

## 2026-07-07 결정된 것
- **이미지 생성**: 이 방향으로 진행 확정. 화면에는 **토글(켜고 끄기 버튼)** 로 "AI 프롬프트 변환" 기능을 켜고 끌 수 있게 만들기로 합의
- **영상 생성**: 적용 안 하기로 확정. 3x3 이미지랑 프롬프트를 이미 B서비스가 같이 주기 때문에, 영상 쪽은 그 프롬프트를 수정하는 기능만 있으면 됨
- 캐릭터 시트 "고정 요소" 기능은 아직 논의 못 함 — 다음 회의에서 계속

## 한 줄 요약
사용자가 한국어로 입력하면, 바로 이미지를 만들지 말고 **"이런 영어 프롬프트로 만들 거예요"를 한 번 보여주고 승인받은 다음** 이미지를 생성하자는 제안입니다.

## 왜 필요한가
- 지금 IA에는 "한국어 입력 → 바로 이미지 생성"만 있고, 중간 확인 단계가 없습니다.
- 번역이 이상하게 되면 이미지 생성까지 헛돈이 나갑니다 (이미지 1장 생성이 텍스트 변환보다 훨씬 비쌉니다).
- 캐릭터 시트·배경 시트처럼 "여러 장을 같은 느낌으로" 만들어야 하는 작업에서 특히 도움이 됩니다.

## 완전히 새로운 예산이 필요한 건 아닙니다
이미 비용 계획에 "프롬프트 변환 도와주는 AI 호출"이 들어가 있었습니다. 이번 제안은 그걸 **실제 화면 흐름으로 구체화**하는 것뿐입니다.

## 지금 IA에는 없습니다 (확인 완료)
7월 3일 확정한 IA 파일을 직접 확인했습니다. 이미지/영상 프롬프트 입력 항목 어디에도 이 "변환 확인" 단계가 없었습니다. 그래서 완전히 새로운 논의 주제입니다.

## 2주차 진행에는 영향 없습니다
이번 주(2주차) 통과 기준에는 "프롬프트 텍스트 입력만 되면" 통과입니다. 이 제안은 그 다음 단계(A등급 이후) 품질을 높이는 것이라, 지금 당장 결정 안 해도 2주차 일정에 문제없습니다.

## 어떻게 동작하나 (간단히)
```
1. 사용자가 한국어로 입력
2. AI가 영어 프롬프트로 정리해서 보여줌
3. 사용자가 그대로 쓰거나 직접 수정
4. 승인하면 그제서야 이미지 생성 시작
```
