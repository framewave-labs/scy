### 2.1 최대 생성 수량 — 해결: 수량 스테퍼 자체를 없앰

"수량 2~4장이 같은 캐릭터의 여러 버전인지, 다른 캐릭터 여러 명인지" 헷갈렸던 문제(구 버전 참고)를 수량 옵션 자체를 없애는 걸로 정리했다. **목적(캐릭터/배경)을 몇 개 선택했는지가 곧 출력 장수를 결정**한다 — 캐릭터만 선택하면 캐릭터 1장, 배경만 선택하면 배경 1장, 둘 다 선택하면 캐릭터 1장+배경 1장(총 2장). "여러 명의 다른 캐릭터를 한 번에" 같은 케이스는 이번 범위에 없음 — 필요해지면 후속으로 별도 설계.

기존에 있던 "1회 요청당 1~4장 스테퍼", "일일 한도" 관련 미해결 항목도 이걸로 자동 해소.

### 3.1 속성 선택 사이드바 — 손찬용 PM 검토 의견 (2026-07-09, 와이어프레임 v4 확인 후 갱신)

전체 구조는 승인. `이슬기_속성값_작업.png`(와이어프레임 v4, lsg)로 실제 옵션 목록까지 확인한 뒤 정리한 조정·확인 사항 5가지:

1. ~~파트2 세계관에 "사극·시대극" 추가 검토 필요~~ → **해결(2026-07-09): "역사" 옵션을 중세·서양 느낌으로 수정.** 무협·동양이 이미 동양 사극/시대극 계열을 커버하므로, "역사"는 겹치지 않게 서양·중세 쪽으로 방향을 잡음(이슬기 PM 반영).
2. **"필수/선택 구분 없음"의 실제 동작 방식 정정.** 와이어프레임을 보니 모든 항목에 기본값이 이미 채워져 있는 방식(예: 성별 기본값 남성, 나이대 기본값 20대)이라, "미선택 시 생략"이 아니라 **"항상 어떤 값이든 선택된 상태 — 사용자는 기본값을 바꾸기만 함"**이 정확한 동작. 이전 검토에서 "미선택 항목은 프롬프트에서 생략"이라고 정리했던 부분을 이걸로 정정.
3. ~~색상 프리셋(헤어·눈·의상)과 PD 표준형 템플릿의 헥스코드 표기 간 매핑은 후속 작업으로 분리~~ → **해결(2026-07-09 PM·BE 통합회의): 프리셋 라벨 대신 색상 항목을 헥스코드로 직접 선택하는 방식으로 변경.** 프리셋→헥스코드 매핑 작업 자체가 필요 없어짐.
4. ~~메뉴 A 결과물과의 연동은 이번 범위에서 제외, 인간형 캐릭터로 범위 한정~~ → **해결(2026-07-09): 이슬기 PM 의견 그대로 채택.** 후속안건 6번(레퍼런스 이미지 Vision 자동판단으로 속성 자동 세팅) 방향에 동의하되, 우선순위는 "메뉴 B 기본 구현 후 착수"(이슬기 PM 안 그대로) — 지금 범위에선 사용자가 속성을 처음부터 직접 선택. 인간형 캐릭터로 범위 한정하는 것도 그대로 유지, 비인간형은 후속안건 6번과 함께 확장.
5. **악세서리(파트5)는 복수 선택 가능, 나머지 항목은 전부 단일 선택.** 와이어프레임에 악세서리만 "복수 선택 가능"이라고 별도 표기돼 있음 — 다른 속성은 값 하나(문자열)로 API 요청을 받으면 되지만, 악세서리만 값 여러 개(배열)로 받아야 함. 이 차이를 놓치면 "안경+목걸이 동시 착용" 같은 조합을 표현 못 하는 버그가 생기므로 API 파라미터 설계 시 반영 필요(BE).
6. **미해결 — 상세 프롬프트 직접 입력과 속성 선택 사이드바의 관계.** IA v0.4에도 "텍스트 직접 입력 vs 속성 선택 사이드바 vs 혼합 — 미결정"으로 남아있던 부분. 아래 참고.

### 3.2 상세 프롬프트 직접 입력 vs 속성 선택 — 진행 방식 제안 (미해결)

**문제**: 사용자가 상세 프롬프트를 이미 다 써놨으면 속성 선택은 불필요하고, 안 써놨으면 속성 선택으로 채워야 하는데 — 이 둘을 어떻게 UI에서 자연스럽게 연결할지가 아직 안 정해짐.

**제안(택 1)**:

| 옵션 | 동작 | 장단점 |
| --- | --- | --- |
| A. 모드 전환(추천) | 상단에 "직접 입력" / "속성으로 만들기" 탭 2개. 직접 입력 탭이면 자유 텍스트 박스만 노출(속성 사이드바 숨김). 속성으로 만들기 탭이면 사이드바만 노출(자유 텍스트 없음) | 사용자 입장에서 헷갈릴 일이 없음(둘 중 뭘 쓰는지 명확). 다만 "일단 속성으로 만들다가 세부만 텍스트로 수정" 같은 혼합 사용은 못 함 |
| B. 동시 노출, 텍스트 우선 | 자유 텍스트 박스 + 속성 사이드바 둘 다 항상 보임. 텍스트 박스에 뭔가 입력돼 있으면 속성 사이드바는 무시(비활성화 표시), 비어있으면 속성 사이드바 값으로 프롬프트 자동 생성 | 유연하지만, "왜 내가 고른 속성이 반영 안 되지?" 하는 혼동이 생길 수 있어 UI 문구로 명확히 안내해야 함 |
| C. 사이드바만 유지, 자유 텍스트 없음 | 지금 와이어프레임 v4 그대로 — 자유 텍스트 입력창 자체가 없고 속성 선택만 있음 | 가장 단순하고 지금 이미 이렇게 설계돼 있음. 다만 "PD가 준 것 같은 완성된 프롬프트를 그대로 붙여넣고 싶은" 사용자는 못 씀 |

지금 와이어프레임 v4에는 자유 텍스트 입력창이 안 보여서(속성 선택만 있음) 사실상 C로 이미 가 있는 상태로 보입니다. A나 B로 바꾸려면 이슬기 PM님과 화면 구조를 다시 논의해야 하고, 그냥 C 그대로 갈 거면 이 항목은 "이미 해결됨"으로 정리하면 됩니다 — 어느 쪽으로 할지 확인 필요.
