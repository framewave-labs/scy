## 0. 전체 디자인 방향

Higgsfield를 레퍼런스로 따라가는 방향으로 진행. 상세 보기·공유 화면 포맷(6절)도 Higgsfield 예시 그대로 참고.

## 2. 이미지 생성 — 메뉴 A(초기 레퍼런스)

| 옵션 | 값 |
| --- | --- |
| 모델 | GPT Image 2 (고정, 선택 UI 불필요) |
| 목적 | 캐릭터 / 배경 — 복수 선택 가능. 캐릭터만 선택 시 캐릭터 1장, 배경만 선택 시 배경 1장, 둘 다 선택 시 캐릭터 1장 + 배경 1장 출력 (별도 수량 옵션 없음) |
| 품질 | 표준(medium, 기본) / 고화질(high, 옵션) |
| 비율 | Auto(기본) / 1:1 / 16:9 / 9:16 |
| 레퍼런스 첨부 | Style + Character + Add, 최대 8장, PNG/WEBP/JPG 50MB 이하 |
| AI 프롬프트 교정(기능명 재검토 중) | ON/OFF 토글. ON 시 Claude Sonnet이 변환한 GPT Image 2 최적화 프롬프트(한국어 입력 → 영어)를 화면에 보여줌. 디자이너 피드백: 한국어→영어로 갑자기 바뀌어 보이면 사용자가 당황할 수 있어 기능명을 다시 정하기로 함(아래 미해결) |
| 캐릭터 다각도 출력 | 정면·측면·후면을 1장에 담아 출력. 프롬프트 엔지니어링으로 처리(BE 별도 작업 불필요) |

### 2.1 "AI 프롬프트 교정" 기능명 후보

기존 이름의 문제는 "교정"이 단순 오탈자 수정 정도로 읽혀서, 실제로 한국어가 영어로 통째로 바뀌는 걸 사용자가 예상 못 한다는 것. 그래서 **변환이 일어난다는 걸 이름에서부터 알 수 있게** 하는 방향으로 후보를 다시 잡았다.

| 후보 | 방향 |
| --- | --- |
| 한국어 프롬프트 변환 | 가장 직관적 — 뭘 하는 기능인지 이름만 보고 바로 이해됨 |
| 한국어로 쓰기 | 사용자 관점의 혜택 중심("한국어로 편하게 입력하세요") — 토글 이름으로는 짧고 자연스러움 |
| AI 번역·최적화 | "번역"을 명시해서 언어가 바뀐다는 걸 예고 |
| 스마트 프롬프트 | 브랜드감은 있지만 언어 변환을 예고하지 못해 원래 문제 해결은 약함 |

**추천: "한국어로 쓰기"** — 토글 라벨은 짧아야 하는데, 이게 "이 기능을 켜면 한국어로 써도 된다"는 혜택을 가장 직관적으로 전달하면서 언어가 바뀐다는 것도 암시함.

## 3. 이미지 생성 — 메뉴 B(키이미지)

| 옵션 | 값 |
| --- | --- |
| 모델 | GPT Image 2 (고정) |
| 목적 | 키이미지 캐릭터시트 (표시만, 메뉴 진입으로 확정) |
| 품질 | 표준(medium) / 고화질(high) |
| 비율 | Auto / 1:1 / 16:9 / 9:16 |
| 수량 | 1장 고정, UI 미노출 |
| 입력 방식 | 속성 선택 사이드바 + 자유 텍스트 프롬프트 입력창, 둘 다 제공 |
| 속성 선택 사이드바 | 파트1 캐릭터 기본(성별·나이대·체형) / 파트2 아트 스타일·세계관(세계관 중 "역사"는 중세·서양풍, "무협·동양"은 동양 사극·시대극 계열) / 파트3 헤어 / 파트4 얼굴 / 파트5 의상·악세서리. 전체 선택형 — 모든 항목에 기본값이 미리 채워져 있고 사용자는 값을 바꾸기만 함 |
| 복수 선택 항목 | 악세서리만 복수 선택 가능(예: 안경+목걸이 동시 착용), 나머지 속성은 전부 단일 선택 |
| 컬러 항목(헤어색상·눈색상·의상색상) | 프리셋 라벨이 아닌 컬러 팔레트(스와치) + 헥스코드 직접 입력, 둘 다 제공 |
| 기본값 고정(UI 미노출) | 포즈: 정면+측면+후면 / 배경: 흰 배경 / 수량: 1장 |
| AI 프롬프트 교정(기능명 재검토 중) | ON/OFF. ON 시 Claude Sonnet이 변환한 최적화 프롬프트를 화면에 보여줌. 기능명 재검토 사유는 2절 참고 |
| 메뉴 A 결과물과의 연동 | 이번 범위 제외 — 사용자가 처음부터 속성을 직접 선택(자동 연동은 후속안건으로 별도 확장) |
| 캐릭터 범위 | 인간형 한정(성별: 남성/여성/젠더리스). 동물형·로봇형·수인형은 이번 범위 밖 |

| AI 프롬프트 교정(기능명 재검토 중) | ON/OFF. ON 시 Claude Sonnet이 변환한 최적화 프롬프트를 화면에 보여줌. 기능명 재검토 사유는 2절 참고 |

### 4.1 영상 생성 단가표 (fal.ai 공식 기준, 15초 영상 1개 기준)

| 모델 | 해상도 | 초당 단가 | 15초 영상 1개 |
| --- | --- | --- | --- |
| Kling 3.0 Standard | 1080p | $0.084/초 | $1.26 (약 1,700원) |
| Kling 3.0 Pro | 1080p | $0.112/초 | $1.68 (약 2,300원) |
| Kling 3.0 | 4K | $0.42/초 | $6.30 (약 8,700원) |
| Seedance 2.0 Mini | 480p | $0.043/초 | $0.65 (약 900원) |
| Seedance 2.0 | 720p | $0.3034/초 | $4.55 (약 6,300원) |
| Seedance 2.0 | 1080p | $0.682/초 | $10.23 (약 14,000원) |
| Seedance 2.0 | 4K | 비공개(Early Access, 별도 승인 필요 — 현재 일반 사용 불가) | — |

**Seedance는 720p→1080p로 한 단계만 올려도 비용이 2배 넘게 뜁니다.** Seedance 1080p($10.23/15초)는 Kling 4K($6.30)보다도 비쌉니다. 이 표가 아래 일일 생성 한도를 모델·해상도별로 나눈 이유입니다.

### 4.2 Kling 해상도 선택 UI 초안

문제: Seedance는 해상도가 한 모델 안에서 자유 선택 파라미터지만, Kling은 Standard/Pro/4K가 사실상 **서로 다른 모델**이라 해상도가 선택 파라미터가 아니라 모델 자체에 고정돼 있음. 그래서 Seedance와 같은 "해상도 드롭다운" UI를 그대로 못 씀.

**진행안**: v1(이번 스프린트)에서는 **Kling을 Standard 하나로 고정**하고 Pro·4K는 노출하지 않는다. Kling은 원래 "테스트/저비용" 포지션으로 잡아뒀던 모델이라 굳이 여러 해상도를 지원할 필요가 크지 않고, 이렇게 하면 UI도 단순해진다(Kling 선택 시 해상도 필드는 "1080p 고정"으로 읽기 전용 표시, Seedance 선택 시에만 해상도 드롭다운 활성화). Kling Pro·4K 노출은 필요해지면 후속으로 별도 설계.
