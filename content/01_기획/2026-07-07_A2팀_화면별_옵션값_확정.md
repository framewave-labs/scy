# A2팀 화면별 옵션값 확정
> 작성일: 2026-07-07 | 갱신: 2026-07-24 | 작성자: 손찬용 PM
> 목적: 와이어프레임에 들어갈 옵션값을 디자이너·BE가 한 곳에서 확인할 수 있도록 정리
> 상태: 아래 "8. 미해결 항목"만 남으면 v1.0(확정본)으로 승격

## 0. 전체 디자인 방향

Higgsfield를 레퍼런스로 따라가는 방향으로 진행. 상세 보기·공유 화면 포맷(6절)도 Higgsfield 예시 그대로 참고.

## 1. 홈 화면

| 항목 | 값 |
| --- | --- |
| 히어로 메시지 | 헤드라인: "방송 제작에 필요한 이미지와 영상, AI로 빠르게" / 서브텍스트: "캐릭터·배경·영상까지, 아이디어를 몇 분 안에 완성된 비주얼로" (1.1절 A안 확정) |
| 입력창 | 프롬프트 입력 → 칩(초기 레퍼런스 / 키이미지 / 영상) 선택 → 엔터 → 해당 메뉴로 이동, 프롬프트 자동 채움. 칩 미선택 시 기본값은 초기 레퍼런스(메뉴 A). 순수 이동(라우팅) 용도이며 여기서 직접 생성하지 않음 |
| 전체 구성 | 대시보드 중심. 좌측 사이드바 여부는 별도 논의 중 |

### 1.1 히어로 카피 후보

입력창이 이미 "어떤 이미지나 영상을 만들어볼까요?"로 액션을 유도하므로, 히어로 카피는 그거랑 겹치지 않게 **한 단계 위에서 가치 제안**을 하는 역할로 잡았다.

| 후보 | 헤드라인 | 서브텍스트 |
| --- | --- | --- |
| A. 베네핏 중심 | 방송 제작에 필요한 이미지와 영상, AI로 빠르게 | 캐릭터·배경·영상까지, 아이디어를 몇 분 안에 완성된 비주얼로 |
| B. 전문성 강조 | 제작진을 위한 AI 비주얼 제작 도구 | 방송·영상 콘텐츠에 바로 쓸 수 있는 이미지와 영상을 생성하세요 |
| C. 간결형 | 상상한 장면을 지금 만들어보세요 | 캐릭터, 배경, 영상 생성까지 한 곳에서 |

**확정: A안.** "방송 제작"이라는 실제 타깃 맥락을 헤드라인에 바로 넣어서 서비스 정체성이 분명하고, 서브텍스트가 캐릭터/배경/영상이라는 3대 기능을 자연스럽게 소개함.

## 2. 이미지 생성 — 메뉴 A(초기 레퍼런스)

| 옵션 | 값 |
| --- | --- |
| 모델 | GPT Image 2 (고정, 선택 UI 불필요) |
| 목적 | 캐릭터 / 배경 — 복수 선택 가능. 캐릭터만 선택 시 캐릭터 1장, 배경만 선택 시 배경 1장, 둘 다 선택 시 캐릭터 1장 + 배경 1장 출력 (별도 수량 옵션 없음) |
| 품질 | 표준(medium, 기본) / 고화질(high, 옵션) |
| 비율 | Auto(기본) / 1:1 / 16:9 / 9:16 |
| 레퍼런스 첨부 | Style + Character + Add, 최대 8장, PNG/WEBP/JPG 50MB 이하 |
| AI 프롬프트 변환 | ON/OFF 토글. ON 시 Claude Sonnet이 변환한 GPT Image 2 최적화 프롬프트(한국어 입력 → 영어)를 내부 생성 요청에 사용. **변환된 영어 프롬프트는 사용자 화면에 노출하지 않음(2026-07-14 확정)**. 명칭 확정 경위는 2.1절 참고 |
| 캐릭터 다각도 출력 | 정면·측면·후면을 1장에 담아 출력. 프롬프트 엔지니어링으로 처리(BE 별도 작업 불필요) |

### 2.1 토글 명칭 확정 — "AI 프롬프트 변환"

기존 이름의 문제는 "교정"이 단순 오탈자 수정 정도로 읽혀서, 실제로 한국어가 영어로 통째로 바뀌는 걸 사용자가 예상 못 한다는 것. 검토한 후보: 한국어 프롬프트 변환, 한국어로 쓰기, AI 번역·최적화, 스마트 프롬프트, 매직 프롬프트.

**최종 확정: "AI 프롬프트 변환" (2026-07-23, PM)** — 문서·화면 전체를 이 명칭으로 통일한다.

경위: 07-07에는 "프롬프트 향상"(Leonardo AI `Prompt Enhance`, Adobe Firefly `Enhance prompt` 계열의 한국어화)으로 정했으나, 실제 FE 화면은 초기 가칭 "AI 프롬프트 교정"이 그대로 남아 문서와 화면이 갈렸다. 07-16 디자이너 전달분과 유즈케이스 v1.2가 "AI 프롬프트 변환"을 쓰고 있어, 사용자가 실제로 보는 화면 기준에 맞춰 **"AI 프롬프트 변환"으로 최종 통일**한다.

> 폐기 표기: "AI 프롬프트 교정"(초기 가칭), "프롬프트 향상"(07-07~07-22 문서 표기). 신규 문서·UI에서는 사용하지 않는다.

## 4. 영상 생성 화면

| 옵션 | 값 |
| --- | --- |
| 모델 | Seedance 2.0 / Kling 3.0 — 카드 형태로 노출. **둘 다 스토리보드 영상 생성 용도(2026-07-13 확정)**, 역할 분담 없음 (기존 "데모용/테스트용" 포지셔닝은 폐기) |
| 길이 | Kling 3.0: 3~15초 / Seedance 2.0: 4~15초 — 모델별 최소값이 다름. 선택 모델이 지원 안 하는 값은 UI에서 비활성화 처리. **기본값: 5초(2026-07-24 확정 — 두 모델 모두 지원)** |
| 비율 | Seedance: `21:9, 16:9, 4:3, 1:1, 3:4, 9:16` (**`auto` 제거 — 2026-07-24 확정**) / Kling: `16:9, 9:16, 1:1` — **Kling도 비율 선택 UI 제공(2026-07-24 확정)**. 지원 안 하는 값은 UI에서 비활성화 처리. **기본값: 16:9** |
| 해상도 | Seedance 2.0: 사용자가 직접 선택 — **720p · 1080p · 4K 모두 사용 가능(2026-07-24 확인)**. Kling 3.0: v1에서는 Standard로 고정(1080p), 해상도 필드는 읽기 전용 표시 — 이유는 4.2절 참고 |
| Bitrate | High 고정, UI 미노출. BE 환경변수(demo/prod/showcase)로 제어. Kling은 Bitrate 선택지 자체가 없어 해당 없음 |
| 스토리보드 입력 | **9컷 3×3 그리드 이미지 1장 + 프롬프트 1건** 업로드. 그리드는 B서비스에서 실시간으로 전달받는 것이 아니라 **예시로 수령한 이미지 또는 자체 생성분**을 사용한다(2026-07-13 정정). 컷 순서는 좌상단=1번 → 우하단=9번이며 이 순서대로 scene/shot이 전개되고, 프롬프트는 변환 없이 그대로 생성에 투입한다. 자체 생성 시 컷 번호 오버레이가 영상에 새겨질 수 있으므로 **영상 입력용은 번호 없는 버전으로 생성**한다. 모델별 전달 방식은 4.3절 참고 |
| AI 프롬프트 변환 | **영상 화면에는 미적용 확정(2026-07-13)** — 07.07 합의(이미지 생성에만 적용) 및 B서비스 프롬프트 무변환 투입 전제에 따라 토글 없음 |

### 4.1 영상 생성 단가표 (fal.ai 공식 기준, 15초 영상 1개 기준)

| 모델 | 해상도 | 초당 단가 | 15초 영상 1개 |
| --- | --- | --- | --- |
| Kling 3.0 Standard | 1080p | $0.126/초 (`generate_audio: true` 확정 반영) | $1.89 (약 2,600원) |
| Kling 3.0 Pro | 1080p | $0.112/초 | $1.68 (약 2,300원) |
| Kling 3.0 | 4K | $0.42/초 | $6.30 (약 8,700원) |
| Seedance 2.0 Mini | 480p | $0.043/초 | $0.65 (약 900원) |
| Seedance 2.0 | 720p | $0.3034/초 | $4.55 (약 6,300원) |
| Seedance 2.0 | 1080p | $0.682/초 | $10.23 (약 14,000원) |
| Seedance 2.0 | 4K | $0.008/1000토큰 (**2026-07-24 fal 공식 확인 — 대기열·승인 없이 사용 가능**) | **$23.33 (약 36,200원)** |

**Seedance는 720p→1080p로 한 단계만 올려도 비용이 2배 넘게 뜁니다.** Seedance 1080p($10.23/15초)는 Kling 4K($6.30)보다도 비쌉니다. 이 표가 아래 일일 생성 한도를 모델·해상도별로 나눈 이유입니다.

**Seedance 4K 단가 구조(2026-07-24 확인)**: 토큰 단가는 4K가 $0.008/1000으로 720p·1080p($0.014/1000)보다 싸지만, 픽셀 수가 4배라 **총액은 1080p의 약 2.3배**다. 토큰 계산식은 `(가로 × 세로 × 초 × 24) ÷ 1024`. 5초 기준 4K는 약 $7.78. **fal.ai에 자동 차단 상한이 없으므로 4K는 최종 데모 하이라이트 1편에만 사용하고, 테스트·QA는 720p로 진행한다.**

### 4.2 Kling 해상도 선택 UI 초안

문제: Seedance는 해상도가 한 모델 안에서 자유 선택 파라미터지만, Kling은 Standard/Pro/4K가 사실상 **서로 다른 모델**이라 해상도가 선택 파라미터가 아니라 모델 자체에 고정돼 있음. 그래서 Seedance와 같은 "해상도 드롭다운" UI를 그대로 못 씀.

**진행안**: v1(이번 스프린트)에서는 **Kling을 Standard 하나로 고정**하고 Pro·4K는 노출하지 않는다. 이렇게 하면 UI가 단순해진다(Kling 선택 시 해상도 필드는 "1080p 고정"으로 읽기 전용 표시, Seedance 선택 시에만 해상도 드롭다운 활성화). 화면 라벨은 티어명 없이 "Kling 3.0"으로만 노출하고, 티어는 BE 엔드포인트 선택의 문제로 둔다. Kling Pro·4K 노출(MVP 제출본 고사양 구성 의견 포함)은 필요해지면 후속으로 별도 설계.

### 4.3 스토리보드 입력(3×3 그리드 + 프롬프트)의 모델별 전달 방식 — fal.ai API 확인 결과 (2026-07-13)

입력은 ① 9컷 3×3 그리드 이미지 1장(예시 수령분 또는 자체 생성분), ② 컷 순서(1=좌상단 → 9=우하단)대로 scene/shot을 서술한 프롬프트 1건이며, 프롬프트는 그대로 투입하는 것이 전제다. **Seedance·Kling 모두 스토리보드 영상 생성 용도로 확정(2026-07-13)** — 역할 분담 없이 두 모델 다 같은 입력을 받는다. **(2026-07-21 확정) 그리드 1장을 그대로 투입하는 방식으로 확정한다.** 서비스가 최종적으로 받는 입력 자체가 3×3 그리드 이미지 1장이므로 이를 9장으로 잘라 넣는 전처리는 기획 의도와 어긋난다고 판단해, **슬라이싱 기능은 개발하지 않는다**(2026-07-21 회의록 확정 사항 1). 07-13에 세워둔 "비교 테스트 결과에 따라 슬라이싱 개발 여부 결정" 갈림길은 **기획 판단으로 선행 종결**됐다. 모델별 전달 방식은 아래와 같다.

**Seedance 2.0 — `bytedance/seedance-2.0/reference-to-video` (3×3 그리드 1장 그대로 입력, 확정)**

- 3×3 그리드 이미지 **1장을 레퍼런스로 그대로 입력**하고, 컷 순서를 서술한 프롬프트 1건을 함께 투입한다. 엔드포인트는 `reference-to-video`를 유지한다(이미지 최대 9장 + 영상 3개 + 오디오 3개 지원 슬롯 중 이미지 1장만 사용).
- 오디오: `generate_audio` 기본 ON, OFF 선택 가능. **오디오 여부와 무관하게 요금 동일**(fal 공식 문서 명시).
- `fast/` 티어(저비용)도 별도 존재.

> **07-13 검토안 — 2026-07-21 미채택 (이력)**: 슬라이스한 9장을 순서대로 레퍼런스 입력하고 프롬프트에서 `@Image1`~`@Image9`로 컷을 참조하는 방식. 멀티샷 컷 전환(15초 내) 네이티브 지원으로 9컷 구조와 맞는다는 이점이 있었으나, 슬라이싱 미개발 확정으로 채택하지 않는다.

> **확인 필요 (BE)**: 위 방식에 딸려 있던 `@ImageN` 참조 형식 변환 전처리는 슬라이싱 미개발에 따라 불요로 보이나, 그리드 1장 입력 시 프롬프트를 무변환으로 그대로 넣는 것이 맞는지 BE 확인이 필요하다.

**Kling 3.0 Standard — `fal-ai/kling-video/v3/standard/image-to-video` (시작/끝 프레임 + 그리드 레퍼런스, BE 현행 방식)**

- 이미지 9장 동시 입력이 불가한 엔드포인트(레퍼런스 슬롯 최대 4장 — BE 확인, 2026-07-13). BE 현행 방식: **1번 컷 = `start_image_url`(시작 프레임), 9번 컷 = `end_image_url`(끝 프레임), 3×3 그리드 전체를 레퍼런스로 추가** 입력해 중간 컷의 시각 정보를 보완. 시작·끝은 보장되지만 중간 2~8번 컷 준수 수준은 비교 테스트로 검증.
- 컷N→컷N+1 세그먼트 체이닝(ffmpeg concat) 방식은 순서 100% 보장되나 최소 3초 × 8세그먼트 = 24초 과금 + 이음새 리스크로 **제외**.
- `multi_prompt`(샷별 prompt+duration)는 최대 6샷 + 총 15초 제한이라 9컷과 1:1 매핑 불가.
- `generate_audio` **기본값 true로 확정(2026-07-20)** — 소리 포함, UI 토글 미노출. 요금 ON $0.126/초(+50%, OFF 대비) 기준으로 4.1절 단가표 반영 완료.
- 비율·해상도 입력 파라미터 없음(Standard 1080p 고정). 출력 비율이 입력 이미지 비율을 따르는지 BE 검증 필요.
- **프롬프트 2,500자 이내 권장**(fal 공식 문서, 2026-07-21 확인). 9컷 서술형 프롬프트는 3,000자를 넘기 쉬우므로 Kling 투입 시 압축본을 사용한다(테스트 문서 시나리오 3 기준 1,789자).

> **해소(2026-07-23, BE·PM)**: 그리드 1번·9번 컷 추출 방식은 슬라이싱 미개발 확정과 별개 사안으로 정리됐다. 기획상 실사용이 요구되는 모델은 **Seedance 2.0**뿐이고, Kling은 단가가 저렴해 테스트용으로 곁다리로 추가한 옵션이라 이런 임시방편 구조(1·9번 컷 추출)로 들어간 것 — Kling 자체의 구조적 정합성을 Seedance와 동일 기준으로 맞출 필요는 없다.

**BE 비교 테스트 — 실행 완료**: 같은 스토리보드(번호 오버레이 없는 그리드)로 3건, 전부 **15초·720p** (4초에 9컷은 컷당 0.4초라 조건상 불가 — 컷당 1.5~2초 확보가 전제)로 요청한 테스트다.

> **해소(2026-07-23, PM)**: 이 절은 앞서 "비교 테스트 3건 결과 1번 채택"으로 기재돼 있었으나, 근거 문서와 맞지 않아 2026-07-22에 일단 "확인 필요"로 정정해뒀다. 이후 PM과 영상 개발 담당자(김유신)가 함께 결과를 확인한 결과, 테스트는 **실제로 실행·수령됐고** 최종적으로 **슬라이싱 미개발**로 확정됐다 — 07-21 회의록의 "기획 판단으로 선행 종결" 기록과 결론이 일치한다.
