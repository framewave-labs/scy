# A2팀 화면별 옵션값 확정
> 작성일: 2026-07-07 | 작성자: 손찬용 PM
> 목적: 와이어프레임에 들어갈 옵션값을 디자이너·BE가 한 곳에서 확인할 수 있도록 정리. 와이어프레임 링크와 함께 전달한다.
> 상태: 일부 값은 기존 문서에서 확정, 일부는 PM 결정 대기 — "결정 필요" 표시된 항목은 값이 채워지는 대로 갱신

## 1. 이미지 생성 화면

| 옵션 | 값 | 상태 |
| --- | --- | --- |
| 모델 | GPT Image 2 (고정, 선택 UI 불필요) | 확정 |
| 품질 | medium(기본) / high(옵션) | 확정 — 화면에 두 단계로 노출 |
| 비율 | _(값 목록 미정)_ | **결정 필요** |
| 최대 생성 수량 | _(1회 요청당 몇 장? 일일 한도?)_ | **결정 필요** |

## 2. 영상 생성 화면

| 옵션 | 값 | 상태 |
| --- | --- | --- |
| 모델 | Kling 3.0(테스트/저품질) / Seedance 2.0(데모/고품질) | 확정 — 사용자 노출 방식은 결정 필요 |
| 길이 | 3~15초 | 확정 — 화면에 몇 단계(예: 5/10/15초)로 쪼갤지는 결정 필요 |
| 비율 | _(값 목록 미정)_ | **결정 필요** |
| 품질 | _(모델 선택과 통합할지, 별도 옵션으로 둘지)_ | **결정 필요** |

## 3. 홈 화면

| 항목 | 내용 | 상태 |
| --- | --- | --- |
| 히어로 메시지 | _(PM 카피 작성 필요)_ | **결정 필요** |
| 전체 구성 | 대시보드 중심, 좌측 사이드바 여부는 IA 후속안건으로 별도 논의 중 | 참고 |

디자이너가 "홈 구성이 더 매력적이면 좋겠다"는 의견을 줬다 — 히어로 메시지 확정과 함께 이 부분 톤앤매너도 같이 정리해서 전달한다.

## 4. 전달 방식

- 이 문서를 와이어프레임 링크와 함께 Discord로 전달한다.
- "결정 필요" 항목은 값이 정해지는 대로 이 문서를 갱신하고, BE에게는 API 요청 파라미터(enum 값) 확정 시점에 별도로 공유한다.
