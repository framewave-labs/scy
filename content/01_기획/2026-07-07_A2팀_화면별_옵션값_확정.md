### 3.1 속성 선택 사이드바 — 손찬용 PM 검토 의견 (2026-07-09, 와이어프레임 v4 확인 후 갱신)

전체 구조는 승인. `이슬기_속성값_작업.png`(와이어프레임 v4, lsg)로 실제 옵션 목록까지 확인한 뒤 정리한 조정·확인 사항 5가지:

1. **파트2 세계관에 "사극·시대극" 추가 검토 필요.** 와이어프레임 확정 목록은 현대 도시/판타지 왕국/SF·우주/무협·동양/역사/포스트 아포칼립스. "역사"가 사극을 포괄하는 걸로 볼지, 별도 옵션으로 뺄지는 이슬기 PM 확인 필요 — 방송 제작 도구 특성상 사극 수요가 있을 것으로 예상되어 제안. 나머지 옵션은 그대로 유지.
2. **"필수/선택 구분 없음"의 실제 동작 방식 정정.** 와이어프레임을 보니 모든 항목에 기본값이 이미 채워져 있는 방식(예: 성별 기본값 남성, 나이대 기본값 20대)이라, "미선택 시 생략"이 아니라 **"항상 어떤 값이든 선택된 상태 — 사용자는 기본값을 바꾸기만 함"**이 정확한 동작. 이전 검토에서 "미선택 항목은 프롬프트에서 생략"이라고 정리했던 부분을 이걸로 정정.
3. ~~색상 프리셋(헤어·눈·의상)과 PD 표준형 템플릿의 헥스코드 표기 간 매핑은 후속 작업으로 분리~~ → **해결(2026-07-09 PM·BE 통합회의): 프리셋 라벨 대신 색상 항목을 헥스코드로 직접 선택하는 방식으로 변경.** 프리셋→헥스코드 매핑 작업 자체가 필요 없어짐.
4. **메뉴 A 결과물과의 연동은 이번 범위에서 제외, 인간형 캐릭터로 범위 한정.** 메뉴 A에서 이미 만든 캐릭터를 메뉴 B 속성에 자동 반영(이슬기 PM 후속안건 6번, Vision 자동판단)하는 건 이번엔 안 하고 사용자가 처음부터 다시 선택하는 걸로 확정. 파트1~5가 전부 사람 캐릭터 전제(성별: 남성/여성/젠더리스)라 동물형·로봇형·수인형은 이번 범위 밖 — 후속안건 6번과 함께 나중에 확장.
5. **악세서리(파트5)는 복수 선택 가능, 나머지 항목은 전부 단일 선택.** 와이어프레임에 악세서리만 "복수 선택 가능"이라고 별도 표기돼 있음 — 이 예외를 놓치지 않도록 API 파라미터 설계 시 반영 필요(BE).

## 4. 영상 생성 화면

| 옵션 | 값 | 상태 |

| 모델 | Seedance 2.0(기본, 데모/고품질) / Kling 3.0(테스트/저비용) — 카드 형태로 노출 | 확정 |
| 길이 | 모델별 최소값이 다름 — **Kling 3.0: 3~15초 / Seedance 2.0: 4~15초.** UI는 전체 범위(3~15초)를 노출하되, 선택한 모델이 지원 안 하는 값은 비활성화 처리(모델 변경 시 활성/비활성 갱신) | 확정 — 기존 "4~15초 고정" 표기 정정, 비율과 동일한 활성화/비활성화 방식으로 통일 |

| 해상도 | Kling 3.0: 4K 지원 확인됨(BE 실테스트) / Seedance 2.0: 4K는 fal.ai 문서상 별도 4K 모델로 존재하지만 **BE 테스트에서는 아직 미확인**. UI는 해상도 칩도 길이·비율과 동일하게 모델별 활성화/비활성화 방식으로 노출 | **미해결(Seedance 4K만)** — 관건은 **최종 시연 때 Seedance 4K를 실제로 쓸 수 있는지**. BE가 재확인 중이며, 안 되면 최종 시연도 1080p로 대체. 확인 전까지는 Seedance 4K 칩을 비활성 상태로 노출 |
| Bitrate | High 고정, UI 미노출. BE 환경변수(demo/prod/showcase)로 제어 | 확정 — Kling은 애초에 Bitrate 선택지 자체가 없어 해당 없음. Seedance는 Bitrate를 선택할 수 있는 모델이지만, UI에 노출하지 않고 BE에서 공통으로 High 고정 처리하는 방향으로 BE와 논의 중(같은 결론으로 수렴 중) |
| 스토리보드 입력 | B서비스가 제공하는 9컷 3×3 그리드 이미지 1장 업로드 | 확정 |
