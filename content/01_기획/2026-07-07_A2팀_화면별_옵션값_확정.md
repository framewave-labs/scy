# A2팀 화면별 옵션값 확정
> 작성일: 2026-07-07 | 갱신: 2026-07-09 | 작성자: 손찬용 PM
> 목적: 와이어프레임에 들어갈 옵션값을 디자이너·BE가 한 곳에서 확인할 수 있도록 정리
> 상태: 아래 "5. 미해결 항목"만 남으면 v1.0(확정본)으로 승격

## 1. 홈 화면

| 항목 | 값 |
| --- | --- |
| 히어로 메시지 | 온보딩·히어로 섹션은 만든다. 실제 카피만 미정(아래 미해결 참고) |
| 입력창 | 프롬프트 입력 → 칩(초기 레퍼런스 / 키이미지 / 영상) 선택 → 엔터 → 해당 메뉴로 이동, 프롬프트 자동 채움. 칩 미선택 시 기본값은 초기 레퍼런스(메뉴 A). 순수 이동(라우팅) 용도이며 여기서 직접 생성하지 않음 |
| 전체 구성 | 대시보드 중심. 좌측 사이드바 여부는 별도 논의 중 |

## 2. 이미지 생성 — 메뉴 A(초기 레퍼런스)

| 옵션 | 값 |
| --- | --- |
| 모델 | GPT Image 2 (고정, 선택 UI 불필요) |
| 목적 | 캐릭터 / 배경 — 복수 선택 가능. 캐릭터만 선택 시 캐릭터 1장, 배경만 선택 시 배경 1장, 둘 다 선택 시 캐릭터 1장 + 배경 1장 출력 (별도 수량 옵션 없음) |
| 품질 | 표준(medium, 기본) / 고화질(high, 옵션) |
| 비율 | Auto(기본) / 1:1 / 16:9 / 9:16 |
| 레퍼런스 첨부 | Style + Character + Add, 최대 8장, PNG/WEBP/JPG 50MB 이하 |
| AI 프롬프트 교정 | ON/OFF 토글. ON 시 Claude Sonnet이 변환한 GPT Image 2 최적화 프롬프트(한국어 입력 → 영어)를 화면에 보여줌 |
| 캐릭터 다각도 출력 | 정면·측면·후면을 1장에 담아 출력. 프롬프트 엔지니어링으로 처리(BE 별도 작업 불필요) |

## 3. 이미지 생성 — 메뉴 B(키이미지)

| 옵션 | 값 |
| --- | --- |
| 모델 | GPT Image 2 (고정) |
| 목적 | 키이미지 캐릭터시트 (표시만, 메뉴 진입으로 확정) |
| 품질 | 표준(medium) / 고화질(high) |
| 비율 | Auto / 1:1 / 16:9 / 9:16 |
| 수량 | 1장 고정, UI 미노출 |
| 속성 선택 사이드바 | 파트1 캐릭터 기본(성별·나이대·체형) / 파트2 아트 스타일·세계관(세계관 중 "역사"는 중세·서양풍, "무협·동양"은 동양 사극·시대극 계열) / 파트3 헤어 / 파트4 얼굴 / 파트5 의상·악세서리. 전체 선택형 — 모든 항목에 기본값이 미리 채워져 있고 사용자는 값을 바꾸기만 함 |
| 복수 선택 항목 | 악세서리만 복수 선택 가능(예: 안경+목걸이 동시 착용), 나머지 속성은 전부 단일 선택 |
| 컬러 항목(헤어색상·눈색상·의상색상) | 프리셋 라벨이 아닌 헥스코드(hex code)를 직접 선택 |
| 기본값 고정(UI 미노출) | 포즈: 정면+측면+후면 / 배경: 흰 배경 / 수량: 1장 |
| AI 프롬프트 교정 | ON/OFF. ON 시 Claude Sonnet이 변환한 최적화 프롬프트를 화면에 보여줌 |
| 메뉴 A 결과물과의 연동 | 이번 범위 제외 — 사용자가 처음부터 속성을 직접 선택(자동 연동은 후속안건으로 별도 확장) |
| 캐릭터 범위 | 인간형 한정(성별: 남성/여성/젠더리스). 동물형·로봇형·수인형은 이번 범위 밖 |

## 4. 영상 생성 화면

| 옵션 | 값 |
| --- | --- |
| 모델 | Seedance 2.0(기본, 데모/고품질) / Kling 3.0(테스트/저비용) — 카드 형태로 노출 |
| 길이 | Kling 3.0: 3~15초 / Seedance 2.0: 4~15초 — 모델별 최소값이 다름. 선택 모델이 지원 안 하는 값은 UI에서 비활성화 처리 |
| 비율 | Seedance: `auto, 21:9, 16:9, 4:3, 1:1, 3:4, 9:16`(전부 지원) / Kling: `16:9, 9:16, 1:1`. 지원 안 하는 값은 UI에서 비활성화 처리 |
| 해상도 | Kling 3.0: 4K까지 지원 / Seedance 2.0: 720p·1080p 지원, 4K는 아직 미확인(아래 미해결 참고). 해상도도 길이·비율과 동일하게 모델별 활성화/비활성화 방식으로 노출 |
| Bitrate | High 고정, UI 미노출. BE 환경변수(demo/prod/showcase)로 제어. Kling은 Bitrate 선택지 자체가 없어 해당 없음 |
| 스토리보드 입력 | B서비스가 제공하는 9컷 3×3 그리드 이미지 1장 업로드 |
| AI 프롬프트 교정 | ON/OFF. ON 시 Claude Sonnet이 변환한 최적화 프롬프트를 화면에 보여줌 |

## 5. 미해결 항목 (v1.0 승격 전 처리)

| # | 항목 | 필요한 조치 | 담당 |
| --- | --- | --- | --- |
| 1 | 이미지·영상 일일 생성 한도 | API 비용 운영안과 연동해 크레딧 소진 방지용 소프트 캡 결정 | PM |
| 2 | Claude Sonnet 프롬프트 변환 레이어 실제 연동 시점 | BE 개발 일정 검토 결과 대기 | BE |
| 3 | Seedance 2.0 4K 해상도 실제 지원 여부 | BE 재테스트. 최종 시연에 쓸 수 있는지가 관건 — 안 되면 1080p로 대체 | BE |
| 4 | 메뉴 B 입력 방식 확정 | 현재는 속성 선택 사이드바만 있고 자유 텍스트 프롬프트 입력창은 없음 — 이대로 갈지, 자유 입력을 추가할지 결정 | PM |
| 5 | 홈 화면 히어로 메시지 카피 | 온보딩·히어로 섹션은 만드는 걸로 확정됐으니 실제 문구 작성 필요 | PM |

## 6. 전달 방식

- 위 미해결 항목이 정리되면 v1.0으로 승격하고, 와이어프레임 링크와 함께 Discord로 전달한다.
- BE에게는 API 요청 파라미터(enum 값) 확정 시점에 별도로 공유한다.
