### 2.1 데일리 스크럼 스캔 범위 — 결정 대기 중 (사용자가 "내일부터 진행" 이라고 함)

문제: `build-scrum.mjs`가 최신 회의록 1개만 읽어서, 06-30~07-03 회의록의 미완료 액션 아이템(전부 "예정" 상태로 방치, 실제로는 이미 끝난 일들)이 스캔 범위에 안 걸린다. 제안했던 개선안(전체 회의록 스캔 + 상태 "완료" 아닌 것만 필터)을 테스트해봤더니 06-30~07-03 회의록까지 걸려서 47개 항목이 쏟아지는 노이즈 문제가 나왔다.

**두 가지 옵션을 사용자에게 제시해뒀고 아직 답을 못 받았다**:
1. 06-30~07-03 회의록 4개를 일괄 완료 처리
2. 스캔 범위를 07-06 이후로 제한 (이 세션이 추천한 옵션)

사용자가 "내일부터 진행"이라고 했으니, **다음 세션 시작할 때 먼저 이 결정을 물어보고 진행할 것.** 구현 코드는 이미 한 번 짜봤다(전체 스캔 버전) — 필요하면 아래 로직을 참고해서 다시 만들면 된다.

```js
// section 2를 이렇게 교체하면 "전체 스캔 + 완료 제외" 방식이 된다
const meetingFiles = fs.readdirSync(meetingDir)
  .filter(f => /^A2팀_\d{4}-\d{2}-\d{2}_회의록\.md$/.test(f))
  .sort();
// (07-06 이후로 제한하려면 여기서 파일명 날짜 필터 추가)
// 각 파일의 "## 액션 아이템" 표를 파싱해서 상태가 정확히 "완료"가 아닌 행만 모은다
```

**주의**: 지금 `scripts/build-scrum.mjs`는 git상 깨끗한(수정 전) 상태로 되돌려뒀다. 로컬에 반쯤 고친 코드가 남아있지 않으니 새로 짜야 한다.
