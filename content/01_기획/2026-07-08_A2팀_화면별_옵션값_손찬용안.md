## 0. 조사 근거

값을 감으로 정하지 않고, 우리보다 먼저 이 문제를 푼 서비스와 실제 API 제약을 같이 봤다.

| 항목 | 조사 내용 | 출처 |
| --- | --- | --- |
| 이미지 비율 | GPT Image 2는 WIDTH×HEIGHT 임의 지정 가능(16의 배수, 비율 1:3~3:1, 최대 3840×2160) — UI 프리셋은 자유이나 실제 API 호출 시 이 값으로 변환 필요 | [OpenAI API Reference](https://developers.openai.com/api/reference/resources/images/methods/generate) |
| 이미지 품질 | GPT Image 2는 low/medium/high/auto 지원 | 상동 |
| 이미지 수량(n) | GPT Image 2 API에 공식 상한 없음(다만 n은 "같은 프롬프트의 랜덤 샘플"이라 서로 다른 결과를 보장하지 않음 — 다각도 시트처럼 확실히 다른 결과가 필요하면 n보다 프롬프트 엔지니어링이 더 중요) | [Apiyi GPT Image 멀티이미지 가이드](https://help.apiyi.com/en/gpt-image-multi-image-generation-consistency-guide-en.html) |
| 이미지 비율 프리셋 관행 | Higgsfield는 16:9/9:16/1:1/4:3/3:4/21:9/2.39:1까지 폭넓게 제공(영상 기준이지만 이미지 툴들도 유사 관행) | [Higgsfield AI Video](https://higgsfield.ai/ai-video) |
| 업스케일/해상도 옵션 관행 | Magnific은 2K/4K 프리셋 + Creative/Precision 모드 구분(크리에이티브 자유도 vs 원본 충실도) — 우리 "품질" 옵션의 medium/high 구분과 유사한 사고방식 | [Magnific Image Upscaler](https://www.magnific.com/ai/image-upscaler) |
| 영상 해상도(Seedance 2.0) | fal.ai 기준 480p/720p(Fast)·1080p(Standard 티어)·4K(전용 4K 모델) 실제 지원 확인됨 — 이슬기 PM 안의 "BE 확인 필요" 항목이 해소됨 | [fal.ai Seedance 2.0](https://fal.ai/seedance-2.0) |
| 영상 비율(Seedance 2.0) | fal.ai 공식 스펙상 16:9/9:16/1:1/21:9/auto — **4:3·3:4는 확인 안 됨** (이슬기 PM 안에는 포함되어 있어 크로스체크 필요) | [fal.ai Seedance 2.0 API](https://fal.ai/models/bytedance/seedance-2.0/image-to-video/api) |
| 영상 길이 | fal.ai 기준 4~15초 또는 auto | 상동 |

## 1. 이미지 생성 화면

| 옵션 | 제안값 | 근거 |
| --- | --- | --- |
| 모델 | GPT Image 2 (고정) | 기존 확정 유지 |
| 품질 | medium(기본) / high(옵션) | 기존 확정 유지. Magnific의 Creative/Precision처럼 "빠른 기본값 vs 정교한 옵션" 2단계면 충분 — 3단계 이상은 방송 제작 현장에서 선택 피로만 늘림 |
| 비율 | **Auto(기본) / 1:1 / 16:9 / 9:16** | 방송 제작 실사용 시나리오(가로 영상 삽입용 16:9, 숏폼용 9:16, SNS 썸네일용 1:1)로 좁혔다. Higgsfield처럼 4:3·21:9까지 넓히는 건 지금 단계에서 과함 — 필요해지면 후속 확장 |
| 최대 생성 수량 | **1회 요청당 1~4장(+/- 스테퍼), 일일 한도는 별도 크레딧 정책 문서 기준** | GPT Image 2 n에 공식 상한이 없어 UI 상한은 비용/레이턴시로 정함. 단, n은 "같은 프롬프트의 유사 변형"이라 캐릭터 다각도처럼 확실히 다른 구도가 필요한 경우 n을 늘리는 게 아니라 프롬프트 자체(턴어라운드 지시)로 해결해야 한다 — 이 점은 프롬프트 가이드에 명시 필요 |

## 2. 영상 생성 화면

| 옵션 | 제안값 | 근거 |
| --- | --- | --- |
| 모델 | Seedance 2.0(기본, 데모/고품질) / Kling 3.0(테스트/저비용) | 기존 확정 방향 유지 — 카드 형태로 노출 |
| 길이 | 4~15초, 5초 단위 스테퍼(4·9·15) 또는 자유 입력 | fal.ai 실제 지원 범위가 4~15초라 기존 "3~15초"에서 하한을 4초로 정정 필요 |
| 비율 | Seedance: **Auto/16:9/9:16/1:1/21:9** (fal.ai 공식 스펙 기준, 4:3·3:4 제외) / Kling: Auto/16:9/9:16/1:1 | 이슬기 PM 안의 4:3·3:4는 fal.ai 공식 문서에서 확인이 안 돼 제외했다 — **BE가 실제 파라미터 검증 후 최종 확정 필요** |
| 해상도 | 720p(기본, Fast 티어) / 1080p(옵션, Standard 티어) / 4K(시연 전용, 별도 4K 모델) | fal.ai 확인 결과 Seedance 2.0이 1080p·4K를 실제로 지원 — 이슬기 PM 안의 "BE 확인 필요" 해소. 단 4K는 별도 모델/비용 구조라 데모·프로덕션과 분리해 "시연 전용"으로 한정 추천 |
| 품질(Bitrate) | High 고정, UI 미노출 | 기존 이슬기 PM 안과 동일 — BE 환경변수(demo/prod/showcase)로 제어하는 방식에 동의 |

## 4. 이슬기 PM 안과의 주요 차이점 요약

| 항목 | 이슬기 PM 안 | 손찬용 안 | 비고 |
| --- | --- | --- | --- |
| 이미지 수량 상한 | 1~4장 | 1~4장 | 동일 — 단, n의 성격(랜덤 샘플)에 대한 주의사항 추가 |
| 영상 길이 하한 | 명시 안 됨(모델별 표로만 제시) | 4초부터(fal.ai 실제 하한 반영) | 기존 "3~15초" 표기 정정 필요 |
| 영상 비율(Seedance) | Auto/16:9/9:16/4:3/3:4/1:1/21:9 | Auto/16:9/9:16/1:1/21:9 | **4:3·3:4 포함 여부 — BE 크로스체크 필요** |

## 5. 아직 결정 안 된 것

- 영상 비율 4:3/3:4 포함 여부 — BE가 fal.ai 콘솔에서 직접 확인 후 확정
- 이미지·영상 일일 생성 한도(크레딧 소진 방지용 소프트 캡) — API 비용 운영안과 연동해서 별도 결정 필요
