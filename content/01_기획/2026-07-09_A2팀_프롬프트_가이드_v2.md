## 1. 공통 원칙

| 원칙 | 설명 |
| --- | --- |
| 장면 목적 먼저 | 무엇을 보여주려는 장면인지 먼저 쓴다. |
| 방송 제작 맥락 포함 | 프로그램 장르, 분위기, 촬영 의도를 함께 적는다. |
| 시각 요소 분리 | 장소, 인물, 구도, 조명, 색감, 분위기를 나눠 적는다. |
| 과도한 옵션 제한 | 모델별 세부 파라미터보다 장면 설명을 우선한다. |
| 재사용 가능성 | 생성 결과를 라이브러리에서 다시 찾을 수 있게 핵심 키워드를 남긴다. |
| 프롬프트 최대 길이 | GPT Image 2: 최대 32,000자(OpenAI 공식 API 스펙). Seedance 2.0: 공식 문서에 글자 수 제한 명시 없음(제한 없는 문자열). "2~4문장"류 권장치는 기술적 한계가 아니라 가독성·반영률을 위한 실용적 가이드라인 — PD 표준형 같은 긴 구조화 프롬프트도 두 모델 지원 범위 안에 있다 |

## 2. 이미지 프롬프트 구조

이미지 생성 화면은 메뉴 A(초기 레퍼런스)와 메뉴 B(키이미지) 두 단계로 나뉜다.

- **메뉴 A**: 간단한 프롬프트로 배경 1장 또는 캐릭터 다각도(정면·측면·후면) 1장을 빠르게 생성. 캐릭터/배경 복수 선택 가능
- **메뉴 B**: 완성도 높은 캐릭터 키이미지(=캐릭터 시트) 생성. 속성 선택 사이드바 또는 자유 텍스트 프롬프트, 둘 다 지원

두 메뉴 모두 "AI 프롬프트 교정"(가칭, 기능명 재검토 중) 토글을 켜면 한국어 입력을 Claude Sonnet이 GPT Image 2 최적화 영어 프롬프트로 변환해 화면에 보여준다.

```text
[조명]
[색감]
[분위기]
```

### 2.2 캐릭터 다각도 초기 레퍼런스 (메뉴 A)

정면·측면·후면을 1장에 담아 빠르게 출력. 간단형 한 줄 프롬프트로 충분하며, 다각도 구성은 프롬프트 엔지니어링으로 처리한다(BE 별도 작업 불필요).

```text
[인물 정보]
[의상]
[표정]
[헤어/소품]
[기준 포즈]
[시각 스타일]
```

예시:

```text
a woman in her early 30s, simple beige trench coat, short black hair,
calm but tired expression, holding a notebook, neutral standing pose,
character design sheet showing front view, side view, and back view
arranged side by side, white background, photorealistic, high detail
```


### 2.3 캐릭터 키이미지 완성형 (메뉴 B)

한 이미지 안에 턴어라운드(정면/측면/후면), 표정 배리에이션, 액세서리·소품 디테일, 스타일·키컬러를 전부 담는다. 이후 생성 요청에 이 이미지를 계속 참조로 넣어 일관성을 유지한다. 장면 연출(포즈, 액션, 장면 맥락)은 이 이미지가 아니라 생성 시점 프롬프트 텍스트로 직접 지시한다.

메뉴 B는 아래 표준형 프롬프트를 자유 텍스트로 직접 입력하거나, 속성 선택 사이드바(파트1 캐릭터 기본/파트2 아트 스타일·세계관/파트3 헤어/파트4 얼굴/파트5 의상·악세서리)에서 값을 골라 자동 생성할 수 있다. 컬러 항목(헤어·눈·의상)은 프리셋이 아니라 컬러 팔레트 + 헥스코드 직접 입력으로 받는다.

**PD 제공 표준형 (2단 구성)**:

Part 1은 이미지 전체를 설명하는 자연어 문단, Part 2는 아래 필드로 구성된 구조화 블록이다.

| 필드 | 내용 |
| --- | --- |
| SUBJECT | 인물 기본 정보, 헤어/외형, 표정 배리에이션, 포즈(턴어라운드 각도) |
| CLOTHING & STYLING | 의상 아이템별 소재·색상(헥스코드 포함) |
| ACCESSORIES | 액세서리별 소재·색상 |
| SCENE | 장면/레이아웃 유형, 핵심 요소 |
| BACKGROUND | 배경 요소 (그리드, 텍스트 블록, 컬러 스와치 등) |
| LIGHTING | 조명 방식 |
| TECHNICAL | 촬영/렌더링 기술 정보 (해당 없으면 N/A) |
| COMPOSITION | 구도 방식 |
| STYLE | 아트 스타일, 주요 색상 |
| ASPECT RATIO | 출력 비율 |
| CONSTRAINTS - Must keep | 반드시 유지해야 할 요소 |
| CONSTRAINTS - Avoid | 피해야 할 것 |

전체 예시(원문)는 `2026-07-07_A2팀_키이미지_정의_PD확인.md`를 참고한다.

## 3. 영상 프롬프트 구조

영상 생성 입력은 3x3 스토리보드 그리드 이미지 1장과 scene(=shot) 1~9 프롬프트다. 기본 모델은 Seedance 2.0(Kling 3.0은 테스트/저비용 대안)이라, scene 프롬프트는 Seedance 2.0이 실제로 읽는 방식에 맞춰 쓴다.

9개 프롬프트는 각각 독립 장면 설명이지만, 최종 영상 요청에서는 하나의 영상 맥락으로 함께 전달한다.

### 3.1 Seedance 2.0 특화 원칙

Seedance는 이미지 생성용 키워드("아름다운 조명, 4K")에는 반응하지 않는다. **영화 촬영 지시서**처럼, 무엇이 움직이고 카메라가 어떻게 움직이는지를 구체적으로 써야 한다.

| 순서 | 항목 | 설명 |
| --- | --- | --- |
| 1 | 주체·액션 | 무엇이 일어나는가를 가장 먼저 쓴다 |
| 2 | 카메라 움직임 | dolly, tracking shot, handheld, POV, aerial shot, rack focus 등 실제 촬영 용어를 쓴다 — 모델이 이를 문체가 아니라 촬영 지시로 해석한다 |
| 3 | 사운드 | "천둥소리", "양철 지붕 위 빗소리"처럼 구체적인 음향을 명시한다. 장르에 맞는 배경음(액션=타악기 리듬, 풍경=앰비언트)도 자동 반영됨. 대사·립싱크는 아직 정확도가 낮아 다른 오디오 요소를 우선한다 |
| 4 | 장면 전환 | 멀티샷일 때 각 샷이 어떻게 이어지는지 |

**한 샷당 원칙**: 주요 액션 1개 + 카메라 움직임 1개만 담는다. "캐릭터가 달리고 카메라도 팬 하면서 번개까지 치고 무드도 바뀌는" 식으로 과부하를 주면 모델이 임의로 우선순위를 골라버린다.

**라벨 필수**: "Scene 1:", "Scene 2:" 라벨을 반드시 붙인다 — 라벨이 없으면 여러 장면이 하나의 연속 샷으로 뭉쳐서 생성된다.

**추가 팁**

| 팁 | 내용 |
| --- | --- |
| 정적 구도도 의도적으로 명시 | 카메라를 안 움직일 거면 "static tripod", "fixed-camera"라고 명시한다 — 아무 지시가 없으면 모델이 임의로 움직임을 넣을 수 있다 |
| 모호한 카메라 지시 금지 | "좋은 앵글로" 대신 "close-up on the visor reflection"처럼 무엇을, 어디를 비추는지 구체적으로 쓴다 |
| 감정은 신체 신호로 | "슬퍼 보이게"보다 "눈가가 붉어짐", "손가락이 떨림", "숨을 고름"처럼 구체적 신체 디테일로 감정을 전달한다 |
| 레퍼런스 이미지 활용 | 여러 장을 참조할 때는 "Image2의 스타일을 유지하되 Image1의 사실감은 남긴다"처럼 어느 이미지에서 뭘 가져올지 명시한다(키이미지 참조와 연결) |
| 스타일 일관성 언급 | 특정 감독·영화 장르·화보 스타일을 명시해야 톤이 일관되게 유지된다(막연히 "영화 같은"만 쓰지 않는다) |

출처: [awesome-seedance (ZeroLu)](https://github.com/ZeroLu/awesome-seedance)

### 3.2 scene 프롬프트 형식

```text
Scene 1: [주체+액션], [카메라 움직임], [사운드], [감정/분위기]
Scene 2: ...
...
Scene 9: ...
```

### 3.3 작성 기준

| 항목 | 기준 |
| --- | --- |
| 순서 | 3x3 그리드의 좌상단부터 우하단 순서로 scene 1~9를 작성한다 |
| 길이 | 단일 샷 2~4문장 권장(멀티샷 맥락에서는 1~2문장으로 압축). 모호한 프롬프트보다 상세한 프롬프트가 항상 낫다 |
| 카메라 | pan, dolly, tracking shot, handheld, POV, aerial shot 등 촬영 용어를 한 샷에 하나만 쓴다 |
| 사운드 | 장면마다 구체적 음향 최소 1개는 명시한다(무음 처리할 의도가 아니면 생략하지 않는다) |
| 움직임 | 인물 또는 대상의 핵심 움직임을 명확히 쓴다 — 정적인 풍경 묘사만으로는 안 됨 |
| 일관성 | 인물, 장소, 시간대, 색감이 갑자기 바뀌지 않게 한다. 반복 테스트 시에는 동일 seed 값으로 시각 구성을 대략 맞출 수 있다 |
| 길이 옵션 | Kling 3.0: 3~15초 / Seedance 2.0: 4~15초(모델별 최소값 다름, 옵션값 확정 문서 4절 참고). 멀티샷 구성일 때는 Auto(모델이 각 장면·전환에 필요한 시간을 자동 배분)를 권장 |

### 3.4 추가 9-scene 테스트 예시

**예능 오프닝 (스튜디오 입장)**

```text
Scene 1: Wide static shot of a colorful variety show studio set, spotlights sweeping across empty chairs, upbeat intro music swelling.
Scene 2: Host walks in through the studio entrance, tracking shot following from behind, audience applause and cheering.
Scene 3: Close-up on the host's excited expression, static tripod, crowd laughter in the background.
Scene 4: Wide shot of four guest chairs, camera pans left to right revealing each seat, energetic chatter.
Scene 5: Handheld shot weaving through the audience, camera bobs slightly, rhythmic clapping.
Scene 6: Close-up on a prop microphone being picked up, rack focus from background to mic, soft click sound.
Scene 7: Medium shot of the host addressing camera directly, static tripod, confident tone in ambient mix.
Scene 8: Crane shot rising above the studio revealing full set and lighting rig, sustained crowd noise.
Scene 9: Slow dolly out from the studio center, lights dimming slightly, music transitioning to theme tune.
```

**다큐멘터리 (전통 시장 아침)**

```text
Scene 1: Static tripod shot of a quiet market alley before opening, faint distant rooster crow.
Scene 2: Handheld shot following a merchant setting up a vegetable stall, crates being stacked, wooden clatter.
Scene 3: Close-up on hands arranging produce, rack focus from hands to face, soft ambient morning birdsong.
Scene 4: Wide shot of the alley slowly filling with early customers, natural handheld sway, murmured conversation.
Scene 5: Tracking shot following an elderly vendor carrying goods, footsteps on pavement, distant market chatter.
Scene 6: Static shot of a steaming food stall, rising steam catching morning light, sizzling sound.
Scene 7: Medium shot of a customer bargaining with a vendor, handheld, overlapping voices and light laughter.
Scene 8: Wide aerial shot revealing the full market alley now bustling, layered ambient market noise.
Scene 9: Slow dolly out from the alley entrance, morning sun fully risen, ambient market sound fading into theme.
```

## 4. 영상 프롬프트 예시 (드라마)

```text
Scene 1: A quiet bookstore exterior at dusk, slow push-in toward the entrance, warm lights turning on inside, faint ambient street noise.
Scene 2: The woman enters the bookstore, handheld medium shot, she looks around cautiously, a soft door bell chime.
Scene 3: Close-up of her hand touching an old notebook on a shelf, shallow depth of field, rack focus from notebook to her face, quiet paper rustle.
Scene 4: A man appears at the end of the aisle, slow dolly shot, tense silence, distant footsteps on wood flooring.
Scene 5: The woman freezes and looks up, close-up, soft warm light on her face, her breath audibly catching.
Scene 6: The man steps closer but stops, medium shot, restrained emotion, low ambient hum.
Scene 7: The notebook falls slightly open, insert shot, old photo visible for a moment, a faint paper flutter.
Scene 8: Both characters stand apart in the aisle, wide shot, emotional distance emphasized, silence except soft room tone.
Scene 9: The camera slowly pulls back through the shelves, tracking shot, quiet unresolved mood, ambient bookstore hum fading out.
```

## 5. 역프롬프트 작성 기준

역프롬프트는 선택 기능이다. 레퍼런스 이미지를 분석해 아래 항목을 추출하는 것을 목표로 한다.

| 항목 | 설명 |
| --- | --- |
| style | 전체 시각 스타일 |
| lighting | 조명 방향과 강도 |
| composition | 구도와 프레이밍 |
| lens | 렌즈 느낌과 심도 |
| color_palette | 주요 색감 |
| mood | 감정과 분위기 |
| prompt_en | 재생성에 사용할 영문 프롬프트 |

### 5.1 예시

**입력(레퍼런스 이미지 설명)**: 노을 지는 창가에 앉아 먼 곳을 바라보는 여성, 역광으로 실루엣이 두드러지고 창밖은 흐릿하게 흐림 처리된 사진

**추출 결과**:


## 6. 피해야 할 작성 방식

**공통**
- 한 프롬프트에 서로 충돌하는 분위기를 많이 넣지 않는다.
- 모델명과 옵션만 나열하고 장면 설명을 생략하지 않는다.
- 최종 후보가 아닌 테스트에서 긴 영상 길이를 기본값으로 쓰지 않는다.

**이미지(GPT Image 2)**
- "고품질", "4K" 같은 막연한 품질 키워드만으로 스타일을 정의하지 않는다 — 구체적 레퍼런스(게임/사조/미학)를 명시한다.
- 여러 장에서 같은 캐릭터를 유지해야 하는데 일관성 문구("얼굴형·헤어스타일·의상 유지")를 빼먹지 않는다.

**영상(Seedance 2.0)**
- 영상 프롬프트에서 장면마다 인물 설정을 새로 바꾸지 않는다.
- 9개 scene을 9개의 독립 영상처럼 쓰지 않는다.
- "아름다운 산 풍경, 4K"처럼 움직임 없는 정적 장면 묘사만 쓰지 않는다 — Seedance는 이미지 생성용 키워드에 반응하지 않는다.
- "Scene 1:" 같은 라벨을 생략하지 않는다 — 없으면 여러 장면이 하나의 연속 샷으로 뭉쳐 생성된다.
- 한 샷에 액션·카메라 움직임·분위기 변화를 동시에 욱여넣지 않는다(한 샷당 액션 1개 + 카메라 움직임 1개 원칙).
- 사운드 묘사를 생략하지 않는다 — 없으면 제네릭한 배경음으로 채워진다.
- "좋은 앵글로" 같은 막연한 카메라 지시를 쓰지 않는다 — 무엇을 어디서 비추는지 구체적으로 쓴다.
- 카메라를 고정하고 싶을 때 "static tripod"·"fixed-camera" 같은 명시 없이 방치하지 않는다 — 임의로 움직임이 들어갈 수 있다.
