## 날짜별 업무 현황

### 07.06(월) - 결제 진행 확인, FE 복귀, 2주차 착수

- **결제 진행 상황 공유**: KX 오프라인 PM 회의(이슬기 PM 참석) 결과 기준 Claude Team·Figma·fal.ai 결제 확정, Vercel·AWS는 별도 진행으로 공유 (fal.ai는 07.07 미확정으로 정정)
- **고정 구독 구성 확정**: Claude Team 4석($120) + Figma Professional 2석($40) + Vercel Pro 1석($20) + AWS EC2 예비($100). 저장 구조는 AWS EC2+S3로 확정, Supabase 제외. 총 예상 비용 $359.92 기준
- **팀원 계정 정보 수집 완료**, AWS 계정은 송영범 계정으로 통일
- **FE 김은재 복귀** — 레포 클론, 로컬 실행, 라우팅·공통 컴포넌트 구조 확인 착수

### 07.07(화) - 결제 정정, 비용 통제 재설계, 키이미지 정의 확정

- **결제 현황 정정**: fal.ai는 결제 미확정으로 확인. Claude Team(좌석 초대 완료)·Figma만 확정, Vercel·fal.ai·AWS 미확정
- **AWS 운영 방식 확정**: 결제 확정과 별개로 개인 계정으로 EC2/S3 우선 세팅 → 회사 결제 확정 시 AWS Organizations 멤버 계정 편입
- **Vercel Team 생성 완료**: FE가 Pro 14일 무료 트라이얼 + 개인 카드 임시 등록 경로로 Team 생성. 남은 조치는 결제 담당자 초대·카드 교체(트라이얼 14일 내)
- **fal.ai Hard Limit 부재 확인**: 공식 문서 확인 결과 자동 차단형 지출 상한 기능이 없음. 비용 통제를 지출 알림 설정 + $50~100 소액 분할 충전으로 전환하고 관련 문서 전체 정정
- **영상 상태값 정리**: `PENDING` 삭제 — DB `RUNNING/SUCCESS/FAILED`, 화면 표시 `processing/completed/failed` 이중 구조로 확정
- **키이미지 정의 PD 확인**: 키이미지는 캐릭터 1명의 고정 레퍼런스 시트(턴어라운드+표정+소품+키컬러)로 확정. 캐릭터 시트와 동일 산출물, 배경은 다각도 시트 불필요. PD 표준형 2단(Prose+구조화 필드) 프롬프트 템플릿 확보
- **프롬프트 변환 흐름 합의**: 이미지 생성에 한→영 변환 토글 UI 적용, 영상 생성에는 미적용
- **UX/UI 와이어프레임 완성** — 1주차부터 "확인 필요"로 남아있던 산출물 항목 해소

### 07.08(수) - 이미지 생성 메뉴 2단계 분리, 옵션값 2안 작성

- **메뉴 A·B 분리 결정**: 한 번의 프롬프트로 완성형 키이미지 세트 출력이 어렵다는 BE·디자이너 판단에 따라, 메뉴 A(초기 레퍼런스: 배경 1장+캐릭터 다각도) → 메뉴 B(키이미지/디테일 생성: 완성형 캐릭터 시트) 2단계로 분리
- **메뉴 B 속성 선택 사이드바**: 파트 1~5(캐릭터 기본/아트 스타일·세계관/헤어/얼굴/의상) 선택형 구조로 확정 (이슬기 PM 정리)
- **프롬프트 가이드 초안 v1.1 갱신**: PD 키이미지 정의 기준으로 캐릭터 시트=키이미지 절 병합, 표준형 템플릿 추가
- **화면별 옵션값 손찬용 안 작성·통합 대조**: 기존 서비스 사례와 GPT Image 2·Seedance 2.0 실제 API 스펙 조사 기반. 이슬기 PM 안과 대부분 일치, 영상 해상도 "BE 확인 필요" 해소. 미해결 2건(Seedance 4:3·3:4 비율, 메뉴 B 파트 구조 검토 의견)은 후속
- **B1팀 로그인/유저 DB 통합 문의**: 현 단계 고려 사항 아님으로 판단, 보류(트래킹 유지)
- **BE 내부 테스트 협의**: fal.ai 연동 내부 테스트를 이번 주 내 가능하도록 준비. 비용 관리상 테스트는 짧은 길이(3초 등) 위주로 진행 요청

### 07.09(목) - 개발 3파트 병행 진행, 진행률 표시 이슈 확인

- **BE(영상)**: 영상 생성 API 연결 후 테스트 2회. 상태 조회 API가 상태값만 제공하고 로그 데이터를 받아오지 못해 퍼센트 진행률 표시가 어려운 이슈 확인 — 모델별 로그 제공 여부 확인 중
- **BE(이미지)**: 업로드 기능 개발 중, 조회·라이브러리 저장은 후속
- **FE**: 이미지 생성 사이드바 공통 레이아웃·홈 화면 작업, 라이브러리 착수. 메뉴 A·B 분리로 캐릭터시트 생성 화면이 추가되는 구조로 확인
- **UX/UI**: 피그마 구글 M3 활용 홈 화면 디자인 진행, 중간보고 전 디자인 완료 목표
- **결정**: 영상 해상도는 별도 입력 없이 Kling 중 낮은 해상도로 고정 / 옵션값은 디자이너 전달 전 BE·PM 통합회의 선행

### 07.10(금) - 영상 입력 방식 변수 발견, 히스토리 저장 정책 확정

- **BE(영상)**: Kling 기준 시작·끝프레임/3x3 이미지 입력 테스트 결과 양호. **Seedance에 최대 9장 다중 이미지 입력 모델이 별도로 존재함을 발견** — PD 제작 영상이 이 방식일 가능성이 있어, 확인 결과에 따라 영상 프롬프트 가이드(9-Shot 텍스트 구조)의 전제가 바뀔 수 있음. PD 원본 영상을 받아 개발단에 문의하기로 함
- **BE(이미지)**: 업로드 완성, 다운로드 거의 완성. 다음 작업은 라이브러리 저장·조회·캐릭터시트
- **FE**: 라이브러리·이미지 생성 화면 디테일 작업, 이미지 생성 로딩 적용 진행
- **"프롬프트 향상" 토글 동작·히스토리 저장 정책 확정**: 히스토리에는 실제 생성에 쓰인 최종 프롬프트를 저장 — 토글 ON이면 변환된 영어, OFF면 입력 원문 그대로. 토글 ON + 빈 프롬프트 처리 방식은 미결
- **영상 생성 비용 리스크 대응**: 시행착오를 줄이기 위해 프롬프트 가이드를 명확히 확정한 뒤 BE에 전달하기로 함

## 주요 결정 사항 (2주차 누적)

- 고정 구독 구성 확정 (Claude Team 4석·Figma 2석·Vercel Pro 1석, 저장 구조 AWS EC2+S3, Supabase 제외)
- AWS는 개인 계정(송영범) 우선 세팅 → 회사 결제 확정 시 Organizations 편입
- fal.ai 비용 통제는 지출 알림 + $50~100 소액 분할 충전 방식으로 전환 (자동 차단형 Hard Limit 미존재 확인)
- 영상 작업 상태값 `PENDING` 삭제 — DB/화면 이중 구조 확정
- 키이미지 = 캐릭터 1명의 고정 레퍼런스 시트 (PD 확정), 배경 시트는 다각도 불필요
- 이미지 생성 화면을 메뉴 A(초기 레퍼런스)·메뉴 B(키이미지) 2단계로 분리, 메뉴 B는 속성 선택 사이드바(파트 1~5) 방식
- 프롬프트 변환(한→영)은 이미지 생성에 토글 UI로 적용, 영상 생성에는 미적용
- 영상 해상도는 Kling 중 낮은 해상도로 고정
- 히스토리 저장 프롬프트는 실제 생성에 쓰인 최종 프롬프트 기준
