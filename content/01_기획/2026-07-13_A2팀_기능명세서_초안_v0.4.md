## 1. 문서 목적

최종 IA의 화면 구조를 개발·QA가 구현 가능한 기능 단위로 재구성한다. IA 한 행을 그대로 기능 한 개로 옮기지 않고, 사용자가 수행하는 행동과 시스템 처리 규칙을 기준으로 관련 IA ID를 묶는다.

1. 최종 IA v2.1
2. 2026-07-13 핸드오프
3. PRD v1.3
4. 화면별 옵션값 v0.7

## 6. 공통 처리 정책

### 6.1 인증·접근

- 생성·라이브러리·프로필·역프롬프트 화면은 로그인 후 접근한다.
- 비로그인 상태에서 보호 화면에 접근하면 로그인 화면으로 이동시키는 방향으로 FE·BE 동작을 확인한다.

### 6.2 자동 저장·찜하기

- 이미지·영상·역프롬프트 재생성 결과는 성공 즉시 자동 저장한다.
- 찜하기는 저장 여부를 결정하는 버튼이 아니라 이미 저장된 작업물의 즐겨찾기 상태다.
- 재생성 결과는 기존 작업물을 덮어쓰지 않고 새 작업물로 저장한다.
- 자동 저장 실패 시 결과 유지·재시도·중복 방지 기준은 BE 확인 전이다.

### 6.3 프롬프트 향상

- 이미지 메뉴 A·B에만 제공한다.
- OFF는 사용자 원문을, ON은 Claude Sonnet이 변환한 영어 최적화 프롬프트를 실제 생성에 사용한다.
- 변환 결과를 생성 전에 보여주고 수정하게 할지는 아직 확정하지 않는다.
- 영상 화면에는 프롬프트 향상 기능을 제공하지 않는다.

### 6.4 생성 상태·실패

- 이미지 생성 중은 스피너를 사용한다.
- 영상은 `processing`, `completed`, `failed`를 사용자 상태로 사용한다.
- 영상의 가짜 퍼센트 진행률은 사용하지 않는다. 스피너·경과시간 중 최종 UI를 결정한다.
- 입력 오류와 API 실패 후에도 사용자의 기존 입력을 유지한다.

### 6.5 한도 (미적용 확정)

- 일일 생성 한도와 크레딧 개념은 적용하지 않는다. (2026-07-15 확정)
- 한도 카운터, 한도 초과 안내·차단 로직은 구현 범위에서 제외한다.
- A002-06은 영상 입력 검증만 담당하며, 한도 관련 IA 항목(IA-084)은 삭제가 필요하다.

## 7. API·데이터 기준

| 기능 | 현재 기준 | 확인 필요 |
| --- | --- | --- |
| 이미지 생성 | `POST /api/generate-image` | 상세 request/response, 오류 형식 |
| 영상 생성 | `POST /api/generate-video` | 모델별 입력 매핑, 오디오·비율 처리 |
| 영상 상태 | `GET /api/video-status` | 조회 키, polling 주기, 실패 응답 |
| 역프롬프트 | `POST /api/reverse-prompt` | 저장 범위, 반환 스키마 |
| 파일 업로드 | `POST /api/file` | multipart 필드명, 허용 파일 기준 |
| 파일 목록·상세 | `GET /api/file`, `GET /api/file/{id}` | 필터·정렬 파라미터, 권한 |
| 파일 다운로드 | `GET /api/file/{id}/download` | stream·redirect·presigned URL 방식 |
| 파일 삭제 | `DELETE /api/file/{id}` | DB·S3 삭제 범위, 복구 정책 |
| 프로필·인증 | 기존 명세에 없음. 이메일-가입 방식 배타는 BE 구현 완료 | 엔드포인트·세션·User 가입 방식 필드 |

현재 DB 기준은 `MediaFile`과 `GenerateJob`이다. 즐겨찾기, 프로필, 사용자 소유권, 공유 링크에 필요한 스키마는 기존 API·DB 명세에 없으므로 BE가 확정하기 전 임의 컬럼을 사용하지 않는다.
