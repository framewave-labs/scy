## 1. 문서 목적

통합 기능명세서의 47개 기능 ID와 IA 추적성을 유지하면서 백엔드가 요청 처리, API, 저장, 예외, 인프라 선행 조건을 빠르게 읽을 수 있도록 재구성한다.

동일한 기능이 FE와 BE 양쪽 문서에 나타날 수 있다. 이는 중복 기능이 아니라 화면과 API의 접점이며, 기능 ID를 공통 키로 사용한다.

## 2. Excel 읽는 순서

1. `읽기안내`: 우선순위와 이번 회의 결정 확인
2. `BE 기능`: 47개 기능을 담당 관점으로 검토

## 5. 공통 구현 기준

- 영상 생성 입력은 3×3 그리드 이미지 1장과 프롬프트 1개다.
- Seedance와 Kling 모두 오디오 생성을 기본 ON으로 적용한다.
- 최종 IA의 모델별 비율 파라미터를 사용한다.
- AI 프롬프트 변환 ON 시 생성에 사용하는 영어 프롬프트는 사용자에게 노출하지 않는다.
- 이미지·영상 생성용 테스트 이미지와 프롬프트는 PM단에서 제작해 전달한다.
- 현재 파일 저장은 로컬 방식이며 AWS·S3 세팅 후 저장 코드가 변경될 예정이다.
- 삭제 기능, 자동 저장 실패 처리, 운영 환경의 영상 상태 처리는 AWS·S3 세팅 이후 진행한다.
- 프로필의 구독 메뉴·구독 상태·플랜은 구현 범위에 포함하지 않는다.
- 일일 생성 한도·크레딧은 적용하지 않는다(07-15 확정). 한도 초과 차단 로직은 구현하지 않으며 A002-06은 입력 검증만 담당한다.

## 6. 완료 판단 원칙

- API 연동 완료, 저장 완료, 운영 환경 완료를 구분한다.
- 로컬 저장 완료를 AWS·S3 운영 저장 완료로 간주하지 않는다.
- API request/response와 DB 필드는 담당자가 확정한 뒤 본 문서에 반영한다.
