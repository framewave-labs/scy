# A2팀 FE 기능명세서 v0.4

> 기준: 통합 기능명세서 v0.4 · 최종 IA v2.1 · 2026-07-14 개발팀·PM 회의

> 상세 명세: [Excel 다운로드](files/2026-07-14_A2팀_FE_기능명세서_v0.4.xlsx)

## 1. 문서 목적

통합 기능명세서의 47개 기능 ID와 IA 추적성을 유지하면서 프론트엔드가 화면 구현과 API 연동에 필요한 정보만 빠르게 읽을 수 있도록 재구성한다.

동일한 기능이 FE와 BE 양쪽 문서에 나타날 수 있다. 이는 중복 기능이 아니라 화면과 API의 접점이며, 기능 ID를 공통 키로 사용한다.

## 2. Excel 읽는 순서

1. `읽기안내`: 우선순위와 이번 회의 결정 확인
2. `FE 기능`: 47개 기능을 담당 관점으로 검토
3. `FE API 연동`: API와 데이터 책임 확인
4. `FE 확인사항`: 확정·보류·선행 조건 확인

## 5. 공통 구현 기준

- 영상 생성 입력은 3×3 그리드 이미지 1장과 프롬프트 1개다.
- Seedance와 Kling 모두 오디오 생성을 기본 ON으로 적용한다.
- 최종 IA의 모델별 비율 파라미터를 사용한다.
- 프롬프트 향상 ON 시 생성에 사용하는 영어 프롬프트는 사용자에게 노출하지 않는다.
- 이미지·영상 생성용 테스트 이미지와 프롬프트는 PM단에서 제작해 전달한다.
- 현재 파일 저장은 로컬 방식이며 AWS·S3 세팅 후 저장 코드가 변경될 예정이다.
- 삭제 기능, 자동 저장 실패 처리, 운영 환경의 영상 상태 처리는 AWS·S3 세팅 이후 진행한다.
- 프로필의 구독 메뉴·구독 상태·플랜은 구현 범위에 포함하지 않는다.
- 일일 생성 한도·크레딧은 적용하지 않는다(07-15 확정). 한도 카운터·초과 안내 UI는 구현하지 않으며 A002-06은 입력 검증만 담당한다.
- 한 이메일은 일반 가입 또는 Google 가입 중 한 방식만 갖는다(07-15 BE 구현 확인). 교차 가입은 차단되며, PROF-002는 계정 연결·해제 없이 가입 방식 표시만 한다.

## 6. 완료 판단 원칙

- UI 화면 완료, mock 동작 완료, 실API 연동 완료를 구분한다.
- Loading·Empty·Error 상태와 입력 유지 여부를 함께 확인한다.
- API 응답 필드가 정해지지 않은 경우 FE에서 임의 필드를 확정하지 않는다.
