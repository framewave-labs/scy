# A2팀 2026-07-15 스크럼 전달사항

> 전달자: 손찬용 PM
> 관련 문서: 기능명세서 v0.4 (07-14 회의 협의 결과 반영본)

## 1. 공유 내용 — 기능명세서 노션 DB

- 기능명세서 v0.4를 노션 데이터베이스로 정리해 **FE 기능명세서 / BE 기능명세서** 페이지 2개로 만들었습니다. (위치: 팀 노션 대시보드 내 페이지 / 링크: 추가 예정)
- 각 47개 기능이며, 어제(07-14) 회의에서 협의한 확인필요 12개 항목 결과까지 반영된 버전입니다.
- 어제는 개인 작업으로만 진행한 상태라 공유가 오늘로 늦어졌습니다. 오늘부터 이 DB를 기능명세 확인·수정의 기준으로 사용해 주시면 됩니다.

## 2. 개발팀 확인 요청

각자 담당 페이지(FE/BE)에서 아래 순서로 확인 부탁드립니다.

1. 담당 행을 실제 구현과 대조한 뒤 **확인 상태**를 `미확인` → `확인`으로 변경
2. **구현 상태**가 비어 있는 행은 현재 상태로 채우기
   - FE: `미착수 / UI 완료 / Mock 연결 / 실API 연동 / 완료 / 착수 보류(AWS 이후)`
   - BE: `미착수 / 진행 중 / 로컬 기준 완료 / 구현 완료 / AWS 이후 / 해당 없음`
3. **확인 질문**이 적힌 행은 답을 **수정 내용**에 작성
4. 명세와 실제가 다른 부분(명칭·동작·범위)은 **수정 내용**에 현재 코드 기준으로 작성

기능 ID와 기획 기능명은 수정하지 말아 주시고, 다른 점은 전부 수정 내용 열에 적어 주시면 PM이 취합해 **PM 반영**으로 관리합니다.

## 3. 작업 기준 리마인드 (07-14 확정)

- **삭제·계정 탈퇴·라이브러리 필터/정렬·자동 저장 실패 처리는 AWS·S3 세팅 이후 착수** — 현재 구현 대상이 아닙니다.
- BE는 로컬 저장 기준으로 끝난 항목을 `구현 완료`가 아닌 `로컬 기준 완료`로 구분해 주세요.
- 문서의 `연동 API` 열은 기획 기준 명칭이며 실제 코드 명칭을 강제하지 않습니다 — 다르면 수정 내용에 적어 주세요.

## 4. PM 후속 예정

- 이미지·영상 생성용 테스트 이미지와 프롬프트(3×3 그리드 1장 포함)는 PM단에서 제작해 전달 예정입니다.
- 개발팀 확인이 끝나는 대로 수정 내용을 반영해 기능명세서 다음 버전으로 갱신하겠습니다.
