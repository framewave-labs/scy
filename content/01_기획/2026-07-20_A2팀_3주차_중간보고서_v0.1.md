## 3주차 목표 vs 실적

마일스톤 v3.3의 3주차 완료 기준(M001~M006 + A006 안정화 → A002~A005 착수 → 3주차 중반 FE+BE 1차 배포) 대비 실적입니다.

| 완료 기준 | 상태 | 비고 |
|---------|------|------|
| M001~M006 + A006 전체 플로우 안정화 | 진행 중 | FE/BE API 연결이 07.20 기준 거의 마무리 단계. 라이브러리 저장은 로컬 저장 방식으로 개발 중이며 S3 전환 대기 |
| A002~A005 영상 생성 착수 | 진행 중 | 영상 입력 방식(3×3 그리드 1장+프롬프트 1개) 확정, 테스트 프롬프트 전달 완료. `generate-video`·`video-status` 구현 진도는 확인 필요 |
| 3주차 중반(07.16) FE+BE 1차 배포 | 미완 — 이월 | BE EC2 세팅(07.15)까지 완료, S3 미세팅. 배포는 4주차로 이월 (07.20 현재 BE가 S3 작업 중) |
| 외부 사용성 테스트(비개발자 1인) | 확인 필요 | 관찰 기록 산출물이 문서로 확인되지 않음 |
| 07.16(목) 3주차 점검 + 4주차 플랜 확정 | 확인 필요 | 별도 플랜 문서로 확인되지 않음. 07.16 스크럼은 진행됨 |

07.17(금) 제헌절 휴무로 실근무 4일이었고, 1차 배포 미완이 이번 주 최대 이월 항목입니다. 배포 후 핫픽스·이슈 정리로 계획됐던 07.20(월)은 실제로는 S3 작업·로그인 연결 진행일이 됐습니다.

## 날짜별 업무 현황

### 07.13(월) - 기획 산출물 일괄 정리, 영상 테스트 시나리오 확정

- **기획 산출물 일괄 정리**: 최종 IA v2.1, 기능명세서 초안(기능 46개, AUTH~SHARE ID 체계), 유즈케이스 v0.1, 사용자스토리 v0.1, 구현매핑 v0.1 작성
- **옵션값 확정 문서 v0.8 갱신**: 영상 화면 프롬프트 향상 토글 제거 확정, Seedance·Kling 둘 다 스토리보드 생성 용도로 확정, 스토리보드 그리드는 예시 수령분·자체 생성분 사용
- **영상 진행률 표시 확정**: 퍼센트 추적 불가 — 스피너·시간 표시 방식으로 전환
- **영상 생성 방식 비교 테스트 시나리오 확정**: 같은 스토리보드로 15초·720p 3건(Seedance 그리드 1장 / 9장 슬라이스 / Kling 시작·끝 프레임+그리드 레퍼런스) 비교, 총 예상 비용 약 $10.5
- **결제 현황**: Vercel 결제 카드 교체 완료. fal.ai·AWS 미확정 지속

### 07.14(화) - 기능명세 확인필요 12건 협의, 기능명세서 v0.4

- **기능명세 확인필요 12개 항목 협의** (PM 2인 + 개발팀): 영상 생성 입력은 **3×3 그리드 이미지 1장 + 프롬프트 1개**로 확정 / 진행 표시는 스피너로 전달 완료 / 비율 파라미터는 최종 IA 기준 / **Seedance·Kling 모두 오디오 생성 기본 `ON`** / 프롬프트 향상 결과(영어 프롬프트)는 사용자에게 미노출
- **AWS·S3 선행 조건 정리**: 현재 로컬 저장 방식으로 개발 중 — 삭제·계정 탈퇴, 자동 저장 실패 처리, 라이브러리 필터·정렬, 웹훅/폴링 확정은 AWS·S3 세팅 이후로 순서 확정
- **PM단 입력 자료 제작 확정**: 이미지·영상 생성용 테스트 이미지와 프롬프트는 PM단에서 자체 제작해 개발팀에 전달
- **기능명세서 FE/BE v0.4 작성·공유**: 구글 로그인 AUTH-003 분리, PROF-002 계정 연동 명확화. 노션 DB용 CSV·복붙용 md 병행 제작
- **옵션값 v0.9→v0.10**: 오디오 기본 ON 정책, 프로필 카테고리·스피너 진행 표시 항목 종결

### 07.15(수) - 테스트 프롬프트 문서 전달, EC2 세팅

- **PM**: 테스트용 프롬프트 문서 작성·전달 — 이미지 생성(GPT Image 2), 영상 생성(Seedance/Kling), 3×3 스토리보드 그리드. 일일 한도·크레딧 미적용 확정 반영
- **BE**: 소셜로그인 작업, API 명세서 최신화, **EC2 세팅 완료**. 캐릭터 시트 개발 진행
- **FE**: 상세보기 작업(공통 컴포넌트로 묶어 진행), GitHub 누락 이슈 확인·정리 착수
- **디자인**: 화면 디자인 진행, 07.16(목) 마무리 목표 공유 — PM 판단: FE가 BE·API 연결 전까지 나오면 되는 일정으로 보고, 세부 일정은 디자이너·FE 직접 협의로 안내

### 07.16(목) - 프리셋 제작 착수, 탐색하기 탭 제안

- **PM**: 프리셋 제작 착수 — 이미지는 GPT Image 2 직접 생성(캐릭터 시트·단일 캐릭터 등), 영상은 Civitai(상업적 사용 가능 조건)에서 수집하는 방향으로 결정
- **디자인**: 와이어프레임 70~80% — 프리셋 노출 영역으로 **탐색하기 탭**(영상/사진 구분, 태그 카테고리, 보라 계열 고급 무드) 제안. 탐색하기 탭은 현재 IA·기능명세서에 없는 신규 화면 개념으로, 와이어프레임 확정 시 IA 행 추가 필요
- **확인 필요로 기록**: 토글 명칭("AI 프롬프트 변환" 전달 예정 vs 07.07 확정 명칭 "프롬프트 향상") — 재확정인지 착오인지 확인 후 문서 3종 갱신 여부 결정
- **옵션값 v0.11**: 프롬프트 향상 결과 미노출(07.14 확정) 반영. 프롬프트 가이드에 Kling 3.0 샷 프롬프트 차이 절(3.1.1) 추가

### 07.17(금) - 제헌절 유급휴가 (전원 휴무)

### 참고: 제출일(07.20)까지의 주말·당일 진행분

- **프리셋 최종본 완성 (07.18)**: GPT 이미지 72개(6카테고리×12) + Civitai 보라·네온 영상 20개(10초·오디오 포함) + 정리 PDF 2종. 검증 완료 후 `scy` 저장소에 업로드 — 단, Civitai 영상 실물(MP4)은 공개 샘플 재배포 권리가 미확정이라 저장소에서 제외하고 팀 드라이브 공유로만 유지, 저장소에는 조사본 PDF·메타데이터(원본 링크·재현 프롬프트)만 게시
- **07.20(월) 스크럼**: FE·BE API 연결 거의 마무리 / BE S3 작업 착수 / FE 로그인 연결 시도 / PM 유즈케이스 문서 통합 작업 착수 예정 / 와이어프레임은 07.21(화)까지로 조정

## 주요 결정 사항 (3주차 누적)

- 영상 생성 입력은 3×3 그리드 이미지 1장 + 프롬프트 1개로 확정
- 영상 진행 표시는 퍼센트 대신 스피너 방식 (개발팀 전달 완료)
- Seedance·Kling 모두 오디오 생성 기본값 `ON`
- 프롬프트 향상 결과(변환된 영어 프롬프트)는 사용자에게 미노출
- 삭제·계정 탈퇴, 자동 저장 실패 처리, 라이브러리 필터·정렬, 웹훅/폴링 확정은 AWS·S3 세팅 이후 착수
- 이미지·영상 생성용 테스트 입력 자료는 PM단에서 제작해 개발팀에 전달
- 일일 한도·크레딧은 미적용 확정
- 프리셋 노출 영역은 탐색하기 탭(디자이너 제안) — IA 반영은 와이어프레임 확정 후
- 이미지 프리셋은 GPT 직접 생성, 영상 프리셋은 Civitai 수집(상업적 사용 조건 개별 확인)
