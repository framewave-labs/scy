# A2팀 유즈케이스 통합본 v1.0

> 작성일: 2026-07-20
> 작성자: 손찬용 PM · 이슬기 PM (scy 유즈케이스 v0.1 + lsg 유즈케이스 v0.5 병합)
> 목적: 두 PM이 병렬 작성한 유즈케이스를 하나로 합치고, 07-14~15 회의 확정 사항을 반영해 기능명세·개발의 단일 기준으로 삼는다.

## 0. 통합 기준

- **ID 체계**: `UC-{도메인}-{번호}` (lsg 방식). 도메인은 기능명세서 ID 체계(AUTH~SHARE)와 정렬한다.
- **범위**: 핵심 흐름(이미지·영상·라이브러리·역프롬프트)은 상세 기술, 정형 CRUD(인증·홈·프로필)는 간결 기술 후 기능명세서 인수 조건을 참조한다.
- **액터**: 핵심 UC는 lsg의 페르소나 서술(방송 PD/AD)을 채택하고, 시스템 액터(생성 API·저장 시스템)는 scy 정의를 유지한다.
- **우선순위**: P0/P1/P2(lsg)와 범위상태(확정/부분확정/후속안건/S등급, scy)를 병기한다.
- **기준 문서**: 통합 시점의 팀 공용 최신본 — 최종 IA v2.1(scy) + lsg IA v0.6 + 기능명세서 FE/BE v0.4(07-14, 노션 DB 공유본) + 화면별 옵션값 확정 v0.11 + 07-14 기능명세 확인필요 협의 회의록. IA 통합본이 확정되면 이 절을 갱신한다.
- 두 원본 문서(scy v0.1, lsg v0.5)는 이 통합본으로 대체하며, 이후 갱신은 이 문서에서만 진행한다.

### 통합 시 반영한 확정 사항 (원본 작성 이후 결정분)

| 항목 | 원본 상태 | 통합본 반영 |
| --- | --- | --- |
| 영상 입력 방식 | scy·lsg 모두 "논의 중/후속 확정" | **3×3 그리드 이미지 1장 + 프롬프트 1개로 확정** (07-14). 모델별 내부 전처리는 BE 구현 사항 |
| 오디오 | scy "후속 확정 필요" | **Seedance·Kling 모두 기본 `ON` 확정** (07-14) |
| 영상 진행 표시 | scy "가짜 퍼센트 금지" vs lsg "가짜 프로그레스 바 0→80%" | **스피너 방식으로 확정, 개발팀 전달 완료** (07-14). 가짜 퍼센트는 사용하지 않는다 |
| 프롬프트 향상 결과 | — | 변환된 영어 프롬프트는 **사용자에게 미노출** (07-14) |
| 일일 한도·크레딧 | scy만 반영 | **미적용 확정** (07-15) — 한도 차단 흐름 없음 |
| 삭제 기능 | — | AWS·S3 세팅 후 착수 (07-14). 유즈케이스는 유지하되 구현 시점 별도 |

## 3. 정형 흐름 (간결 기술)

### UC-AUTH-01 로그인 / UC-AUTH-02 회원가입

- 로그인: 이메일·비밀번호 제출 → 검증 → 인증 → 홈 이동. 실패 시 입력 유지 + 오류 안내.
- 회원가입: 이메일 확인 → 비밀번호 설정 → 약관·개인정보 필수 동의 → 계정 생성. 중복 이메일이면 가입 중단·안내.
- 구글 소셜 로그인은 기능명세서 AUTH-003으로 분리되어 개발 진행 중 — 흐름 상세는 기능명세서 인수 조건을 따른다.

### UC-HOME-01 홈에서 기능 탐색·진입

서비스 소개·주요 생성 CTA·최근 작업물 표시 → 사용자가 CTA 또는 고정 사이드바로 이미지 A/이미지 B/영상/라이브러리 이동. 최근 작업물 선택 시 공통 상세 모달 표시.

### UC-PROF-01 프로필·보안·활동 관리

프로필 이미지·닉네임·이메일(읽기 전용) 표시, 닉네임 수정·저장, Google 연결 상태 확인·해제, 최근 생성일·라이브러리 바로가기. 구독 관련 화면은 최종 IA에서 제외.

### UC-PROF-02 계정 삭제

계정 삭제 선택 → 영구 삭제·복구 불가 확인 모달 → 동의·재인증 → 삭제 처리·로그아웃 전환.
**확인 필요**: 소프트/하드 삭제, S3·라이브러리·계정 데이터 삭제 범위, 재인증 기준 — AWS 저장 구조 확정 후 BE 기준으로 정리 (07-14 협의).

### UC-SHARE-01 작업물 공유 (후속안건)

작업물 상세를 링크로 공유. 공개 범위·링크 만료·삭제 권한·비로그인 접근·원본 다운로드 허용 여부가 미확정이며, BE가 구현 가능한 기본안 기준으로 우선 판단하되 정책 확정 전 구현 범위로 보지 않는다.

## 4. 핵심 흐름 (상세 기술)

### UC-IMAGE-01 메뉴 A — 초기 레퍼런스 생성

| 항목 | 내용 |
| --- | --- |
| 주 액터 | 방송 PD/AD — 결과물을 하나하나 다듬기보다 촬영 전 짧은 시간 안에 캐릭터·배경의 대략적 방향을 빠르게 잡고 싶어 함 |
| 목표 | 간단한 프롬프트로 캐릭터·배경의 초기 형태를 빠르게 확인한다 |
| 사전조건 | 로그인 상태 / 이미지 생성 메뉴 A 진입 |
| 사후조건 | 배경 1장 또는 캐릭터 다각도 이미지가 표시되고 라이브러리에 자동 저장됨 |

기본 흐름:

1. 프롬프트 입력 (한국어/영어 자유), 필요 시 레퍼런스 이미지 첨부
2. 프롬프트 향상 ON/OFF, 비율(1:1 기본), 품질(medium 기본) 선택
3. [생성하기] — ON이면 Claude Sonnet으로 영어 최적화 변환 후 생성 요청
4. 생성 중 스피너 + "이미지를 생성하고 있습니다…" 안내 (예상 10~15초)
5. 결과 표시와 동시에 결과·실제 생성 프롬프트를 라이브러리에 자동 저장
6. 찜하기(UC-LIB-03) 또는 메뉴 B로 연계

대체·예외 흐름:

- 빈 프롬프트면 [생성하기] 비활성화. 미지원 파일 형식·용량이면 입력 유지 + 오류 표시
- 생성 실패 시 에러 메시지 + [다시 시도] (동일 입력 재요청)
- 생성 60초 초과 시 타임아웃 처리 후 [다시 시도] (제안값 — BE 실측 후 조정)

### UC-IMAGE-02 메뉴 B — 키이미지(캐릭터 시트) 생성

| 항목 | 내용 |
| --- | --- |
| 주 액터 | 방송 PD/AD — 표정·소품·컬러까지 정리된 레퍼런스로 연출·미술과 소통하려는 실무자 |
| 목표 | 완성도 높은 캐릭터 시트(턴어라운드+표정+소품+컬러 팔레트) 1장을 만든다 |
| 사전조건 | 로그인 상태 / 메뉴 B 진입 (메뉴 A를 거치지 않은 직접 진입도 가능) |
| 사후조건 | 키이미지가 표시되고 라이브러리에 자동 저장됨 |

기본 흐름:

1. 입력 방식 선택 — **자유 텍스트**(PD 표준형 2단 구조) 또는 **속성 선택 사이드바**(파트1 캐릭터 기본 ~ 파트5 의상·악세서리)
2. (자유 텍스트) 프롬프트 작성, 프롬프트 향상 사용 가능 / (속성 선택) 값 선택 — 악세서리만 복수 선택, 나머지는 단일 선택, 컬러는 팔레트+헥스코드 지정
3. 시스템이 고정값(1장, 흰 배경, 정면·측면·후면)과 사용자 입력을 조합해 생성 요청
4. 생성 중 스피너 안내 (예상 15~20초 — 제안, 실측 후 조정)
5. 결과 표시와 동시에 라이브러리 자동 저장
6. 이후 영상 생성(UC-VIDEO-01)의 참고 레퍼런스로 활용

대체·예외 흐름:

- 빈 프롬프트·필수 속성 미입력 시 [생성하기] 비활성화
- 인간형 외 캐릭터는 이번 범위 미지원
- 생성 실패·타임아웃 처리는 UC-IMAGE-01과 동일

### UC-PROMPT-01 프롬프트 향상 토글 사용

| 항목 | 내용 |
| --- | --- |
| 주 액터 | 방송 PD/AD — 한국어로는 구체적으로 설명할 수 있지만 영어 프롬프트 작성에 서투름 |
| 목표 | 프롬프트 엔지니어링 지식 없이 한국어 입력만으로 생성 품질을 확보한다 |
| 사전조건 | 메뉴 A 또는 메뉴 B(자유 텍스트 경로)에서 프롬프트 입력됨 |
| 사후조건 | 저장된 결과의 프롬프트 메타데이터가 실제 생성에 쓰인 최종 프롬프트(ON이면 변환된 영문) 기준으로 기록됨 |

기본 흐름:

1. 한국어로 장면·캐릭터 입력
2. 프롬프트 향상 토글 ON
3. [생성하기] — 백그라운드에서 Claude Sonnet으로 영어 변환. **변환 결과는 화면에 노출하지 않는다** (07-14 확정)
4. 변환된 프롬프트로 바로 생성 진행

대체·예외 흐름:

- OFF면 입력 원문 그대로 생성 요청 (변환 단계 없음)
- 메뉴 B 속성 선택 경로에는 적용되지 않음 (구조화된 값이라 변환 대상 아님)
- 변환 실패 시 원본 입력으로 생성 시도 (제안 — BE 협의 필요)
- 영상 생성 화면에는 이 토글이 없다 (07-13 확정)

**확인 필요**: 토글 표시 명칭 — 07-07 "프롬프트 향상"으로 확정·문서 3종 반영됐으나, 07-16 디자이너가 "AI 프롬프트 변환"으로 전달 예정 언급. 재확정인지 착오인지 확인 후 명칭 단일화.

### UC-VIDEO-01 스토리보드 영상 생성

| 항목 | 내용 |
| --- | --- |
| 주 액터 | 방송 PD — 촬영 전 장면의 톤·구도·움직임을 미리 확인하려 함. 생성 비용에 민감 |
| 목표 | 3×3 그리드 이미지와 프롬프트로 오디오 포함 영상 클립을 생성한다 |
| 사전조건 | 로그인 상태 / 번호 오버레이 없는 3×3 그리드 이미지 1장과 프롬프트 보유 (B서비스 수령분 또는 자체 생성분) |
| 사후조건 | 생성 영상(오디오 포함)이 표시되고 라이브러리에 자동 저장됨 |

기본 흐름:

1. 3×3 그리드 이미지 **1장**을 업로드한다 (**그리드 1장 + 프롬프트 1개 입력 구조로 확정** — 07-14. 모델별 내부 전처리는 BE 구현 사항)
2. 영상 프롬프트 입력 (Shot별 장면 묘사 — 프롬프트 가이드 기준). 프롬프트 향상 기능 없음
3. 모델 선택 — 기본값 **Kling 3.0**(저비용·반복 테스트), 최종 데모·고품질 확인 시 **Seedance 2.0**
4. 지원되는 길이·비율·해상도 선택 — 모델별 지원값을 시스템이 검증
