# A2팀 단계형 테스트 프롬프트 운영 세트 v0.1

> 목적: 장르별 예시 개수를 늘리는 대신, 하나의 캐릭터와 이야기로 이미지 생성부터 영상·라이브러리까지 전체 흐름을 재현한다.
> 공통 세계관: 테스트 캐릭터 `소연`과 `비 오는 날` 이야기를 모든 단계에서 유지한다.

## 1. 운영 방향

| 기준 | 적용 |
| --- | --- |
| 기준 프롬프트 | 기능별 대표 프롬프트 1개를 유지하고 실패 원인만 반영해 버전을 올린다. |
| 비교 가능성 | 캐릭터·색상·스토리를 고정해 옵션과 모델에 따른 결과 차이를 비교한다. |
| 실제 산출물 | 프롬프트 문서에 실제 생성 PNG를 연결한다. |
| 단계형 테스트 | 간단 생성 → 키이미지 → 일관성 → 3×3 → 영상 → 라이브러리 순으로 진행한다. |
| 실패 기록 | 성공 결과뿐 아니라 숫자 노출·구도 중복·소품 변화 같은 실패도 기록한다. |

## 2. 전체 테스트 순서

| 단계 | ID | 검증 대상 | 입력·문서 | 완료 기준 |
| --- | --- | --- | --- | --- |
| 1 | ONB-01 | 첫 사용자 진입 | 한국어 한 줄 프롬프트 | 3분 안에 결과 생성 요청 완료 |
| 2 | IMG-A-01 | 캐릭터 다각도 | 이미지 문서 1절 | 정면·측면·후면, 동일 캐릭터, 전신 미절단 |
| 3 | IMG-A-02 | 배경 단독 | 이미지 문서 2절 | 인물 없음, 장소·시간·색상 반영 |
| 4 | IMG-B-01 | 키이미지 | 이미지 문서 3절 | 턴어라운드·표정·소품·팔레트 구분 |
| 5 | IMG-R-01 | 레퍼런스 일관성 | 이미지 문서 5절 | 얼굴·헤어·의상·핵심 소품 유지 |
| 6 | GRID-01 | 3×3 구도·연속성 | 스토리보드 문서 2절 | 9칸, 번호 없음, 9개 구도 중복 없음 |
| 7 | VID-01~03 | 영상 모델·입력 방식 | 영상 문서 1~3절 | 순서·컷 식별·전환·오디오 비교 기록 |
| 8 | LIB-01 | 라이브러리 연결 | 각 생성 결과 | 자동 저장·상세 메타데이터·재사용 확인 |

## 3. ONB-01 — 첫 생성 테스트

### 사용자에게 보여줄 입력

```text
비가 내린 이른 아침의 조용한 도시 골목을 만들어줘. 젖은 도로에 따뜻한 가게 불빛이 비치고 사람은 없게 해줘.
```

### 테스트 방법

1. 이미지 생성 메뉴 A의 배경을 선택한다.
2. 위 한국어 문장을 입력한다.
3. 프롬프트 향상 ON, 16:9, 표준 품질로 생성한다.
4. 같은 문장을 프롬프트 향상 OFF로 한 번 더 생성한다.
5. 두 결과와 실제 생성 프롬프트·소요 시간을 기록한다.

### 판정

- 사용자가 별도 설명 없이 입력 위치와 생성 버튼을 찾는다.
- ON 결과가 원래 의도를 잃지 않으면서 장소·조명·색감을 더 구체적으로 반영한다.
- OFF 결과도 오류 없이 원문으로 생성된다.
- 두 결과가 각각 라이브러리에 새 항목으로 자동 저장된다.

## 4. 이미지 생성 테스트

상세 프롬프트와 실제 생성 결과는 [이미지 생성 테스트 프롬프트](이미지/2026-07-15_테스트프롬프트_이미지.md)를 단일 기준으로 사용한다.

### IMG-A-01 캐릭터 다각도

- 입력: 이미지 문서 1절
- 필수 확인: 정면·측면·후면 / 동일 헤어·의상 / 흰 배경 / 전신·신발 미절단
- 실패 예: 세 인물의 얼굴이 서로 다름, 측면 대신 3/4 각도, 발이 잘림

### IMG-A-02 배경 단독

- 입력: 이미지 문서 2절
- 필수 확인: 인물·동물 없음 / 젖은 도로 / 이른 아침 / 따뜻한 가게 조명
- 실패 예: 읽을 수 없는 간판 글자, 인물 등장, 야간 장면으로 변형

### IMG-B-01 키이미지

- 입력: 이미지 문서 3절
- 필수 확인: 턴어라운드 / 표정 5종 / 우산·토트백·신발 / 4색 팔레트
- 실패 예: 소품 누락, 표정 중복, 턴어라운드 인물 비율 변화

### IMG-R-01 레퍼런스 일관성

- 입력: 키이미지 첨부 + 이미지 문서 5절
- 필수 확인: 검정 단발·틸 브릿지·머스터드 레인코트·투명 우산 유지
- 변동 허용: 포즈·카메라 각도·거리 배경·표정
- 실패 예: 의상 색상 변화, 우산 재질 변화, 다른 인물로 보이는 얼굴

## 5. GRID-01 — 3×3 스토리보드 테스트

상세 프롬프트와 현재 채택본은 [3×3 스토리보드 생성 프롬프트](영상/2026-07-15_테스트프롬프트_스토리보드_그리드.md)를 사용한다.

![현재 채택한 구도 차별화 스토리보드](영상/assets/02_비오는날_소연_3x3_스토리보드_구도차별화_v2.png)

### 필수 판정

- 정확히 3열×3행이며 칸 크기가 균등하다.
- 번호·라벨·로고·읽을 수 있는 글자가 없다.
- 좌상단부터 우하단까지 이야기가 진행된다.
- 캐릭터의 얼굴·헤어·의상·색상이 유지된다.
- 초광각·전신·로우앵글·하이앵글·지면앵글·오버숄더·3/4 미디엄·탑다운·클로즈업이 서로 겹치지 않는다.
- 특정 컷에만 앞서 없던 핵심 소품이나 인물이 갑자기 추가되지 않는다.

## 6. VID-01~03 — 영상 비교 테스트

[영상 생성 테스트 프롬프트](영상/2026-07-15_테스트프롬프트_영상.md)의 세 시나리오를 같은 날, 같은 스토리보드와 동일한 영상 문장으로 실행한다.

| ID | 방식 | 바꾸는 변수 | 고정하는 것 |
| --- | --- | --- | --- |
| VID-01 | Seedance 그리드 1장 | 슬라이스 없음 | 스토리·길이·오디오·프롬프트 |
| VID-02 | Seedance 9장 슬라이스 | 입력 이미지만 분할 | 스토리·길이·오디오·프롬프트 |
| VID-03 | Kling 시작·끝+그리드 | 모델·입력 매핑 | 스토리·길이·오디오·프롬프트 |

### 판정 지표

| 지표 | 기록 방식 |
| --- | --- |
| 순서 유지 | 1→9 순서 유지 / 일부 변경 / 유지 안 됨 |
| 컷 식별 | 구분 가능한 장면 수 0~9 |
| 구도 반영 | 스토리보드와 유사한 구도 수 0~9 |
| 캐릭터 일관성 | 상 / 중 / 하 |
| 전환 자연스러움 | 상 / 중 / 하 |
| 오디오 | 의도 반영 / 일부 반영 / 미반영 |
| 소요 시간·비용 | 실제 측정값 기록 |

## 7. LIB-01 — 자동 저장·재사용 테스트

각 단계 완료 후 아래 항목을 확인한다.

- [ ] 성공 결과가 별도 저장 버튼 없이 라이브러리에 표시된다.
- [ ] 실제 생성에 사용된 프롬프트가 상세 메타데이터에 남는다.
- [ ] 비율·품질·모델·생성일시가 요청값과 일치한다.
- [ ] 재생성 결과가 기존 항목을 덮어쓰지 않고 새 항목으로 추가된다.
- [ ] 이미지 상세에서 영상 생성 연결을 선택하면 해당 이미지가 영상 입력으로 전달된다.
- [ ] 다운로드·찜하기·재생성·삭제 등 현재 제공 액션이 정상 동작한다.

## 9. 세트 확장 기준

새 장르 프롬프트는 아래 중 하나가 확인될 때만 추가한다.

- 현재 `소연` 세트로 검증할 수 없는 별도 기능이 생긴 경우
- 동일 실패가 3회 이상 반복되어 원인 분리용 프롬프트가 필요한 경우
- 실사·제품·다인물처럼 스타일 또는 피사체 특성이 모델 결과를 크게 바꾸는지 비교해야 하는 경우

그 외에는 기준 세트의 프롬프트와 결과 버전만 올려 비교 가능성을 유지한다.
