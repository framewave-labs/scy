# A2팀 테스트 프롬프트·온보딩 Handoff v0.2

> 작성자: 손찬용 PM
> 기준일: 2026-07-15
> 범위: 내부 테스트 프롬프트, 실제 생성 이미지, 3×3 스토리보드 고도화, 영상 비교 테스트, 온보딩 프리셋·How-to, `lsg` 참고 비교, Git 상태
> 저장소: `framewave-labs/scy`

## 1. 인수인계 요약

이번 작업은 GeNova(작성 당시 임시명 `KX GenStudio` — 본사 지정명 2026-07-20 수령)의 이미지·영상 생성 기능을 내부에서 재현 가능하게 테스트하고, 같은 산출물을 온보딩 프리셋으로 재사용할 수 있도록 정리한 작업이다.

현재까지 완료된 결과는 다음과 같다.

- 이미지와 영상 테스트 프롬프트 문서를 폴더 단위로 분리했다.
- 공통 테스트 캐릭터 `소연`과 `비 오는 날` 이야기로 세계관을 고정했다.
- 캐릭터 다각도, 배경, 캐릭터 키이미지, 3×3 스토리보드 실제 PNG를 생성했다.
- 최초 3×3 스토리보드에서 구도가 비슷하게 반복되는 문제를 확인했다.
- 9개 컷의 샷 크기·카메라 높이·각도·인물 배치를 모두 다르게 지정한 v2를 다시 생성해 현재 채택본으로 연결했다.
- 프롬프트를 여러 세트로 늘리지 않고 `기준 프롬프트 1개 + 결과 버전 관리` 방식으로 운영하기로 했다.
- 온보딩 히어로, 프리셋 4종, 30초 How-to 영상은 **PM 제안안**으로 작성했으며 팀 합의·기능명세 반영 전에는 확정 기능으로 보지 않는다.
- 테스트를 장르별 예시 개수로 확장하지 않고 이미지 A → 이미지 B → 일관성 → 3×3 → 영상 → 라이브러리로 이어지는 단계형 운영 세트로 만들었다.
- `lsg`의 최신 프롬프트·3×3·QA 문서를 확인하고 방향만 참고했으며, 내용과 테스트 구조는 복제하지 않았다.
- 프리셋 노출 메타데이터는 생성자 랜덤명·품질·사이즈·생성일자·프롬프트로 확정하고 모델은 제외했다.
- 이슬기 PM과 프리셋을 외부 무료 플랫폼에서 수집하는 방식은 중단하고, 카테고리 틀만 참고해 GPT로 자체 생성하는 방향에 합의했다.
- 프롬프트 가이드를 v2.7로 갱신하고 세 영상 시나리오를 팀 표준 형식으로 재작성했다.

### 프롬프트 작성 주의

- 결과 이미지에 숫자가 그려질 수 있으므로 영상 입력용 프롬프트에서는 `Panel 1~9`보다 위치 기반 표현을 우선 검토한다.
- 모든 컷의 공통 캐릭터·의상·색상을 먼저 선언한다.
- 특정 컷에만 새로운 핵심 소품을 갑자기 추가하지 않는다.
- `no numbers, no text, no labels, no logos`를 명시한다.
- 9칸·번호 없음·순서·캐릭터 일관성뿐 아니라 **구도 중복 여부**를 별도 판정한다.

## 8. 프롬프트 운영 결정

### 채택 방식

`기준 프롬프트 1개 + 결과 버전 관리`

- 기능별 대표 프롬프트는 하나만 유지한다.
- 실패한 요소만 기준 프롬프트에 추가하고 문서 버전을 올린다.
- 생성 이미지를 덮어쓰지 않고 v1·v2처럼 남긴다.
- 테스트에서는 캐릭터·스토리·옵션을 최대한 고정하고 한 번에 하나의 변수만 바꾼다.

### 새 세트를 추가하는 조건

- 현재 소연 세트로 검증할 수 없는 기능이 생겼다.
- 같은 실패가 3회 이상 반복되어 원인 분리용 입력이 필요하다.
- 실사·제품·다인물처럼 피사체나 스타일 차이가 모델 결과를 크게 바꾸는지 비교해야 한다.

## 10. 영상 비교 테스트

### 준비된 시나리오

| ID | 방식 | 입력 |
| --- | --- | --- |
| VID-01 | Seedance `reference-to-video` | 현재 채택한 3×3 v2 이미지 1장 + 영상 프롬프트 |
| VID-02 | Seedance `reference-to-video` | v2를 좌상단부터 우하단 순서로 자른 이미지 9장 + `@ImageN` 참조 |
| VID-03 | Kling `image-to-video` | 시작 이미지·끝 이미지·그리드 레퍼런스 + 영상 프롬프트 |

### 고정 조건

- 길이: 15초
- 비율: 16:9
- 오디오: ON
- 캐릭터·스토리·영상 문장: 동일
- 기록: 생성 소요 시간, 실제 비용, 순서 유지, 컷 식별 개수, 구도 반영 개수, 캐릭터 일관성, 전환 자연스러움, 오디오 반영

### 실행 후 결정

- 그리드 1장이 9장 슬라이스와 비슷하면 슬라이스 개발 없이 1장 방식 유지 검토
- 9장 슬라이스가 확실히 좋으면 crop·순서 매핑 개발 검토
- Seedance와 Kling은 같은 스토리보드 영상 생성 용도로 유지하고, 결과에 따라 사용자 선택 기준과 운영 우선순위만 정리

현재는 **프롬프트와 입력 이미지만 준비된 상태**다. 실제 모델 결과가 없으므로 어떤 방식이 더 좋다고 확정하면 안 된다.

## 11. 온보딩·프리셋 Handoff

### 확정된 정책

- 이미지 프롬프트 향상 ON 시 변환된 영어 프롬프트를 생성 전에 사용자에게 보여주거나 수정하게 하지 않는다(07-14 확정).
- 영상 생성에는 프롬프트 향상을 제공하지 않고 입력 원문을 그대로 사용한다.
- 프리셋 노출 메타데이터는 `생성자 랜덤명 / 품질 / 사이즈 / 생성일자 / 프롬프트`이며 모델은 노출하지 않는다.
- 무료 플랫폼 프리셋 수집은 중단하고, 외부 서비스에서는 카테고리 틀만 참고해 GPT로 프리셋을 자체 생성한다.
- 이슬기 PM이 잡는 카테고리 틀에 현재 테스트 자산을 연결하고, 빠진 카테고리는 새로 생성한다.

### PM 제안 — 팀 합의 전 구현 금지

- 온보딩·히어로 섹션 제공
- 히어로 안에는 프롬프트 입력창을 두지 않고 결과 이미지를 먼저 제시
- 우선 CTA `이미지부터 만들기`, 보조 CTA `영상 만드는 과정 보기`
- 프리셋 선택 시 생성 화면 이동 및 프롬프트·옵션 초깃값 주입
- 30초 How-to 영상 제공

위 항목은 IA·기능명세서에 없는 신규 범위다. PM·UX/UI·FE 합의 후 기능명세서 DB에 프리셋 관련 행이 추가되기 전에는 확정 요구사항이나 작업 지시로 사용하지 않는다.

### 현재 테스트 예시 4종 — 최종 세트 아님

| ID | 이름 | 연결 기능 | 자산 |
| --- | --- | --- | --- |
| PRESET-01 | 캐릭터 시작 | 이미지 메뉴 A | 캐릭터 다각도 PNG |
| PRESET-02 | 배경 시작 | 이미지 메뉴 A | 비 오는 거리 PNG |
| PRESET-03 | 캐릭터 자세히 만들기 | 이미지 메뉴 B | 키이미지 시트 PNG |
| PRESET-04 | 영상 흐름 시작 | 영상 생성 | 3×3 스토리보드 v2 PNG |

### 30초 How-to 구성

1. 00~04초: 네 프리셋 결과 제시
2. 04~09초: 한국어 프롬프트 입력과 프롬프트 향상
3. 09~14초: 캐릭터·배경 결과
4. 14~19초: 키이미지
5. 19~25초: 3×3 스토리보드와 영상 생성
6. 25~30초: 결과와 라이브러리 자동 저장

04~09초에는 한국어 원문과 `프롬프트 향상 ON` 상태만 보여준다. 내부 변환 영어 프롬프트를 화면에 표시하거나 수정하는 연출은 사용하지 않는다.

### 아직 필요한 것

- UX/UI의 실제 히어로·카드·모바일 배치
- PM·UX/UI·FE의 온보딩 범위 합의와 기능명세서 행 추가
- UI 확정 후 How-to 영상 화면 캡처
- 자막 최종 카피와 음성·배경음 결정
- 완성 영상의 접근성·모바일 재생 확인

## 13. 역할별 다음 작업

### PM

- 프롬프트와 현재 기능명·옵션값이 일치하는지 최종 검수
- 실제 테스트 일정과 비용 한도 확인
- 이미지·영상 생성 결과 기록표 관리
- 이슬기 PM의 카테고리 틀과 현재 자체 생성 프리셋 예시 연결
- 온보딩 제안 범위 합의 주관 및 합의 후 기능명세서 행 추가
- 모델 비교 결과에 따라 기본 입력 방식을 결정

### UX/UI

- 합의 전에는 히어로·프리셋·CTA를 확정 화면으로 제작하지 않음
- 합의 후 프리셋 카드에 생성자 랜덤명·품질·사이즈·생성일자·프롬프트를 배치하고 모델은 제외
- 합의 후 30초 How-to 영상의 화면 전환·자막·무음 재생 기준 설계
- 생성 상태에서 확정되지 않은 가짜 진행률을 사용하지 않도록 주의

### FE

- 기능명세 행 추가 전에는 프리셋 이동·초깃값 주입을 구현 범위로 보지 않음
- 합의 후 프리셋 선택 시 이미지 A·이미지 B·영상 화면 이동과 시작값 주입 구현
- 프롬프트 향상 ON의 변환 영어 프롬프트를 생성 전에 노출하거나 수정 가능하게 만들지 않음
- 결과·자동 저장·라이브러리 이동 안내 표시
- 영상 입력 화면에서 번호 없는 3×3 이미지 1장을 검증

### BE

- 이미지 프롬프트 향상 ON/OFF 실제 생성 프롬프트·메타데이터 확인
- 생성 결과 자동 저장과 재생성 신규 항목 추가 확인
- Seedance 그리드 1장·9장 슬라이스 입력 매핑 확인
- Kling 시작·끝·레퍼런스 입력 매핑과 오디오 비용 확인
- 상태 조회·타임아웃·오류 응답을 QA 기록과 연결

### QA

- 운영 세트의 ID를 그대로 사용해 결과를 기록
- TXT·GRID·ID·PROP·COMP·ORDER·SAVE·TIME 실패 코드를 적용
- 3×3 테스트에서 구조·순서·일관성과 구도 반영을 따로 평가
- 영상은 모델 간 비교 날짜·입력·옵션을 동일하게 유지

## 16. 문서 관리 규칙

- `테스트프롬프트/README.md`를 진입점으로 유지한다.
- 이미지와 영상 문서·자산은 각각 `이미지/`, `영상/` 아래에 둔다.
- 현재 채택본은 문서에서 명확히 표시하고 이전 생성물은 비교용으로 보관한다.
- 기준 프롬프트는 실패 원인을 반영해 버전을 올리고 여러 유사 문서로 분기하지 않는다.
- 실제 영상 테스트 결과가 나오기 전에는 모델 우열을 확정 문구로 쓰지 않는다.
- Seedance·Kling은 같은 용도의 선택 모델로 표현하고, 기본·보조 역할은 테스트 전 지정하지 않는다.
- `scy`는 소스 문서 저장소로 유지하고 `framewave-docs`는 별도 요청 없이 수정하지 않는다.
- 작업물은 `게시글`이 아니라 `작업물` 또는 `라이브러리 항목`으로 표현한다.
- 영상 흐름은 `이미지 생성 → 저장 → 라이브러리 조회 → 영상 생성 → 상태 조회 → 결과 표시 → 라이브러리 재확인` 순서를 유지한다.
