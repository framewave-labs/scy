# A2팀 테스트 프롬프트 — 3×3 스토리보드 그리드 (GPT Image 2)

> 용도: 영상 생성 입력용 3×3 그리드 이미지 1장을 자체 생성하기 위한 프롬프트. PM단 제작본.
> 기준: 옵션값 확정 문서 4절 — 컷 순서 좌상단=1 → 우하단=9, **영상 입력용은 컷 번호 오버레이 없는 버전**(번호가 영상에 새겨질 수 있음).
> 캐릭터·스타일은 이미지 프롬프트 문서의 테스트 캐릭터 "소연"과 동일.

## 1. 9컷 스토리 정의 — "비 오는 날, 소연"

컷마다 장소·행동이 뚜렷하게 달라야 영상 테스트에서 "컷 식별 개수"를 판정하기 좋다. 15초 기준 컷당 약 1.5~2초.

| 컷 | 장면 |
| --- | --- |
| 1 | 아침, 창가에서 비 내리는 밖을 내다보는 소연 (실내, 잠옷 아님 — 흰 니트) |
| 2 | 현관에서 머스터드 레인코트를 입는 소연 |
| 3 | 투명 우산을 쓰고 비 오는 골목길을 걷는 소연 (전신, 측면) |
| 4 | 횡단보도 앞, 신호를 기다리는 소연 — 젖은 아스팔트에 네온 반사 |
| 5 | 버스 정류장, 물보라를 일으키며 도착하는 버스 |
| 6 | 버스 안 창가 자리, 김 서린 유리창에 손가락으로 그림을 그리는 소연 |
| 7 | 카페 앞에 도착해 우산을 접으며 물기를 터는 소연 |
| 8 | 카페 창가 자리에서 김이 나는 코코아 잔을 양손으로 감싼 소연 |
| 9 | 비가 그친 창밖 무지개를 보며 활짝 웃는 소연 |

## 2. 그리드 생성 프롬프트 — 영상 입력용 (번호 없음, 필수)

- 사용 옵션: 품질=고화질(high), 비율=16:9
- 검증 포인트: 9칸 균등 분할, 컷 번호·텍스트 없음, 전 컷 캐릭터 일관성

## 3. 그리드 생성 프롬프트 — 검수용 (번호 있음, 선택)

컷 순서를 사람이 검수할 때만 쓰는 버전. **영상 입력에는 사용 금지.**

```
(위 2절 프롬프트와 동일하되 첫 문장만 교체)
A 3x3 storyboard grid in a single 16:9 image: nine equal panels separated by thin white gutters, read in order from top-left to bottom-right. Each panel has a small charcoal number from 1 to 9 in its top-left corner, and no other text.
```

## 4. 사용 메모

- 생성된 그리드 1장이 영상 생성 화면의 입력이자, BE 비교 테스트(그리드 그대로 vs 9장 슬라이스)의 원본이다. 슬라이스(단순 crop 3×3)는 BE에서 처리한다.
- 균등 분할이 안 나오면(칸 크기 불균일) 슬라이스 테스트가 어긋나므로, 재생성해서 균등한 결과를 채택한다.
- 결과물은 영상 프롬프트 문서의 컷별 서술과 순서가 일치해야 한다 — 컷 내용이 다르게 나온 경우 이미지를 다시 뽑거나 영상 프롬프트를 결과에 맞게 수정해 **둘을 항상 일치시킨다**.
