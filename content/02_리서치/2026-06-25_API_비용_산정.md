# API 비용 산정

정리일: 2026-06-30
목적: 2026-07-01까지 팀에 공유할 이미지, 영상, 저장, 배포 비용 산정 기준을 정리한다.

## 1. 최신 산정 기준

| 항목 | 기준 |
| --- | --- |
| 이미지 생성 | GPT Image 2.0 기준 |
| 영상 입력 | 3x3 스토리보드 그리드 이미지 1장 + 9개 씬 프롬프트 |
| 영상 생성 단위 | 9개 영상을 따로 생성하지 않고 단일 영상 요청 기준으로 산정 |
| 영상 길이 | Higgsfield 기준 3~15초 |
| 향후 확장 | Seedance 2.5에서 30초 생성 가능 시 별도 재산정 |
| 프론트엔드 배포 | Vercel Hobby 플랜 |
| 백엔드 배포 | AWS EC2 |
| 파일 저장 | AWS S3 |
| DB / 메타데이터 | Supabase 또는 AWS 기반 DB 비교 후 확정 |

## 2. 비용 산정에서 제외할 과거 공식

기존 문서에는 `스토리보드 1세트 = 9클립` 방식의 영상 비용 계산이 남아 있었다. 최신 방향에서는 3x3 그리드 이미지 1장과 9개 씬 프롬프트를 묶어 단일 영상 생성 요청으로 처리하므로, 9클립 세트 계산을 그대로 쓰지 않는다.

```text
사용하지 않는 방식:
9개 컷 × 영상 길이 × 초당 단가

현재 필요한 방식:
단일 영상 요청 비용
또는
영상 길이 × 초당 단가
```

## 3. 이미지 생성 비용 항목

| 항목 | 산정 기준 | 확인 필요 |
| --- | --- | --- |
| 개발 테스트 | medium 품질 위주 | 팀당 예상 호출 수 |
| 중간점검 | 데모 후보 이미지만 high 사용 | 발표용 후보 수 |
| 최종 발표 | 최종 후보 이미지만 high 사용 | 최종 시연 이미지 수 |
| 재생성 | 실패와 프롬프트 수정 반복 포함 | 버퍼 비율 |

## 4. 영상 생성 비용 항목

| 항목 | 산정 기준 | 확인 필요 |
| --- | --- | --- |
| 개발 테스트 | 3~5초 짧은 영상 | Higgsfield 단가 |
| 중간점검 | 영상 기능은 필수 Pass 범위가 아님 | 필요 시 최소 호출 |
| 최종 발표 | 10~15초 후보 영상 | 데모 후보 수 |
| 실패 버퍼 | 실패 호출과 재생성 포함 | 버퍼 비율 |
| 향후 확장 | Seedance 2.5 30초 옵션 | 출시 시점과 단가 |

## 5. 저장 및 배포 비용 항목

| 항목 | 산정 기준 | 확인 필요 |
| --- | --- | --- |
| Vercel Hobby | 프론트엔드 무료 플랜 기준 | 사용량 제한 |
| AWS EC2 | 백엔드 서버 기준 | 인스턴스 타입, 운영 시간 |
| AWS S3 | 이미지/영상 원본 저장 | 저장 용량, 요청 수, egress |
| DB | Supabase 또는 AWS 기반 DB | 무료/유료 플랜, 저장량 |

## 6. 평가 기준별 비용 우선순위

| 목표 | 포함 기능 | 비용 관리 기준 |
| --- | --- | --- |
| 2주차 Pass | 이미지 생성 + 저장 + 라이브러리 | 이미지 호출 수와 저장 용량 중심 |
| B등급 | 이미지 생성 + 라이브러리 + 배포 | 배포 비용 포함 |
| A등급 | 이미지 생성 + 영상 생성 + 라이브러리 + 배포 | 영상 길이와 호출 수 제한 |
| S등급 | A등급 + 역프롬프트 재생성 루프 | 역프롬프트는 선택 기능으로 별도 산정 |

## 7. 수요일까지 확인할 항목

| 담당 논의 | 확인 내용 |
| --- | --- |
| PM | 목표 등급별 예상 호출 수 |
| Backend | Higgsfield 호출 방식, 단가, 상태 조회 비용 |
| Backend | AWS EC2/S3 예상 비용 |
| Frontend | Vercel Hobby 제한과 배포 방식 |
| 전체 | Supabase 유지 여부와 AWS 기반 DB 대안 |

## 8. 비용 관리 원칙

- 영상 테스트는 3~5초로 제한한다.
- 10~15초 영상은 최종 후보에만 사용한다.
- 역프롬프트는 선택 기능이므로 A등급 흐름이 안정화된 뒤 비용을 추가 산정한다.
- 생성 실패율이 높으면 프롬프트와 입력 구조를 먼저 조정한다.
- 비용 산정표에는 모델 단가의 출처와 확인 날짜를 함께 남긴다.
