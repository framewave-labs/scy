# A2팀 API DB 배포 검토 메모

## 1. 문서 목적

이 문서는 PRD에서 제외한 API, DB, 배포 관련 검토 항목을 개발팀 협의용으로 분리한 메모다.

아래 내용은 확정 명세가 아니라 개발자 협의 전 초안이다. 실제 엔드포인트, 데이터 구조, 인프라 구성은 Backend / Frontend 논의 후 확정한다.

## 2. 현재 제품 기준

| 영역 | 기준 |
| --- | --- |
| 이미지 생성 | GPT Image 2.0 / `gpt-image-2` |
| 영상 생성 입력 | 3x3 스토리보드 그리드 이미지 1장 + scene 1~9 프롬프트 |
| 영상 길이 | Higgsfield 기준 3~15초 |
| 프론트엔드 배포 | Vercel Hobby 플랜 |
| 백엔드 배포 | AWS EC2 |
| 파일 저장 | AWS S3 우선 검토 |
| DB / 메타데이터 | Supabase 또는 AWS 기반 DB 비교 후 확정 |
| 선택 기능 | 역프롬프트, 검색/필터, 다운로드 |

## 3. 개발팀 협의 필요 항목

| 항목 | 확인 질문 |
| --- | --- |
| API 서버 구조 | Next.js API Route로 충분한가, 별도 EC2 백엔드가 필요한가 |
| 이미지 저장 | 생성 이미지를 바로 S3에 저장할지, 임시 URL을 거쳐 저장할지 |
| 영상 생성 상태 | request id를 어디에 저장하고 어떤 주기로 조회할지 |
| 영상 결과 저장 | Higgsfield 결과 URL을 그대로 쓸지, S3에 복사 저장할지 |
| DB 선택 | Supabase를 유지할지, AWS RDS/DynamoDB 계열로 갈지 |
| 인증 | MVP에서 로그인 없이 진행할지, 최소 인증을 둘지 |
| 비용 제한 | 영상 생성 길이와 호출 수를 어디서 제한할지 |

## 4. API 후보

| Method | Endpoint 후보 | 목적 | 확정 여부 |
| --- | --- | --- | --- |
| POST | `/api/generate-image` | 이미지 생성 요청 | 개발팀 협의 필요 |
| POST | `/api/media` | 생성 결과 저장 | 개발팀 협의 필요 |
| GET | `/api/media` | 라이브러리 조회 | 개발팀 협의 필요 |
| GET | `/api/media/:id` | 미디어 상세 조회 | 개발팀 협의 필요 |
| POST | `/api/generate-video` | 영상 생성 요청 제출 | 개발팀 협의 필요 |
| GET | `/api/video-status/:requestId` | 영상 생성 상태 조회 | 개발팀 협의 필요 |
| POST | `/api/reverse-prompt` | 선택 기능: 이미지 분석 후 프롬프트 추출 | S등급 착수 시 협의 |

## 5. 데이터 모델 후보

### 5.1 `media_items`

이미지와 영상을 한 목록에서 보여주기 위한 후보 테이블이다.

| 필드 | 설명 |
| --- | --- |
| id | 미디어 고유 ID |
| type | image 또는 video |
| title | 사용자 입력 또는 자동 생성 제목 |
| prompt | 대표 프롬프트 |
| model | 사용 모델 |
| result_url | 생성 결과 URL |
| thumbnail_url | 썸네일 URL |
| status | saved, queued, processing, completed, failed |
| metadata | 영상 입력 정보, 이미지 옵션 등 추가 정보 |
| created_at | 생성일 |
| updated_at | 수정일 |

### 5.2 `video_jobs`

영상 생성 요청과 상태를 관리하기 위한 후보 테이블이다.

| 필드 | 설명 |
| --- | --- |
| id | 영상 작업 고유 ID |
| request_id | 영상 생성 API request id |
| grid_image_url | 3x3 스토리보드 그리드 이미지 URL |
| scene_prompts | scene 1~9 프롬프트 배열 |
| duration | 영상 길이 |
| model | Higgsfield 또는 최종 선택 모델 |
| status | queued, processing, completed, failed |
| result_url | 생성 완료 영상 URL |
| error_message | 실패 시 에러 메시지 |

## 6. 저장 구조 후보

```text
s3://kx-genstudio/
├─ images/{yyyy-mm}/{uuid}.png
├─ videos/{yyyy-mm}/{uuid}.mp4
└─ thumbnails/{yyyy-mm}/{uuid}.jpg
```

## 7. 배포 검토

| 영역 | 현재 방향 | 확인 필요 |
| --- | --- | --- |
| Frontend | Vercel Hobby | 환경변수, 도메인, GitHub 연동 |
| Backend | AWS EC2 | 인스턴스 크기, 배포 방식, HTTPS |
| Storage | AWS S3 | 버킷 정책, 공개 URL, 비용 |
| DB | Supabase 또는 AWS 기반 DB | 비용, 구현 난이도, 팀 숙련도 |

## 8. 개발팀 회의 전 준비물

| 담당 | 준비 내용 |
| --- | --- |
| Backend | 영상 생성 API의 request id, polling, result 저장 흐름 조사 |
| Frontend | 생성 화면과 라이브러리에서 필요한 상태값 목록 정리 |
| PM | 비용 산정 기준과 기능 우선순위 공유 |
| UX/UI | 로딩, 실패, 완료 상태 화면 기준 공유 |
