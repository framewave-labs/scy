# A2팀 API DB 명세서

## 1. 문서 목적

이 문서는 사용자가 업로드한 API 명세서와 DB 명세서 자료를 바탕으로, A2팀 생성형 AI 통합 엔진의 현재 백엔드 연동 기준을 정리한 문서다.

기획/프론트엔드/백엔드가 같은 기준으로 생성, 진행상태 조회, 파일 저장/조회 흐름을 확인할 수 있도록 API 목록과 DB 테이블 구조를 함께 둔다.

| --- | --- |
| API 명세 | 사용자가 업로드한 API 명세 자료, 2026-07-03 |
| DB 명세 | 사용자가 업로드한 DB 명세 자료, 2026-07-03 |

## 3. 핵심 요약

| 영역 | 현재 명세 |
| --- | --- |
| 생성 API | 이미지 생성, 역프롬프트 생성, 영상 생성 |
| 작업 상태 API | 영상 진행상태 조회 |
| 파일 API | 이미지 업로드, 파일 삭제, 파일 다운로드, 파일 목록 조회, 파일 상세 조회 |
| 미디어 자산 테이블 | `MediaFile` |
| 생성 작업 테이블 | `GenerateJob` |
| 저장 경로 | `MediaFile.file_path`에 AWS S3 또는 로컬 전체 경로 저장 |
| 작업 상태값 | `RUNNING`, `SUCCESS`, `FAILED` (2026-07-07: PENDING 상태 삭제 결정) |

### 4.1 전체 API 목록

| 구분 | 기능 | Method | URI |

| 생성 관련 API | 역프롬프트 생성 | `POST` | `/api/reverse-prompt` |
| 생성 관련 API | 영상 생성 | `POST` | `/api/generate-video` |

| 파일 관련 API | 이미지 업로드 | `POST` | `/api/file` |

### 4.2 생성 관련 API

| Method | URI | 기능 | 연결 데이터 |
| --- | --- | --- | --- |
| `POST` | `/api/generate-image` | 이미지 생성 요청 | 생성된 자산은 `MediaFile.type = IMAGE`로 저장 |
| `POST` | `/api/reverse-prompt` | 역프롬프트 생성 요청 | 추출 결과는 `MediaFile.reversed_prompt`와 연결 가능 |
| `POST` | `/api/generate-video` | 영상 생성 요청 | 생성 작업은 `GenerateJob.type = VIDEO`로 관리 |

### 4.3 JOB 관련 API

| Method | URI | 기능 | 연결 데이터 |
| --- | --- | --- | --- |
| `GET` | `/api/video-status` | 영상 생성 진행상태 조회 | `GenerateJob.status` 상태값 기준 |

진행상태 조회 API의 요청 파라미터 또는 조회 키는 업로드된 자료 기준으로 별도 표기되어 있지 않다. 프론트엔드 연동 전 `job id`, `file id`, query parameter 사용 여부를 백엔드와 확인해야 한다.

### 4.4 파일 관련 API

| Method | URI | 기능 | 연결 데이터 |
| --- | --- | --- | --- |
| `POST` | `/api/file` | 이미지 업로드 | 업로드 결과는 `MediaFile`에 저장 |
| `DELETE` | `/api/file/{id}` | 파일 삭제 | `MediaFile.id` 기준 삭제 |
| `GET` | `/api/file/{id}/download` | 파일 다운로드 | `MediaFile.file_path` 기준 다운로드 |
| `GET` | `/api/file` | 파일 목록 조회 | `MediaFile` 목록 조회 |
| `GET` | `/api/file/{id}` | 파일 상세 조회 | `MediaFile.id` 기준 상세 조회 |

## 5. DB 명세

### 5.1 `MediaFile`

이미지와 영상 파일 자산을 저장하는 테이블이다.

| 컬럼명 | 타입 | 제약조건 | 설명 |
| --- | --- | --- | --- |
| `id` | `Long` | `PRIMARY KEY` | 자산 고유 식별자 |
| `type` | `VARCHAR(10)` | `NOT NULL`, `CHECK (type IN ('IMAGE', 'VIDEO'))` | 파일 타입 구분 |
| `file_path` | `VARCHAR` | `NOT NULL` | AWS S3 또는 로컬 전체 경로 |
| `origin_prompt` | `VARCHAR` | `NOT NULL` | 생성에 사용된 원본 프롬프트 |
| `reversed_prompt` | `VARCHAR` | `NULL` | 이미지 분석을 통해 추출된 역프롬프트 |
| `tags` | `VARCHAR(512)` | `NULL` | 검색 및 필터링용 태그 문자열 |
| `created_at` | `TIMESTAMP` | `DEFAULT NOW()`, `NOT NULL` | 생성 일시 |

### 5.2 `GenerateJob`

이미지, 영상, 프롬프트 생성 작업의 상태를 저장하는 테이블이다.

| 컬럼명 | 타입 | 제약조건 | 설명 |
| --- | --- | --- | --- |
| `id` | `Long` | `PRIMARY KEY` | 작업 고유 식별자 |
| `file_id` | `Long` | `NULL`, `REFERENCES media_file(id)` | 완료(`SUCCESS`) 시 생성된 파일 ID |
| `type` | `VARCHAR(20)` | `NOT NULL` | 작업 유형: `IMAGE`, `VIDEO`, `PROMPT` |
| `status` | `VARCHAR(20)` | `NOT NULL`, `DEFAULT 'RUNNING'` | 진행 상태: `RUNNING`, `SUCCESS`, `FAILED` (PENDING 삭제, 작업 생성 시 바로 `RUNNING`으로 기록 — 실제 큐 대기 처리 방식은 BE 확인 필요) |
| `created_at` | `TIMESTAMP` | `DEFAULT NOW()`, `NOT NULL` | 작업 요청 시간 |
| `updated_at` | `TIMESTAMP` | `DEFAULT NOW()`, `NOT NULL` | 마지막 상태 변경 시간 |

## 6. API와 DB 연결 흐름

### 6.1 이미지 생성

1. 프론트엔드에서 `/api/generate-image`로 이미지 생성을 요청한다.
2. 백엔드는 생성 작업을 `GenerateJob`으로 관리한다.
3. 생성 성공 시 결과 파일 정보를 `MediaFile`에 저장한다.
4. `GenerateJob.file_id`는 완료된 `MediaFile.id`를 참조한다.

### 6.2 역프롬프트 생성

1. 프론트엔드에서 `/api/reverse-prompt`로 역프롬프트 생성을 요청한다.
2. 백엔드는 이미지 분석 결과로 추출한 프롬프트를 반환한다.
3. 저장이 필요한 경우 `MediaFile.reversed_prompt`에 연결한다.

### 6.3 영상 생성 및 상태 조회

1. 프론트엔드에서 `/api/generate-video`로 영상 생성을 요청한다.
2. 백엔드는 영상 생성 작업을 `GenerateJob`에 기록한다.
3. 프론트엔드는 `/api/video-status`로 영상 진행상태를 조회한다.
4. 상태값은 `RUNNING`, `SUCCESS`, `FAILED`를 기준으로 표시한다 (2026-07-07: PENDING 삭제).
5. 영상 생성 성공 시 결과 파일 정보를 `MediaFile`에 저장하고 `GenerateJob.file_id`와 연결한다.

### 6.4 파일 관리

1. 업로드된 이미지는 `/api/file`을 통해 `MediaFile`에 등록된다.
2. 라이브러리 화면은 `/api/file`과 `/api/file/{id}`를 통해 목록과 상세 정보를 조회한다.
3. 다운로드는 `/api/file/{id}/download`를 사용한다.
4. 삭제는 `/api/file/{id}`를 사용한다.

## 7. 프론트엔드 연동 시 확인 필요 사항

| 항목 | 확인 내용 |
| --- | --- |
| 요청/응답 스키마 | 각 API의 request body, response body, error response 형식 |
| 진행상태 조회 키 | `/api/video-status`에서 어떤 식별자로 작업을 조회하는지 |
| 파일 업로드 필드명 | `multipart/form-data` 사용 여부와 파일 필드명 |
| 파일 삭제 정책 | DB row 삭제만 수행하는지, 실제 S3/로컬 파일도 함께 삭제하는지 |
| 다운로드 응답 | redirect URL, stream, presigned URL 중 어떤 방식인지 |
| 역프롬프트 저장 범위 | 결과를 즉시 반환만 하는지, `MediaFile.reversed_prompt`에 저장하는지 |
| 테이블명 표기 | 문서 표기 `MediaFile`과 FK 표기 `media_file(id)` 중 실제 물리 테이블명 |
| 인증/권한 | MVP에서 인증 없이 사용하는지, 파일 접근 권한을 둘지 |
