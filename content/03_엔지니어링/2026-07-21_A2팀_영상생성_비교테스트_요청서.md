# 영상 생성 방식 비교 테스트 요청

**같은 스토리보드 1장으로 3가지 방식을 만들어보고, 어느 게 제일 나은지 정하는 테스트입니다.**

## 테스트 3건

| # | 방식 | 모델 |
| --- | --- | ---: |
| 1 | 그리드 1장을 그대로 입력 | Seedance R2V |
| 2 | 그리드를 9장으로 잘라서 입력 | Seedance R2V |
| 3 | 컷1=시작, 컷9=끝 + 그리드 레퍼런스 (지금 방식) | Kling |

- 전부 **15초·720p, 오디오 ON**
- 합계 약 **$10.5** — fal.ai에 지출 상한이 없으니 **3건만** 부탁드려요. 더 돌려야 할 것 같으면 먼저 알려주세요.

## 입력 자산 (scy 저장소)

- 그리드 이미지: `테스트프롬프트/영상/assets/02_비오는날_소연_3x3_스토리보드_구도차별화_v2.png`
- 영상 프롬프트: `테스트프롬프트/영상/2026-07-15_테스트프롬프트_영상.md`

## 각 건 기록

- 결과 영상 + 생성 소요 시간
- **순서 유지 / 컷 식별 개수(9컷 중 몇 개) / 전환 자연스러움**

## 결과로 정할 것

- 1번이 2번만큼 잘 나오면 → 자르는 처리 개발 안 함
- 2번이 확실히 나으면 → 자르는 처리 개발
- 3번이 제일 나으면 → Kling 방식 유지

Kling 출력 비율이 입력 이미지 비율을 따라가는지도 이번에 같이 확인해주시면 좋습니다.
