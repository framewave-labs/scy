# A2팀 영상 생성 방식 비교 테스트 요청서

> 요청자: 손찬용 PM | 대상: 영상 담당 BE(김유신) | 기준일: 2026-07-21
> 근거: 옵션값 확정 v0.11 4.3절, 07-13 확정 시나리오. 3주차 이월 항목.
> 목적: 3×3 스토리보드 그리드 1장을 영상으로 만들 때 **BE 내부 처리 방식을 어느 것으로 확정할지** 데이터로 정한다.

## 1. 배경 (왜 하는가)

- 사용자 입력은 **"3×3 그리드 이미지 1장 + 프롬프트 1개"로 확정**(07-14). 사용자가 9장을 개별 업로드하는 게 아니다.
- 그 1장을 모델에 어떻게 넣느냐(그대로 vs 9장 슬라이스 vs Kling 현행)는 **미확정**이며, 아래 3건을 나란히 비교해 정한다.
- I2V 모델은 입력 이미지를 첫 프레임으로 쓰기 때문에 그리드 1장을 통째로 넣으면 그리드가 화면에 노출될 수 있다는 게 슬라이스 가설의 출발점이나, "슬라이스가 애초에 불필요할 수도 있다"는 가설(1번)도 함께 검증한다.

## 2. 입력 자산 (PM 제작·전달 완료)

| 자산 | 위치 |
| --- | --- |
| 3×3 스토리보드 그리드 이미지 (번호 오버레이 없음, 검수 완료) | `scy/content/01_기획/테스트프롬프트/영상/assets/02_비오는날_소연_3x3_스토리보드_구도차별화_v2.png` |
| 컷별 영상 프롬프트 (scene/shot 1~9) | `scy/content/01_기획/테스트프롬프트/영상/2026-07-15_테스트프롬프트_영상.md` |
| 그리드 생성 기준·컷 정의 | `scy/content/01_기획/테스트프롬프트/영상/2026-07-15_테스트프롬프트_스토리보드_그리드.md` |

- 컷 순서: 좌상단=1 → 우하단=9. 컷당 1.5~2초(15초 기준).

## 3. 테스트 3건 (조건 통제)

공통 조건: 같은 스토리보드 1장, **15초·720p**, 프롬프트 원문 투입, 매 건 생성 소요 시간 기록.

| # | 방식 | 모델·엔드포인트 | 예상 비용 |
| --- | --- | --- | ---: |
| 1 | 그리드 **1장 그대로** 입력 (슬라이스 불필요 가설) | Seedance R2V `bytedance/seedance-2.0/reference-to-video` | 약 $4.6 |
| 2 | **9장 슬라이스** 입력 (`@Image1`~`@Image9` 참조) | Seedance R2V (동일) | 약 $4.6 |
| 3 | **Kling 현행 방식** — 컷1=`start_image_url`, 컷9=`end_image_url`, 그리드 전체 레퍼런스 추가 | Kling 3.0 Standard `fal-ai/kling-video/v3/standard/image-to-video` | 약 $1.3 |

- 오디오: 두 모델 기본 `ON` 확정. (Seedance는 요금 동일, Kling은 ON 시 +50%)
- 3건 합계 약 $10.5. fal.ai는 자동 지출 상한(Hard Limit)이 없으니 **딱 3건만** 돌려주세요. 반복·추가 생성이 필요하면 먼저 공유 바랍니다.

## 4. 각 건에 기록할 것

1. 결과 영상 파일
2. 생성 소요 시간 (진행 표시 UI 판단용)
3. 판정 3항목:
   - **순서 유지**: 컷 1→9 순서대로 전개되는가
   - **컷 식별 개수**: 9컷 중 몇 개가 알아볼 수 있게 나오는가
   - **전환 자연스러움**: 컷 간 전환이 매끄러운가

## 5. 판정 갈림길 (결과 → 결정)

- **1번 ≈ 2번** (1장이 슬라이스만큼 잘 나옴) → 슬라이스 전처리 개발 **불필요**, 그리드 1장 방식으로 확정 (BE 작업량 최소)
- **2번이 확실히 우세** → 슬라이스 전처리(crop + `@ImageN` 변환) 개발 진행
- **3번이 둘 다 이김** → Kling 현행 방식을 기본으로 검토

이 결과로 옵션값 확정 문서 4.3절과 유즈케이스 통합본 미결 #2·#3을 종결한다.

## 6. 참고 (엔드포인트 메모)

- Seedance R2V: 레퍼런스 이미지 최대 9장, `@ImageN` 프롬프트 참조, 멀티샷 컷 전환 네이티브 지원. 2번은 프롬프트를 `@ImageN` 형식으로 변환하는 전처리 필요(규칙 기반 치환 가능 여부 확인).
- Kling Standard: 레퍼런스 슬롯 최대 4장이라 9장 동시 입력 불가 → 시작·끝 프레임 + 그리드 레퍼런스 방식. 1080p 고정(출력 비율이 입력 이미지 비율을 따르는지도 이번에 확인해주면 좋음).
