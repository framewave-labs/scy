# A2팀 QA 및 평가 기준 v2

> 갱신일: 2026-07-20 | 작성자: 손찬용 PM
> 기준 문서: framewave-docs `00_통합문서` — 최종 기능명세서 v0.4, 최종 유즈케이스 v1.2, 최종 마일스톤 v3.3, 최종 PRD v1.2
> v0.1(06-30) 대비: 통합문서 기능 ID 체계로 전면 갱신, 3주차까지의 확정 사항(자동 저장·스피너·오디오 ON·AI 프롬프트 변환·한도 미적용) 반영, 등급 판정을 기능 ID 매핑 기준으로 재정의

## 1. 문서 목적과 역할 분담

이 문서는 **판정 기준**을 관리한다 — 무엇이 통과이고, 등급이 어떻게 결정되는지.

항목별 실행 체크는 이슬기 PM의 **기능 QA 체크리스트 v2**(`lsg/content/04_QA/2026-07-20_기능QA체크리스트_v2_통합문서기준.md`, 47개 기능 전수)로 수행한다. 두 문서의 관계:

| 문서 | 역할 |
| --- | --- |
| 기능 QA 체크리스트 v2 (이슬기 PM) | 기능별 Pass/Fail 실행 기록, 버그 기록, FE/BE 구현 현황 대조 |
| QA 및 평가 기준 v2 (이 문서) | Pass 판정 원칙, 등급(B/A/S) 판정 조건, 생성 품질 기준, 데모 리허설 체크 |

## 2. 판정 원칙

1. **Pass 기준은 배포 URL**이다. 로컬에서만 동작하는 기능은 Pass가 아니다 (A007 배포가 B등급 필수이므로).
2. **정책 미확정 항목은 Fail이 아니라 "보류"**로 기록한다 (체크리스트 v2와 동일 원칙). 보류 항목은 등급 판정에서 제외하되, 최종 기능명세서의 확인필요 목록과 대조해 관리한다.
3. **AWS·S3 이후 항목**(삭제·계정 탈퇴·자동 저장 실패 처리·라이브러리 필터/정렬 세부·웹훅/폴링 확정)은 07-14 협의로 착수 시점이 뒤로 확정된 것이므로, 미구현 상태는 Fail이 아니라 "일정 내"로 판정한다. 단 **S3 전환 완료 후에는 정상 판정 대상**으로 전환한다.
4. 버그 심각도는 체크리스트 v2의 P0~P3를 따르며, 등급 판정에는 다음과 같이 반영한다:
   - **P0(블로커) 미해결**: 해당 기능 ID를 미완료로 취급 → 등급 조건에서 제외
   - **P1(주요) 미해결**: 기능은 완료로 인정하되 발표 전 수정 목록에 필수 등재
   - P2·P3: 등급 판정에 영향 없음

## 3. 등급 판정 기준 (기능 ID 매핑)

PRD의 등급 임계값을 QA 통과 조건으로 옮긴 것이다. 각 등급은 **해당 기능 ID 전부가 배포 URL에서 P0 없이 동작**해야 성립한다.

| 등급 | 필요 기능 | QA 관점 성립 조건 |
| --- | --- | --- |
| **B** | M001~M006 + A002~A005 + A006 + A007 | 이미지 생성 전체 흐름(입력→생성→표시→자동 저장→실패 처리) + 영상 생성 전체 흐름(입력→생성→상태 조회→결과 표시→재생성) + 통합 라이브러리 + FE Vercel·BE AWS 배포 |
| **A** | B + A001 | B 조건 + 방향성 레퍼런스 확보(캐릭터·배경 시트) — 메뉴 A/B 흐름으로 캐릭터 시트·배경 레퍼런스가 실제 생성·저장됨 |
| **S** | A + S001~S004 | A 조건 + 역프롬프트 루프(업로드→분석→추출→수정→재생성) 전체 시연 가능 |
| C | B 미달 | 이미지 생성만 시연 가능 |
| Fail | — | 핵심 API 연동 실패 (GPT Image 2 또는 배포) |

등급 판정 시점: 4주차 최종평가. 중간 판정은 체크리스트 v2 진행률로 갈음한다.

## 4. 핵심 흐름별 완료 기준 (3주차 확정 반영)

체크리스트 v2가 항목 단위를 커버하므로, 여기서는 **흐름 단위의 합격선**과 확정 정책 위반 여부만 정의한다.

### 4.1 이미지 생성 (M001~M006, A001)

| 확인 | 완료 기준 |
| --- | --- |
| 메뉴 A 흐름 | 프롬프트(한/영) 입력→생성→결과 표시→**자동 저장** (저장 버튼이 있으면 정책 위반) |
| 메뉴 B 흐름 | 속성 선택 또는 자유 텍스트→키이미지 1장(턴어라운드+표정+소품+팔레트) 생성·자동 저장 |
| AI 프롬프트 변환 토글 | 명칭이 "AI 프롬프트 변환"으로 표기, ON 시 변환 영문이 **화면에 노출되지 않음**, 히스토리에는 실제 생성 프롬프트 저장 |
| 진행 표시 | 스피너 + 안내 문구. 퍼센트·가짜 프로그레스 바가 있으면 정책 위반 |
| 한도 | 일일 한도·크레딧 UI나 차단 흐름이 **없어야 함** (07-15 미적용 확정) |

### 4.2 영상 생성 (A002~A005)

| 확인 | 완료 기준 |
| --- | --- |
| 입력 | 3×3 그리드 이미지 **1장** + 프롬프트 1개. 9장 개별 업로드 UI가 있으면 정책 위반 |
| 모델·옵션 | Kling 3.0 / Seedance 2.0 선택, 길이는 모델별 범위(3~15초/4~15초) 내 선택, 미지원 옵션 비활성화 |
| 오디오 | 기본 `ON` (07-14 확정) — 무음으로 나오면 Fail |
| 상태 조회 | 대기 중→생성 중→완료/실패 구분, 스피너 방식, 화면 이탈 후에도 서버 측 생성·저장 지속 |
| 결과·재생성 | 인라인 재생(오디오 포함), 재생성 시 기존 결과 유지 + 새 항목 저장 |

### 4.3 라이브러리 (A006)

| 확인 | 완료 기준 |
| --- | --- |
| 통합 조회 | 이미지·영상 혼합 그리드, 최신순, 타입 필터, 빈 상태 CTA |
| 상세 | 공통 모달: 미디어·프롬프트·모델·옵션·생성일·찜 여부, 다운로드 동작 |
| 찜하기 | 즐겨찾기 토글로 동작(저장 버튼 아님), 해제해도 항목 유지 |
| 삭제 | S3 전환 전에는 보류 항목, 전환 후 확인 모달 포함 동작 |

### 4.4 배포 (A007)

| 확인 | 완료 기준 |
| --- | --- |
| FE | Vercel 배포 URL에서 전체 화면 접근 가능 |
| BE | AWS 배포, FE에서 실 API 호출 성공 (CORS·환경변수·health check) |
| 통합 | 배포 URL 기준으로 3절의 등급 조건 흐름이 재현됨 |

## 5. 생성 품질 QA (PM 검증 몫)

기능 동작과 별개로, 결과물이 방송 레퍼런스 용도에 부합하는지 PM이 판정한다. 테스트 입력은 PM 제작 자산(테스트 프롬프트 문서, 프리셋 72종)을 사용한다.

| 항목 | 판정 기준 |
| --- | --- |
| 키이미지 구성 요소 | 캐릭터 시트에 턴어라운드(정면·측면·후면)+표정+소품+컬러 팔레트가 실제 포함되는가 |
| 키이미지 일관성 | 같은 설정으로 재생성 시 캐릭터 동일성이 유지되는가 (키이미지 정의 07-07 PD 확인 기준) |
| 배경 시트 | 공간의 시각적 특징이 식별 가능한가 (다각도 불필요 — PD 확인 기준) |
| 한국어 프롬프트 품질 | AI 프롬프트 변환 ON에서 한국어 입력 결과가 의도와 일치하는가 |
| 영상 컷 유지 | 3×3 스토리보드의 컷 순서·장면이 영상에서 식별되는가 (비교 테스트 판정 기준과 동일: 순서 유지/컷 식별 수/전환 자연스러움) |
| 금지 요소 | 결과물에 로고·워터마크·읽히는 브랜드 문구·기존 IP가 없는가 |

## 6. 데모 전 최종 체크 (발표 리허설)

체크리스트 v2의 "클라이언트(PD) 관점 최종 체크 7문항"과 함께 발표 전날 수행한다.

- 데모 계정·환경변수 준비 완료
- fal.ai 잔액·지출 알림 확인 (Hard Limit 부재 — 한도 기능이 없으므로 수동 확인 필수)
- 생성 실패 시 대체 시나리오: 라이브러리에 사전 생성된 샘플(프리셋 이미지 72종·영상, 테스트 프롬프트 결과물) 준비
- 시연용 3×3 그리드 이미지·프롬프트 세트 준비 (번호 오버레이 없는 버전)
- 발표용 3분 시연 흐름 리허설 (로그인→이미지 A→이미지 B→영상→라이브러리)
- 영상 생성 대기 시간(수십 초~수 분)을 발표 흐름에서 어떻게 처리할지 결정 (사전 생성분 전환 타이밍)
