# A2팀 QA 및 평가 기준

## 1. 문서 목적

이 문서는 A2팀의 중간점검과 최종평가에서 확인할 QA 항목을 정리한다.

PRD는 제품 요구사항을 다루고, 이 문서는 실제 화면에서 무엇을 점검할지 관리한다.

## 2. 2주차 중간점검 QA

| 항목 | 확인 내용 | 완료 기준 |
| --- | --- | --- |
| 프롬프트 입력 | 사용자가 이미지 생성 프롬프트를 입력할 수 있는가 | 빈 입력과 정상 입력이 구분된다. |
| 이미지 생성 | GPT Image 2.0 호출이 성공하는가 | 테스트 프롬프트로 이미지가 생성된다. |
| 결과 표시 | 생성 이미지가 화면에 보이는가 | 결과 영역에 이미지가 표시된다. |
| 저장 | 생성 결과를 라이브러리에 저장할 수 있는가 | 저장 후 성공 상태가 보인다. |
| 조회 | 저장 결과를 다시 볼 수 있는가 | 라이브러리에서 저장 이미지가 확인된다. |
| 상태 UI | 로딩, 실패, 빈 상태가 보이는가 | 사용자가 현재 상태를 이해할 수 있다. |

## 3. 4주차 최종평가 QA

| 항목 | 확인 내용 | 완료 기준 |
| --- | --- | --- |
| 이미지 생성 | 배경 시트, 캐릭터 시트, 키이미지를 생성할 수 있는가 | 생성 결과가 저장 가능하다. |
| 영상 입력 | 3x3 그리드 이미지와 9개 씬 프롬프트를 입력할 수 있는가 | 누락 입력이 검증된다. |
| 영상 생성 | 3~15초 영상 생성 요청이 가능한가 | 생성 요청 후 상태가 표시된다. |
| 영상 상태 | 대기, 진행, 완료, 실패 상태가 구분되는가 | 완료 시 영상 미리보기가 가능하다. |
| 라이브러리 | 이미지와 영상이 함께 관리되는가 | 타입 구분과 상세 확인이 가능하다. |
| 배포 | 데모 URL에서 주요 흐름이 동작하는가 | 평가자가 접속해 시연할 수 있다. |

## 4. 선택 기능 QA

역프롬프트는 S등급 선택 기능이므로 A등급 핵심 흐름이 안정화된 뒤 점검한다.

| 항목 | 확인 내용 | 완료 기준 |
| --- | --- | --- |
| 이미지 업로드 | 레퍼런스 이미지를 업로드할 수 있는가 | 지원 형식만 허용된다. |
| 프롬프트 추출 | 이미지 분석 결과가 프롬프트로 표시되는가 | 사용자가 재사용 가능한 문장으로 확인한다. |
| 프롬프트 수정 | 추출 결과를 직접 수정할 수 있는가 | 수정 내용이 재생성에 반영된다. |
| 이미지 재생성 | 수정 프롬프트로 이미지를 생성할 수 있는가 | 재생성 결과가 화면에 표시된다. |
| 저장 | 재생성 결과가 라이브러리에 저장되는가 | 저장 후 조회가 가능하다. |

## 5. 최종평가 등급 매핑

| 등급 | QA 관점 기준 |
| --- | --- |
| S | 이미지, 영상, 역프롬프트 재생성 루프, 라이브러리, 배포가 모두 시연된다. |
| A | 이미지, 영상, 라이브러리, 배포가 시연된다. |
| B | 이미지 생성, 라이브러리, 배포가 시연된다. |
| C | 이미지 생성만 시연된다. |
| Fail | 핵심 API 연동이 실패한다. |

## 6. 데모 전 최종 체크

- 데모 계정 또는 환경변수가 준비되어 있다.
- API 비용 한도가 설정되어 있다.
- 생성 실패 시 보여줄 대체 시나리오가 있다.
- 저장된 샘플 이미지와 영상이 준비되어 있다.
- 발표용 3분 시연 흐름이 정리되어 있다.
