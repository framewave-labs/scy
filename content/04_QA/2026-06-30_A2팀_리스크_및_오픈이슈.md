# A2팀 리스크 및 오픈 이슈

## 1. 문서 목적

이 문서는 PRD에서 분리한 리스크와 오픈 이슈를 추적하기 위한 QA/이슈 관리 문서다.

## 2. 주요 리스크

| 리스크 | 영향 | 대응 |
| --- | --- | --- |
| GPT Image 2.0 API 호출 실패 | 2주차 Pass 실패 가능 | 1주차에 최소 호출 테스트를 먼저 진행한다. |
| 영상 생성 비용 증가 | 예산 초과 | 테스트는 3~5초, 데모 후보만 10~15초로 제한한다. |
| 영상 생성 시간 지연 | 사용자 경험 저하 | 대기, 진행, 완료, 실패 상태를 화면에 분명히 보여준다. |
| 3x3 그리드 해석 실패 | 영상 품질 저하 | scene 1~9 프롬프트를 명확히 분리하고 그리드 순서를 안내한다. |
| DB 선택 지연 | 저장 기능 일정 지연 | Supabase와 AWS 기반 DB를 비용/난이도 기준으로 빠르게 비교한다. |
| 배포 구조 복잡도 증가 | 최종 시연 지연 | 프론트 Vercel, 백엔드 AWS 역할을 명확히 분리한다. |
| 기능 범위 과다 | 핵심 흐름 미완성 | 2주차 Pass와 4주차 A등급 범위를 우선한다. |

## 3. 오픈 이슈

| 항목 | 확인 필요 내용 | 기한 |
| --- | --- | --- |
| Higgsfield 단가 | 3~15초 영상 생성 비용과 호출 방식 확인 | 2026-07-01 |
| Seedance 2.5 | 30초 생성 가능 시점, 단가, API 제공 방식 확인 | 추후 |
| DB 선택 | Supabase 유지 또는 AWS 기반 DB 전환 여부 | 2026-07-01 |
| 배포 비용 | Vercel Hobby와 AWS EC2/S3 기준 예상 비용 산정 | 2026-07-01 |
| UX/UI 미팅 | 디자인 시스템과 컬러칩 방향 확정 | 2026-07-01 |
| 영상 입력 UX | 3x3 그리드와 9개 씬 프롬프트를 한 화면에서 입력하는 방식 확정 | 2026-07-01 |

## 4. 의사결정 기준

| 상황 | 판단 기준 |
| --- | --- |
| 비용이 높아질 때 | 영상 길이와 생성 횟수를 먼저 줄인다. |
| 일정이 지연될 때 | 역프롬프트와 검색/필터를 후순위로 내린다. |
| 기술 선택이 갈릴 때 | 2주차 Pass와 4주차 배포 가능성을 우선한다. |
| 결과 품질이 낮을 때 | 프롬프트 구조와 레퍼런스 입력을 먼저 조정한다. |
