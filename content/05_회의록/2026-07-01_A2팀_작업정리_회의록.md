# A2팀 작업 정리 회의록

## 기본 정보

- 일시: 2026-07-01, 작업 정리 기준
- 참석자: A2팀 팀원
- 장소 또는 채널: 팀 작업 공유
- 작성자: 손찬용 PM

## 안건

- API 비용 확정 및 팀별 예산안 제출
- PM 문서 작업 현황 공유
- 백엔드 Git 세팅 진행 상황 공유
- UX/UI 디자인 방향 정리
- 2주차 Pass 기준 및 이후 작업 범위 재확인

## 논의 내용

### 1. API 비용 및 예산안

2026-07-01 작업에서 A2팀의 API 비용 기준을 확정했다.

PM은 이미지 생성, 영상 생성, 고정 구독 항목을 포함해 API 예산안을 설정했고, 각 팀별 예산안을 제출했다. 이후 팀별 예산과 기획 내용을 통합 문서에 반영했다.

비용 운영에서는 영상 생성 비용 리스크가 가장 크므로, 개발 테스트와 데모 호출을 분리해 관리하기로 했다. 또한 fal.ai 사용 시 spend limit 설정을 통해 과금 리스크를 줄이는 방향을 확인했다.

### 2. PM 문서 작업

손찬용 PM은 A2팀 PRD 초안을 업로드했다.

PRD 초안에는 프로젝트 목표, 대상 사용자, MVP 범위, 2주차 Pass 기준, 4주차 A등급 목표, S등급 도전 범위, 리스크 및 다음 액션을 정리했다.

이슬기 PM은 전체 일정 관리를 위한 마일스톤 초안을 작성했다.

마일스톤 초안은 2026-07-02부터 최종 발표 전까지의 작업 흐름을 주차별로 나누고, PM, Backend, Frontend, UX/UI 역할별 진행 기준을 확인할 수 있도록 작성했다.

### 3. Backend 작업

백엔드 담당자는 본격적인 개발 착수를 위해 Git 세팅에 집중했다.

이번 작업의 목적은 이후 API 연동, 배포, 협업 작업을 원활하게 진행할 수 있도록 저장소 및 협업 기반을 먼저 맞추는 것이다. 다음 단계에서는 브랜치 전략, mock API 구조, 이미지 생성 및 저장/조회 API 기준을 정리해야 한다.

### 4. UX/UI 작업

UX/UI 담당자는 서비스 방향에 맞는 디자인 레퍼런스를 조사했다.

디자이너들은 프로젝트 컨셉 이해도를 맞추는 작업을 진행했고, 메인 컬러와 화면 레이아웃 방향을 잡기 시작했다. 이후 생성 화면, 결과 화면, 라이브러리 화면을 중심으로 와이어프레임과 화면 구조를 구체화할 예정이다.

### 5. 2주차 Pass 기준 재확인

2주차 중간점검에서는 전체 기능을 넓게 보여주기보다, 이미지 생성부터 저장 및 조회까지의 핵심 흐름이 끊기지 않는 것을 우선 기준으로 둔다.

```text
프롬프트 입력
→ 캐릭터/배경 이미지 후보 생성
→ 결과 표시
→ 저장
→ 라이브러리 조회
```

영상 생성, 영상 상태 조회, 역프롬프트, 고도화된 검색/필터 기능은 2주차 Pass 이후의 목표로 분리한다.

## 결정 사항

- API 비용 기준을 확정하고 각 팀별 예산안을 제출했다.
- PM은 예산안과 기획 내용을 통합 문서에 반영했다.
- 손찬용 PM은 PRD 초안을 업로드했다.
- 이슬기 PM은 마일스톤 초안을 작성했다.
- 백엔드 담당자는 Git 세팅에 집중했다.
- UX/UI 담당자는 레퍼런스 조사, 컨셉 이해도 정렬, 메인 컬러, 레이아웃 방향 작업을 진행했다.
- 2주차 Pass 기준은 프롬프트 입력, 이미지 생성, 결과 표시, 저장, 라이브러리 조회 흐름으로 유지한다.
