## 논의 내용

### 1. 프로젝트 소개

OT 이후 팀원들과 만나 A2팀 프로젝트의 기본 방향을 소개했다.

현재 프로젝트는 생성형 AI 통합 엔진을 만드는 것으로, 이미지 생성, 영상 생성, 라이브러리 저장 및 조회 흐름을 중심으로 진행한다. 우선 2주차 중간점검에서는 핵심 기능이 끊기지 않고 동작하는 것을 목표로 하고, 이후 영상 생성과 고도화 기능으로 확장한다.

### 2. 파트별 고민사항 확인

각 파트 담당자에게 현재 고민사항이나 확인이 필요한 부분이 있는지 질문했다.

특히 프론트엔드와 백엔드 파트에는 배포 방식과 저장소 관리 방식을 함께 확인해야 한다고 공유했다.

### 3. API 비용 산정 일정

2026-07-01 수요일까지 팀 차원의 API 비용 산정안을 준비해야 한다.

비용 산정에는 이미지 생성 API, 영상 생성 API, 저장소, 배포 비용을 함께 고려한다. 영상 생성은 3x3 스토리보드 그리드 이미지 1장과 9개 씬 프롬프트를 기반으로 한 단일 영상 생성 구조를 기준으로 다시 산정한다.

### 4. 배포 방식 논의

프론트엔드와 백엔드 배포 방식을 논의했다.

초기 검토안으로 Railway와 Vercel을 비교했으나, 현재 팀 결정은 다음과 같다.

| 영역 | 결정 |
| --- | --- |
| 프론트엔드 | Vercel Hobby 플랜 |
| 백엔드 | AWS EC2, S3 |

프론트엔드는 Vercel Hobby 플랜으로 배포하고, 백엔드는 AWS EC2와 S3를 사용하는 방향으로 정리했다.

### 5. GitHub 관리 방식 논의

GitHub는 조직을 만들어 관리할지, 다른 방식으로 관리할 수 있을지 확인이 필요하다.

팀 저장소 관리 방식은 아직 최종 확정 전이므로, GitHub Organization 생성 방식과 대안 방식을 비교해 결정한다.

### 6. UX/UI 파트 공유 및 미팅 일정

UX/UI 파트 담당자에게 레퍼런스를 받았다고 전달했다.

또한 2026-07-01 수요일에 다른 팀 UX/UI 팀원들과 미팅이 있을 예정이라고 공유했다. 해당 미팅에서는 디자인 시스템과 컬러칩을 정할 예정이며, UX/UI 담당자에게 참석 가능한 시간을 확인했다.

## 결정 사항

- 프론트엔드 배포는 Vercel Hobby 플랜을 사용한다.
- 백엔드 배포는 AWS EC2와 S3를 사용한다.
- API 비용 산정안은 2026-07-01 수요일까지 준비한다.
- UX/UI 파트는 2026-07-01 수요일 디자인 시스템 및 컬러칩 관련 미팅 가능 시간을 확인한다.
- GitHub 저장소 관리 방식은 Organization 생성 여부와 대안을 비교한 뒤 확정한다.
