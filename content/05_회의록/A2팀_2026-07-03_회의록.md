## 논의 내용

### 1. PM·UX/UI IA 협의

2026-07-03에는 PM과 UX/UI가 화면 구조를 맞추기 위한 IA 협의를 진행했다.

협의는 `A2팀_IA_초안_v0.1_20260703.md`를 기준으로 진행했으며, MVP의 중심 흐름은 이미지 생성, 영상 생성, 라이브러리 저장·조회로 정리했다.

초기 화면 구조는 홈, 이미지 생성, 영상 생성, 라이브러리, 역프롬프트, 내 프로필·설정으로 나눠 1차 정리했다. 방향별 레퍼런스는 별도 화면으로 분리하지 않고 이미지 생성 화면 안의 탭 또는 섹션으로 포함하는 방향으로 잡았다.

생성 결과물은 `게시글`이 아니라 `작업물` 또는 `라이브러리 항목`으로 표현하기로 했다. 커뮤니티, 게시글, 댓글, 추천 기능은 MVP 핵심 범위에서 제외하고 추후 논의 항목으로 분리한다.

### 2. IA 산출물 정리

PM은 협의 결과를 바탕으로 IA 1차 정리본과 공유용 자료를 작성했다.

IA 1차 정리본은 `A2_IA_1차_확정본_20260703.xlsx`로 정리했다. 해당 정리본에는 로그인 전/후 화면 구조, 이미지 생성, 영상 생성, 라이브러리, 역프롬프트, 추가 안건이 포함되어 있다.

팀 공유용 시각 자료는 `A2팀_IA_공유용_20260703.html`, `A2팀_IA_공유용_20260703.svg`, `A2팀_IA_공유용_20260703.png`로 정리했다.

다만 좌측 사이드바 고정 내비게이션 포함 여부, 라이브러리에서 좋아요·찜하기를 어떻게 다룰지, 사전 프롬프트 가이드와 공유 링크 기능은 후속 안건으로 남겼다.

### 3. Backend API·DB 명세서 정리

Backend는 API 명세서와 DB 명세서 기준을 공유했다. 해당 내용은 `content/03_엔지니어링/2026-07-03_A2팀_API_DB_명세서.md`로 정리했다.

현재 API는 이미지 생성, 역프롬프트 생성, 영상 생성, 영상 진행상태 조회, 파일 업로드·삭제·다운로드·목록 조회·상세 조회 기준으로 정리되어 있다.

DB 기준은 미디어 자산을 저장하는 `MediaFile`과 생성 작업 상태를 관리하는 `GenerateJob`으로 정리했다. 영상 생성 상태는 `PENDING`, `RUNNING`, `SUCCESS`, `FAILED`를 기준으로 표시한다.

프론트엔드 연동 전에는 요청/응답 스키마, 진행상태 조회 키, 파일 업로드 필드명, 삭제 정책, 다운로드 응답 방식, 역프롬프트 저장 범위, 실제 테이블명 표기, 인증/권한 여부를 추가로 확인해야 한다.

### 4. 1주차 중간보고서 반영

1주차 중간보고서에는 2026-07-03 업무 종료 후 확인된 IA 협의 결과와 Backend API·DB 명세서 내용을 반영했다.

보고서에서는 실제 산출물로 확인된 IA 정리본, IA 공유용 자료, API·DB 명세서를 중심으로 기록했다. 2026-07-03 기준 UX/UI 와이어프레임 산출물은 별도 파일로 확인되지 않아 `확인 필요` 상태로 남겼다.
