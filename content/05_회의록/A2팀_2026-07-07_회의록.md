### 2. AWS 운영 방식 확정

AWS는 결제가 아직 미확정이지만, 그와 별개로 세팅 진행 방식은 확정됐다.

`2026-07-06_A2팀_배포환경_결제_운영안.md`의 권장 방식(회사 소유 계정에서 Billing/IAM 분리)과 다르게, **우선 개인 계정으로 EC2/S3를 세팅**하고, 이후 회사 결제가 확정되면 **AWS Organizations로 개인 계정을 멤버 계정으로 편입**하는 방식으로 진행한다. EC2/S3 요금산정과 AWS 상세 예산도 이 개인 계정 기준으로 먼저 진행한다. 초기 세팅 단계의 임시 조치이며, 회사 계정 소유·권한 분리 기준은 Organization 편입 시점에 다시 적용한다.

### 3. Vercel Team 생성 방식 문의 대응

FE로부터 Vercel Team 생성 관련 문의가 들어왔다. Vercel은 Hobby 플랜으로는 Team을 생성할 수 없어, **Pro 플랜 14일 무료 트라이얼**로 우선 시작하고 **Billing 정보는 개인 카드로 임시 등록**한 뒤, 회사 결제 담당자가 확정되면 그때 카드를 교체하는 방식으로 진행해도 되는지 문의였다.

이슬기 PM 확인 기준 늦어도 2026-07-13(월)까지는 회사 결제 담당자가 확정될 예정이라, 14일 트라이얼 기간 안에 카드 교체가 가능하다고 판단해 **진행하도록 회신**했다. 확정이 지연되면 PM 쪽에서 회사에 재확인하기로 했다.

> **[2026-07-07 정정, Vercel 공식 문서 확인]** Vercel Pro 트라이얼은 **시작 시점에 결제수단 등록이 필수가 아니다** — 트라이얼 종료(14일) 시점에만 결제수단이 필요하며, 등록이 없으면 자동으로 Hobby로 전환되고 등록돼 있으면 그때부터 과금이 시작된다. 즉 07-13까지 회사 결제 담당자가 확정되면 **개인 카드를 아예 쓰지 않고 트라이얼을 시작했다가, 종료 시점에 바로 회사 결제수단을 등록**하는 방식도 가능하다. FE에게 이 부분을 추가로 안내한다.

**[2026-07-07 실행 확인]** 위 정정 안내와 별개로, FE 김은재가 이미 아래 계획대로 실행을 완료했다 (이슬기 PM·팀 채널 공유 기준).
1. 김은재 개인 카드로 임시 등록 후 Pro Trial(14일 무료) 시작, Team 생성 완료
2. 이후 결제 담당자를 Team에 초대(Billing 권한 부여) 예정
3. 14일 안에 결제 담당자 카드로 결제 정보 수정 후 결제 진행 예정

즉 실제로는 개인 카드를 쓰지 않는 경로 대신, FE가 이미 임시 등록 경로로 진행했다. Team 생성 자체는 완료됐으므로 이 항목은 해소됐고, 남은 건 14일 내 결제 담당자 초대와 카드 교체뿐이다.

### 4. UX/UI 와이어프레임 진행 상황

UX/UI가 이미지 생성·결과·라이브러리 화면 와이어프레임을 약 1시간 내로 완성할 예정이라고 공유했다. PRD·통합 핸드오프에 "확인 필요"로 남아있던 와이어프레임 산출물 항목이 오늘 중 해소될 전망이다.

### 5. 영상 상태값에서 PENDING 삭제

영상 생성 작업의 상태값에서 `PENDING`을 모두 삭제하기로 했다. 기존 `PENDING`, `RUNNING`, `SUCCESS`, `FAILED` 4개 상태값이 `RUNNING`, `SUCCESS`, `FAILED` 3개로 줄어든다. 사용자 화면 표시 기준도 `queued`가 빠지고 `processing`, `completed`, `failed`로 정리된다.

이 변경은 API/DB 명세서, PRD, 통합 핸드오프, 개발자 전달사항, 일일 작업 체크리스트에 모두 반영했다. `GenerateJob.status`의 기본값도 `PENDING`에서 `RUNNING`으로 변경했는데, 영상 생성 요청이 들어오는 즉시 `RUNNING`으로 기록한다는 의미다. 다만 실제로 큐 대기가 발생하는 상황(동시 요청 과다 등)을 어떻게 표시할지는 BE 확인이 필요해 개발자 전달사항의 확인 질문에 추가했다.

### 6. fal.ai Hard Limit 기능 실제 확인 결과

마일스톤부터 비용 문서까지 여러 곳에 반복되던 "fal.ai Hard Limit $100/$120 설정"을 fal.ai 공식 문서로 직접 확인했다.

- [Pricing - fal](https://fal.ai/docs/documentation/model-apis/pricing): 선불 크레딧 기반 pay-as-you-go. 실패 호출·큐 대기 시간은 무과금.
- [Managing Teams - fal](https://fal.ai/docs/documentation/organizations/managing-teams): "Org admins with billing access can... set spending alerts for any team"라고만 되어 있고, 특정 금액 도달 시 호출을 자동 차단하는 지출 상한(Hard Limit) 기능은 어디에도 없다.

**결론**: fal.ai에는 자동 차단형 Hard Limit이 없다. 지금까지 여러 문서에 쓰인 "Hard Limit $100/$120 설정"은 실제로 존재하지 않는 기능을 전제로 한 계획이었다. 실질적인 대체 방안으로 아래 두 가지를 채택했다.
1. 팀 Billing에서 지출 알림(spending alert) 설정 — 금액 도달 시 알림만 발생
2. 크레딧을 한 번에 대량 충전하지 않고 **$50~100 단위로 소액 분할 충전** — 크레딧 소진 시 호출이 자동 실패하므로 사실상의 상한 역할

이 내용을 API 결제 운영안, API 테스트 시나리오 및 비용, 통합 핸드오프, 일일 작업 체크리스트, 마일스톤(scy), 그리고 framewave-docs의 최종 마일스톤·최종 API 비용 산정에 모두 반영했다.

## 결정 사항

- Claude Team 좌석 초대, Figma 결제는 완료 확인. Vercel·fal.ai·AWS 3개 항목은 결제 미확정 상태로 정정한다.
- AWS는 결제 확정 여부와 별개로 개인 계정으로 우선 세팅하고, 추후 회사 결제 확정 시 AWS Organizations 멤버 계정으로 편입한다.
- EC2/S3 요금산정 및 AWS 상세 예산도 개인 계정 기준으로 먼저 진행한다.
- 배포환경 결제 운영안 문서의 AWS 후속조치를 이 진행 순서에 맞게 갱신한다.
- Vercel Team은 Pro 14일 무료 트라이얼로 우선 시작하고, Billing은 개인 카드로 임시 등록한다 — FE가 이미 이 경로로 Team 생성까지 완료. 회사 결제 담당자 확정 시(늦어도 2026-07-13 목표) 초대 및 카드 교체.
- 영상 작업 상태값에서 PENDING을 삭제하고 RUNNING, SUCCESS, FAILED 3개로 정리한다. 관련 문서(API/DB 명세서, PRD, 통합 핸드오프, 개발자 전달사항, 일일 체크리스트)를 모두 갱신한다.
- fal.ai에는 자동 차단형 Hard Limit이 없음을 확인. 비용 통제는 지출 알림 설정 + $50~100 단위 소액 분할 충전으로 전환하고, 관련 문서를 전부 갱신한다.
