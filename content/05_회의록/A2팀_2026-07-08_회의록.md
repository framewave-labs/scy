## 논의 내용

### 1. 이미지 생성 메뉴 2단계 분리 결정 (이슬기 PM 전달)

한 번의 프롬프트로 최종 키이미지 세트(정면·측면·후면·표정·장비 클로즈업 등)를 바로 출력하는 것은 어렵다고 BE·디자이너와 함께 판단, 메뉴를 2단계로 분리하기로 했다.

- **메뉴 A(초기 레퍼런스)**: 간단한 프롬프트 입력 → 배경 이미지 1장 + 캐릭터 여러 각도(정면/측면/후면) 이미지 출력
- **메뉴 B(키이미지/디테일 생성)**: 메뉴 A 다음 단계. 디테일한 프롬프트(스타일·의상·소품·색상 코드 등) 입력 → 턴어라운드+표정+소품 클로즈업을 담은 완성형 캐릭터 디자인 레퍼런스 시트 출력

이 구조는 `2026-07-08_IA_v0.4.md`(이슬기 PM, lsg 저장소)에 반영됐고, 07-07 PD가 확정한 키이미지 정의(캐릭터 1명의 고정 레퍼런스 시트)와 정확히 부합한다.

### 2. 키이미지 프롬프트 예시 및 사이드바 속성 선택 UI 검토

메뉴 B 디테일 생성 단계에서 쓸 프롬프트는 PART 1(서술형 설명) + PART 2(SUBJECT/CLOTHING & STYLING/ACCESSORIES/SCENE/BACKGROUND/LIGHTING/TECHNICAL/COMPOSITION/STYLE/ASPECT RATIO/CONSTRAINTS JSON 구조) 형식을 그대로 쓴다.

사용자가 처음부터 상세 프롬프트를 직접 입력하지 않는 경우를 대비해, 사이드바에 헤어·의상·색상 등 속성을 선택할 수 있는 UI를 넣는 방향도 함께 검토 중이며, 이후 이슬기 PM이 **파트 1~5(캐릭터 기본/아트 스타일·세계관/헤어/얼굴/의상) 전체 선택형** 구조로 확정해 `2026-07-08_속성값_확정안.md`(lsg)로 정리했다.

### 3. 디자이너(이예린) — 영상 생성 유료 계정 문의

이미지는 레퍼런스 사이트 무료 크레딧으로 어느 정도 테스트가 가능했으나, 영상은 무료 생성이 어려워 KX 측 유료 계정 정보 공유를 문의함. **계정 정보 공유는 어렵다고 안내**했다.

### 4. BE(김유신) — 내부 테스트 일정 및 비용 관리

- fal.ai API 연동 상태에서 내부 테스트 가능 여부 문의 → **늦어도 이번 주, 빠르면 오늘 중 테스트 가능**하도록 준비 가능하다는 답변
- BE 자체 임의 테스트 관련: 테스트까지 감안해 예산을 여유 있게 책정했다고 안내했으나, fal.ai가 Hard Limit 설정이 불가능해 사용량을 계속 확인해야 하는 번거로움이 있어 **테스트는 적당히 진행**해달라고 요청. 비용 최소화 예시로 Seedance 2.0 내부 테스트 시 15초가 아닌 3초 등 짧은 길이로 진행하는 방식을 제안함

### 5. FE(김은재) — Vercel PD 계정 초대

PD님 계정을 Vercel에 초대해 둔 상태이나, 아직 PD님이 수락하지 않으신 상태. **PD님 확인 후 초대 수락 여부 확인 필요** → 수락 후 결제 정보 등록 필요.

> **PM 액션 아이템(이관됨)**: PD님께 Vercel 초대 확인 및 수락 요청 → 수락 후 결제 정보 등록 안내

### 6. B1팀 — 사내 로그인/유저 DB 통합 문의

B1팀 PM 고다희님이 로켓단 DEV 채널에 "사내 로그인 및 유저 데이터(user 테이블) 통일 필요 여부"를 전체 팀에 문의함. B1팀 방향은 자유 회원가입 금지, 화이트리스트 인원만 구글 소셜 로그인(OAuth).

A2팀 BE(김유신·송영범) 답변: 현재는 JWT 기반 이메일 로그인만 구현, 소셜 로그인은 메인 기능 개발 이후 구현 예정. user 테이블 컬럼은 `id, email, password, name, role, createdAt/updatedAt`. 회원가입 정책은 아직 미정.

**PM 간 논의 결과**: 로그인 통합은 현 단계에서 고려할 사항이 아니라고 판단, 다만 추후 재논의 가능성이 있어 트래킹은 계속한다. (→ 이슬기 PM 후속안건 4번으로 "보류" 처리됨)

### 7. 프롬프트 가이드 초안(06-24) — PD 키이미지 정의 기준 갱신

`2026-06-24_프롬프트_가이드_초안.md`가 PD의 07-07 키이미지 정의 확정 이전 내용(캐릭터+배경 통합 장면)을 그대로 갖고 있어 갱신이 필요했다. `2026-07-08_A2팀_프롬프트_가이드_초안_갱신_제안.md`로 갱신안을 정리한 뒤 그대로 반영해 v1.1로 갱신 완료 — 캐릭터 시트=키이미지로 절 병합, PD 표준형 Part1/Part2 템플릿 추가, 배경 시트 360도 불필요 문구 추가.

### 8. 화면별 옵션값 — 손찬용 안 작성 및 이슬기 PM 안 통합

이슬기 PM의 `2026-07-08_속성값_확정안.md`(본인이 "확정안이라는 제목은 무시해달라"고 명시 — 제안 단계)와 별개로, 손찬용 PM 명의로 Higgsfield·Magnific 등 기존 서비스 사례와 GPT Image 2·fal.ai Seedance 2.0 실제 API 스펙을 조사해 `2026-07-08_A2팀_화면별_옵션값_손찬용안.md`를 작성했다. 두 안을 `2026-07-08_A2팀_옵션값_통합_제안.md`에서 항목별로 대조한 결과:

- 대부분 항목(이미지 비율·품질·수량, 영상 모델 구성, Bitrate 방식) **일치**
- 영상 해상도: 손찬용 안이 fal.ai 문서로 직접 확인해 "BE 확인 필요" 상태를 해소(720p/1080p/4K 시연 전용)
- **미해결**: 영상 비율에서 Seedance의 4:3·3:4 포함 여부(이슬기 PM 안에는 있으나 fal.ai 공식 문서에서 미확인 — BE 크로스체크 필요), 메뉴 B 속성 선택 파트1~5 구조에 대한 손찬용 검토 의견(현재 이슬기 PM 안 잠정 채택)
- 홈 히어로 메시지 항목은 IA v0.4의 "히어로 섹션 폐지" 결정에 따라 옵션값 문서에서 삭제하기로 두 안이 일치

### 9. 영상 상태값 용어 재확인

lsg IA v0.4의 상태 흐름도가 `processing/completed`를 쓰는 걸 보고 scy의 `RUNNING/SUCCESS` 표기와 불일치하는지 검토했으나, 확인 결과 07-07에 이미 "DB/백엔드: RUNNING/SUCCESS/FAILED, 사용자 화면 표시: processing/completed/failed"로 의도적으로 이중 구조를 결정해뒀던 것으로 확인 — **불일치 아님, 기존 구조 유지**. 다만 lsg IA v0.4의 흐름도에 07-07에 없애기로 한 `queued` 표기가 남아있는 점은 이슬기 PM에게 구두로 전달 필요.

### 10. 데일리 스크럼 자동화 — 스캔 범위 수정

`scripts/build-scrum.mjs`가 최신 회의록 1개만 읽어 오래된 미완료 액션 아이템이 스캔에서 누락되는 문제가 있어, 2026-07-06 이후 회의록 전체에서 상태가 "완료"로 시작하지 않는 항목만 모으도록 수정했다. 다만 07-08 07:00 KST 크론 자동 커밋이 발생하지 않아 GitHub Actions 점검이 별도로 필요하다.

- 메뉴 B 상세 프롬프트 입력은 속성 선택 사이드바(파트 1~5, 필수/선택 구분 없음) 방식으로 확정한다
