# A2팀 회의록 - 2026-07-09 작업 정리

- 참석자: A2팀 PM(손찬용·이슬기), Backend(김유신-영상, 송영범-이미지), Frontend(김은재), UX/UI(이예린)

## 논의 내용

### 1. PM — 화면별 옵션값 PM간 검토 후 BE·디자이너 통합회의 예정

옵션값 통합 제안(손찬용안+이슬기 PM안)을 오늘 PM끼리 먼저 확인한다. BE 의견이 필요한 부분이 있어, 디자이너에게 넘기기 전에 BE와 함께 논의하는 통합회의를 가질 예정이다.

### 2. PM — 프롬프트 가이드 통합 및 공유

마일스톤상 오늘 프롬프트 가이드 공유 시간이 있다. 이미지 생성이 메뉴 A·B로 나뉘어 있어 그 구조에 맞춰 프롬프트 가이드 통합 작업을 진행 중이며, 통합되는 대로 공유할 예정이다.

### 3. PM — IA 수정

메뉴 A·B 분리 등을 반영하기 위해 IA 수정을 소폭 진행할 예정이다.

### 4. BE(영상, 김유신) — 영상 생성 API 연동 테스트 현황

- 어제 영상 생성 API 연결 후 2회 테스트 진행
- 3x3 그리드 이미지에서 시작하는 부분에 이슈가 있었으나 2회 모두 사용은 함
- 해상도는 별도 입력 없이 Kling 중 낮은 해상도로 확정해 진행하기로 함
- **진행률 표시 이슈**: 퍼센트 진행률을 보여주려 했으나, 상태 조회 API는 진행중/완료/성공 정도의 상태값만 제공. 공통 API 응답에 로그가 포함된다고 해서 로그 내용 기반 진행도 표시를 시도했으나 실제로는 로그 데이터를 받아오지 못하는 중. 모델별로 로그 제공 여부가 다른지 확인 중이며, 로그를 못 받으면 진행도 표시를 어떻게 할지 별도 고민 필요

### 5. BE(이미지, 송영범) — 이미지 생성 개발 현황

- 현재 업로드 기능 개발 중
- 마일스톤상 조회·라이브러리 저장 기능도 필요해 추후 진행 예정
- 이미지 파트도 영상과 동일하게 진행률 없이 상태만 조회 가능 — 진행 상태만 표시하는 방향으로 갈 것 같으나 구체적 표시 방법은 고민 중

### 6. FE(김은재) — 사이드바·홈 화면 작업, 라이브러리 예정

- 이미지 생성 사이드바 공통 레이아웃 및 홈 화면 작업 중
- 마일스톤상 오늘 라이브러리 작업 진행 예정
- 메뉴 A·B 분리로 인해 캐릭터/배경 생성 화면 외에 캐릭터시트 생성 화면이 하나 추가되는 구조 — 이슬기 PM이 화면 추가로 인한 FE 부담 가능성을 공유함
- FE 확인 요청: "전달받은 시안에서 추가되는 방향으로 생각하면 될까?" → **PM 확인 완료: 캐릭터시트 화면이 추가되는 방향 맞음**

### 7. UX/UI(이예린) — 홈 화면 디자인 진행

- 피그마에서 구글 M3를 활용해 AI로 디자인 초안 생성, 이를 기반으로 디테일 아이콘·생성 작업 진행 중
- 이번 주 목표: 홈 화면 디자인 완료
- 캐릭터시트 화면도 작업 필요할 것으로 예상 — 이미지 생성(메뉴 A·B) 구조가 잡히면 영상·캐릭터시트 디자인도 비슷한 흐름으로 잡힐 것으로 봄
- 목표: 중간보고 전까지 디자인 완료

## 결정 사항

- 영상 생성 해상도는 별도 입력 없이 Kling 중 낮은 해상도로 고정해 진행한다
- 프롬프트 속성값은 BE 의견을 반영하기 위해 디자이너 전달 전 BE·PM 통합회의를 먼저 가진다
- FE 시안은 캐릭터시트 화면이 추가되는 방향으로 진행한다 (기존 캐릭터/배경 생성 화면 + 캐릭터시트 생성 화면 신규 추가)

## 확인 필요 사항

- 영상·이미지 생성 진행률(퍼센트) 표시 방법 — 로그 데이터 수신 불가 문제 해결 또는 대안(상태만 표시) 결정 필요
- 모델(Kling/Seedance 등)별로 상태 조회 시 로그 데이터 제공 여부 차이 (BE 확인 중)
