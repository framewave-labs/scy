# A2팀 회의록 - 2026-07-10 작업 정리

## 기본 정보

- 일시: 2026-07-10
- 참석자: A2팀 PM(손찬용·이슬기), Backend(김유신-영상, 송영범-이미지), Frontend(김은재), UX/UI(이예린)
- 장소 또는 채널: 데일리 스크럼
- 작성자: 손찬용 PM

## 안건

- BE 영상 생성 테스트 결과(Kling 시작·끝프레임/3x3 이미지 입력)
- PD 제작 3x3 이미지 기반 영상의 실제 생성 방식 확인 필요
- 프롬프트 가이드 BE 전달 계획
- BE 이미지 생성 개발 현황
- FE 이미지 생성·라이브러리 작업 현황
- UX/UI 이미지 생성 디자인 착수 예정
- 화면별 옵션값(속성값) 통합 회의 일정

## 논의 내용

### 1. BE(영상) — Kling 기준 시작프레임·끝프레임·3x3 이미지 입력 테스트 결과 양호

시작프레임/끝프레임/3x3 이미지 한 장을 입력으로 넣었을 때 Kling 기준으로는 결과물이 잘 나오는 것을 확인했다.

### 2. BE(영상) — Seedance 다중 이미지 입력 모델 발견, PD 제작 영상 방식 추정

Seedance 쪽에는 이미지를 최대 9장까지 넣어서 생성하는 모델이 별도로 있는 것을 확인했다. PD가 영상을 제작할 때 이 모델을 사용했고, 이를 3x3 스토리보드 이미지 한 장으로 표현(압축)해 전달한 것으로 추정된다 — 즉 실제 파이프라인이 "3x3 그리드 이미지 1장 + 텍스트 프롬프트"가 아니라 "그리드에서 분리한 최대 9장의 개별 이미지를 다중 이미지 입력 모델에 넣는 방식"일 가능성이 있다는 뜻이다.

**확인 필요**: 이 추정이 맞는지 PD가 실제로 제작한 3x3 이미지 기반 영상 원본을 받아 개발단에 문의하기로 했다. 이 결과에 따라 현재 작업 중인 영상 프롬프트 가이드(9-Shot 텍스트 프롬프트 구조)의 전제 자체가 바뀔 수 있다.

### 3. PM — 영상 생성 비용 부담, 프롬프트 가이드 명확화 후 BE 전달

영상 생성은 비용이 많이 들기 때문에 시행착오를 줄이기 위해 프롬프트 가이드를 명확하게 만들어서 백엔드 개발단에 전달하기로 했다.

### 4. BE(이미지, 송영범) — 업로드 완성, 다운로드 거의 완성, 라이브러리·캐릭터시트 예정

- 이미지 업로드 기능 개발 완성
- 다운로드 기능 거의 완성 단계
- 다음 작업: 라이브러리 저장·조회 기능, 캐릭터시트 기능 개발 예정

### 5. FE(김은재) — 라이브러리 작업 중, 이미지 생성 디테일·로딩 적용 예정

- 라이브러리 작업 진행 중
- 이미지 생성 화면 디테일 작업 중
- 오늘 중 이미지 생성 로딩 적용까지 진행 예정

### 6. UX/UI(이예린) — 이미지 생성 디자인 작업 착수 예정

이미지 생성 관련 디자인 작업을 진행할 예정이라고 공유했다.

### 7. PM — 옵션값(속성값) 통합 회의는 자료 취합되는 대로

PM이 언급했던 속성값(옵션값) 관련 내용은 다같이 모여 검토하는 게 좋을 것 같아, 자료가 취합되는 대로 별도 회의를 진행하기로 했다.

### 8. "프롬프트 향상" 토글 동작 및 히스토리 저장 정책

이미지 생성 화면의 "프롬프트 향상" 토글(구 "AI 프롬프트 교정" — 옵션값 확정 문서 2.1절에서 개명 확정)의 동작을 정리했다.

**토글 동작**
- 토글 ON: 한글로 입력된 프롬프트를 Claude Sonnet이 GPT Image 2에 최적화된 영어 프롬프트로 변환해 출력한다
- 토글 OFF: 사용자가 입력한 프롬프트가 변환 없이 그대로 들어간다

**히스토리 저장 정책 (BE 질문에 대한 답)**

BE에서 "히스토리 1건당 이미지 1장을 묶어 기록할 때, 최종 프롬프트를 한국어로 저장할지 변환된 영어로 저장할지" 질문이 있었다. 맥락상 **실제 생성에 쓰인 최종 프롬프트를 저장**하는 방향으로 정리:

| 입력 언어 | 토글 | 히스토리에 저장되는 프롬프트 |
| --- | --- | --- |
| 한국어 | OFF | 입력한 한국어 그대로 |
| 영어 | OFF | 입력한 영어 그대로 |
| 한국어 | ON | 변환된 영어 |
| 영어 | ON | 변환된 영어 |

즉 토글이 활성화된 히스토리는 변환된 영어로, 활성화되지 않은 히스토리는 입력 원문 그대로 저장한다.

**미해결**: 토글을 켠 상태에서 빈 프롬프트를 입력(생성 시도)하는 경우의 처리 방식이 애매함 — 변환할 입력 자체가 없는 상황이라 별도 처리 방침 필요.

### 9. 영상 생성 진행률(퍼센트) 표시 — FE·디자이너 논의 예정

영상 생성 후 상태 조회 시 퍼센트 진행률의 정확한 값 파악이 어려울 것으로 보여(07-09 회의록의 로그 데이터 수신 불가 문제와 연결), 추후 FE·디자이너와 함께 표시 방식을 논의하기로 했다.

## 결정 사항

- 영상 생성 비용 문제로 프롬프트 가이드를 확정해 BE에 전달하는 것을 우선 진행한다
- PD 제작 3x3 이미지 기반 영상 원본을 받아 개발단에 실제 생성 방식(다중 이미지 입력 모델 사용 여부)을 문의한다
- 옵션값 통합 회의는 자료가 취합되는 대로 별도로 잡는다
- 히스토리에 저장하는 프롬프트는 실제 생성에 쓰인 최종 프롬프트로 한다 — 토글 ON이면 변환된 영어, OFF면 입력 원문 그대로 (8절 표 참고)

## 확인 필요 사항

- PD가 제공한 3x3 이미지 기반 영상이 실제로 Seedance의 다중 이미지 입력(최대 9장) 모델로 만들어진 것인지 개발단 확인 필요 — 확인 결과에 따라 영상 프롬프트 가이드의 입력 구조 전제(3x3 그리드 1장 + 텍스트 vs 분리된 개별 이미지 다중 입력)가 달라질 수 있음
- (기존 미해결, 07-09 회의록에서 이월) 영상·이미지 생성 진행률(퍼센트) 표시 방법 — 로그 데이터 수신 불가 문제, FE·디자이너와 표시 방식 논의 예정(9절)
- "프롬프트 향상" 토글 ON + 빈 프롬프트 입력 시 처리 방식 미정(8절)
