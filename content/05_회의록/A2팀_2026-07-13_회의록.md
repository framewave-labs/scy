## 논의 내용

### 1. PM — 이번주 작업 3건

이번주 PM 작업은 아래 세 가지로 진행한다.

1. **IA 문서 업데이트** — 정의된 프로필 페이지 내용을 IA에 추가한다.
2. **기능명세서 초안 작성** — 프론트엔드 파트는 GitHub issue를 보고 작성하고, 백엔드 파트는 노션 대시보드(엔지니어링/진행상황)를 보고 작성한다.
3. **유즈케이스 작성**

### 2. UX/UI — 모델별 지원 해상도·영상 길이 차이 처리 방식 결정 필요

모델별로 지원되는 해상도나 영상 지원 길이 등이 서로 달라서, 화면에서 이를 어떻게 처리할지 정해야 한다. 와이어프레임이 빨리 나와야 프론트엔드에 전달할 수 있기 때문에 우선순위가 높은 결정 사항이다.

### 3. UX/UI — MVP 수록 이미지·영상 소스 결정 필요

MVP에 들어갈 이미지나 영상을 AI 생성물로 넣어도 되는지, 아니면 저작권을 지켜 실제로 사용할 수 있는 이미지·영상을 넣어야 하는지 결정이 필요하다.

### 4. UX/UI — 온보딩 영상 제작 방식 고민

온보딩에 나올 영상을 어떻게 만들지 고민이 필요하다. Seedance처럼 "AI를 어떻게 활용해서 만드는지"를 보여주는 how-to 성격의 영상을 구상 중이다.

### 5. BE — AWS 진행 방식 확정 필요(배포 선행 조건)

AWS를 어떻게 진행할지 확정되어야 배포가 나올 수 있다. 배포(A007) 일정의 선행 조건이므로 확정을 서둘러야 한다.

### 6. BE — 영상 생성 진행률 퍼센트 추적 불가 확정, 대체 표시 방식 필요

영상 생성 후 상태 조회 시 퍼센트로 제작 상태를 확인할 수 있도록 하려고 했으나, 모델에서 오는 log 값이 비어서 오기 때문에 퍼센트 추적이 안 되는 것으로 확인됐다(07-09·07-10 회의록에서 이월된 안건의 확정 결과).

대체 표시 방식으로 아래 세 가지 중에서 고민이 필요하며, 결정 내용은 와이어프레임에 반영한다.

- 스피너 이미지로 표시
- 가짜 퍼센트(추정치) 표시
- 생성 소요 시간 노출

### 7. 결제·인프라 진행 현황

- **Vercel**: 결제 카드 변경이 완료된 것으로 공유됐다. 개인 카드 임시 등록(트라이얼) 상태에서 이월되던 카드 교체 건이 해소됐다.
- **fal.ai**: Hard Limit(자동 차단형 지출 상한) 기능이 없어 진행 방식이 고민인 상태. 당분간은 **최대한 보수적으로 테스트하는 기조로 진행**하고, 필요해지면 사용량 체크를 PD님께 요청드리는 방식을 검토하기로 했다.
- **AWS**: 아직 확인된 내용이 없다(5절 배포 선행 조건 건과 동일).

## 결정 사항

- PM 이번주 작업은 IA 업데이트(프로필 페이지 추가), 기능명세서 초안(FE는 GitHub issue 기준, BE는 노션 대시보드 기준), 유즈케이스 작성 3건으로 진행한다
- 영상 생성 진행률은 퍼센트 추적이 불가한 것으로 확정하고, 대체 표시 방식(스피너/가짜 퍼센트/생성 시간)을 정해 와이어프레임에 반영한다
- fal.ai는 Hard Limit 부재를 감안해 최대한 보수적으로 테스트하는 기조로 진행한다
- 영상 생성 화면에는 "프롬프트 향상" 토글을 두지 않는 것으로 확정 — 07.07 합의(이미지 생성에만 적용) 및 B서비스 프롬프트 무변환 투입 전제에 따름 (옵션값 확정 문서 v0.6 반영)
