# A2팀 회의록 - 2026-07-14 기능명세 확인필요 사항 협의

## 기본 정보

- 일시: 2026-07-14
- 참석자: 손찬용 PM, 이슬기 PM, 개발팀(영범님 포함)
- 작성자: 손찬용 PM
- 기준 문서: `2026-07-13_A2팀_기능명세서_초안_v0.4.xlsx`의 `확인필요` 시트
- 회의 목적: 기능명세서 확인필요 12개 항목의 결정·보류 상태와 개발 선행 조건을 정리하고, 이미지·영상 생성용 입력 자료의 준비 주체를 확정한다.

## 주요 논의 요약

- 영상 생성 입력은 **3×3 그리드 1장과 프롬프트 1개**를 함께 전달하는 구조로 확정했다.
- 영상 생성 진행 표시는 확인 후 개발팀에 전달을 완료했다.
- 비율 파라미터는 IA 기준으로 진행하며, BE 구현에는 문제가 없는 것으로 공유됐다.
- Seedance와 Kling은 모두 오디오 생성을 기본 `ON`으로 적용한다.
- 프롬프트 향상 토글을 `ON`으로 설정해도 변환된 영어 프롬프트는 사용자에게 노출하지 않는다.
- 라이브러리 관련 항목과 자동 저장 실패 처리는 와이어프레임 및 AWS 세팅 이후 논의한다.
- AWS S3가 아직 세팅되지 않아 현재는 로컬 저장 방식으로 개발 중이다. 향후 저장 코드 변경이 예상되므로 삭제 기능은 지금 구현하지 않고 AWS·S3 세팅 후 진행한다.
- 공유 및 삭제·탈퇴의 세부 구현 방식은 BE가 구현 가능한 기준으로 우선 판단하되, 삭제 기능의 실제 착수는 AWS·S3 세팅 이후로 한다.
- 웹훅과 폴링 관련 구조는 현재 로컬 서버 기준으로 개발 중이며, AWS 세팅 후 확정한다.
- FE 미생성 이슈 영역과 BE 범위 밖 기능은 아직 문서화되지 않은 부분이 있어 개발팀이 추후 확인 후 수정한다.
- 이미지·영상 생성에 필요한 이미지와 프롬프트는 PM단에서 자체 제작해 개발팀에 전달한다.

## AWS·S3 및 삭제 기능 논의

- 현재 라이브러리 저장 기능은 AWS S3가 세팅되지 않아 로컬 저장 방식으로 진행하고 있다.
- AWS·S3 세팅 후에는 저장 위치와 저장·조회 코드가 변경될 가능성이 높다.
- 현시점에 삭제 기능을 구현하면 AWS 전환 시 코드를 다시 수정해야 하므로, 영범님은 삭제 기능을 아직 진행하지 않았다.
- 작업물 삭제, 자동 저장 실패 처리, 라이브러리 필터·정렬, 운영 환경의 영상 상태 처리 방식은 AWS·S3 세팅 이후 저장 구조와 함께 재논의한다.
- 계정 탈퇴와 작업물 삭제의 실제 삭제 범위(DB·S3·관련 메타데이터)는 AWS 저장 구조가 확정된 뒤 BE 구현 기준으로 정리한다.

## PM 준비 및 전달 사항

- 이미지 생성에 필요한 테스트·시연용 이미지와 프롬프트는 PM단에서 자체 제작한다.
- 영상 생성에 필요한 3×3 그리드 이미지 1장과 대응 프롬프트도 PM단에서 자체 제작한다.
- 제작한 입력 자료는 개발팀이 이미지·영상 생성 연동과 테스트에 사용할 수 있도록 전달한다.
- PM단은 전달 자료와 기능명세서의 입력 조건이 일치하도록 관리한다.

## 결정 사항

1. 영상 생성 입력은 3×3 그리드 이미지 1장과 프롬프트 1개로 확정한다.
2. 영상 진행 표시는 퍼센트 대신 스피너 방식으로 진행하며 전달을 완료했다.
3. 비율 파라미터는 최종 IA 기준으로 적용한다.
4. Seedance와 Kling은 모두 오디오 생성 기본값을 `ON`으로 적용한다.
5. 프롬프트 향상 결과인 영어 프롬프트는 사용자에게 노출하지 않는다.
6. 공유 및 삭제·탈퇴의 세부 구현 방식은 BE 판단을 따르되, 삭제 기능의 실제 구현은 AWS·S3 세팅 이후 진행한다.
7. 이미지·영상 생성에 필요한 이미지와 프롬프트는 PM단에서 제작해 개발팀에 전달한다.
