## 📌 공통 운영 규칙

- **API 키:** 이미지(Gemini)는 운영팀이 키 배포. 영상·오디오는 게이트웨이(fal.ai, ElevenLabs 등) 키를 **팀별 사용량 상한(Hard limit)** 과 함께 지급.
- **결과물:** 모든 팀은 4주차에 **배포 데모 URL + GitHub Public Repo + 3분 시연 영상 + README** 제출.
- **MVP 원칙:** 완성품이 아닌 최소 기능 제품. 핵심 기능 완주가 우선, 추가 기능은 라인별 *도전 과제*로.
- **비용 모니터링:** 영상·이미지 API는 비용이 빠르게 누적되므로 상시 모니터링 및 키 발급 시 월 한도를 반드시 설정.

## 📽️ 프로젝트 소개

### 프로젝트 A: 생성형 AI 통합 엔진 (이미지·영상 생성 + 역프롬프트)

## 1. 프로젝트 개요

### 1.1 서비스 소개

방송 제작자가 여러 AI 사이트를 돌아다니지 않고 한 화면에서 이미지·영상을 생성하고 보관하는 서비스. 프롬프트를 입력하면 이미지·영상을 만들어 라이브러리에 저장하고, 반대로 마음에 드는 이미지를 넣으면 그 이미지를 분석해 프롬프트를 역으로 추출해 다시 변형 생성까지 돌립니다. 한마디로 AI 그림·영상 자판기 + 역추적기.

### 1.2 학습 목표

| 목표 | 상세 내용 |
| --- | --- |
| 멀티 AI API 연동 | 이미지(Gemini)·영상(fal.ai) API를 한 앱에 통합 |
| 비동기 작업 처리 | 영상 생성처럼 오래 걸리는 작업의 polling/상태관리 구현 |
| 미디어 라이브러리 | 생성물 저장·조회·관리 시스템 구축 |
| 멀티모달 활용 | 이미지→프롬프트(역프롬프트) 비전 모델 활용 |

### 1.3 기술 스택

| 구분 | 기술 | 비고 |
| --- | --- | --- |
| 필수 | Next.js 14+ (App Router) | 프레임워크 |
| 필수 | Gemini API (Nano Banana / Pro) | 이미지 생성·역프롬프트 |
| 필수 | fal.ai (Kling 3.0 / Seedance 2.0) | 영상 생성 게이트웨이 |
| 필수 | Tailwind CSS | 스타일링 |
| 권장 | TypeScript | 타입 안정성 |
| 권장 | Supabase | 인증 + 스토리지 + DB(라이브러리) |
| 배포 | Vercel | 무료 호스팅 |
| 도구 | Git, Figma, VS Code | 개발 도구 |

### 1.4 사용 AI 모델

## 6. 트러블슈팅 가이드

**Q1. 영상 생성이 타임아웃돼요** — 영상은 수십 초~분이 걸려 동기 호출은 실패. fal.ai의 큐(submit→status polling) 방식을 쓰고, 클라이언트는 requestId로 상태를 주기적으로 조회.
**Q2. Gemini 이미지 응답에서 이미지를 못 꺼내요** — 응답의 candidates[].content.parts 중 inlineData(base64) 파트를 찾아 디코딩. 텍스트 파트와 섞여 오므로 타입으로 필터링.
**Q3. 비용이 너무 빨리 쌓여요** — 영상이 주범. 테스트는 짧은 길이(5초)·저해상도로, 최종 데모만 고품질로.

```tsx
}
```

```tsx
import jsPDF from 'jspdf';
import html2canvas from 'html2canvas';

export async function exportToPDF(el: HTMLElement, filename: string) {
  const canvas = await html2canvas(el, { scale: 2 });
  const img = canvas.toDataURL('image/png');
  const pdf = new jsPDF({ orientation: 'landscape', unit: 'px',
    format: [canvas.width, canvas.height] });
  pdf.addImage(img, 'PNG', 0, 0, canvas.width, canvas.height);
  pdf.save(`${filename}.pdf`);
}
```

## 6. 트러블슈팅 가이드

**Q1. SRT 타임코드 형식이 안 맞아 편집툴이 인식 못 해요** — SRT는 `00:00:01,000 --> 00:00:04,000` 형식(쉼표·자릿수 엄격). 직접 만들지 말고 srt / pysubs2 라이브러리로 생성.
**Q2. 자막 싱크가 어긋나요** — Whisper의 segment 단위보다 단어 단위 타임스탬프가 더 정확. 정밀도가 중요하면 AssemblyAI의 word-level timestamp 사용.
**Q3. ASS 스타일이 적용 안 돼요** — ASS는 [V4+ Styles] 헤더의 스타일 정의와 [Events]의 Style 이름이 정확히 일치해야 함. pysubs2로 스타일 객체를 만들면 안전.
**Q4. 큰 오디오 파일이 업로드/변환에서 실패해요** — Whisper API는 파일 크기 제한이 있음. 긴 파일은 ffmpeg로 분할하거나 압축(mp3 16kHz)해 전송.

## 8. 코드 레퍼런스

**[코드 C-1] 텍스트→SFX (ElevenLabs)**

```tsx
export async function POST(req: Request) {
  const { text, duration } = await req.json();
  const res = await fetch('https://api.elevenlabs.io/v1/sound-generation', {
    method: 'POST',
    headers: { 'xi-api-key': process.env.ELEVENLABS_API_KEY!, 'Content-Type': 'application/json' },
    body: JSON.stringify({ text, duration_seconds: duration }),
  });
  const audio = await res.arrayBuffer();
  return new Response(audio, { headers: { 'Content-Type': 'audio/mpeg' } });
}
```

**[코드 C-2] 오디오→텍스트 (Whisper, 타임스탬프)**

```python
from openai import OpenAI
client = OpenAI()

def transcribe(path: str):
    with open(path, "rb") as f:
        res = client.audio.transcriptions.create(
            model="whisper-1", file=f,
            response_format="verbose_json",
            timestamp_granularities=["segment"],
        )
    return res.segments  # 각 segment에 start, end, text
```

**[코드 C-3] 세그먼트 → SRT / ASS**

```python
import pysubs2

def to_subtitle(segments, fmt="srt", out="output"):
    subs = pysubs2.SSAFile()
    for s in segments:
        subs.append(pysubs2.SSAEvent(
            start=pysubs2.make_time(s=s["start"]),
            end=pysubs2.make_time(s=s["end"]),
            text=s["text"].strip()))
    subs.save(f"{out}.{fmt}")  # fmt: "srt" 또는 "ass"
```
