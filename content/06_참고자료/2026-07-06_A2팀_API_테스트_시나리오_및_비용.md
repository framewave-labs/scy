# A2팀 API 테스트 시나리오 및 비용
> 작성일: 2026-07-06 | 작성자: 손찬용 PM
> 개발자 전달용 / 결제 확정 후 API 호출 시 시나리오별 예상 비용과 호출 수칙

## 1. 목적
본 문서는 [[2026-07-06_A2팀_API_결제_운영안]]에 따라 회사 결제가 확정된 뒤, 개발자가 실제 API를 호출할 때 어떤 시나리오에서 얼마의 비용이 나가는지 미리 알 수 있도록 정리한 문서다. 수치는 2026-07-06 KX_전체비용_통합 시트(A2)를 기준으로 하며, 2026-07-06 결제 운영안의 최신 결정(fal.ai 공용 계정, 선불 크레딧)을 반영했다.

## 2. 과금 구조 요약
- fal.ai는 **선불 크레딧에서 사용량만큼 차감**되는 구조다. 크레딧이 소진되면 호출이 실패하므로, 대량 테스트 전에 잔액을 확인한다.
- 과금은 **생성 결과 기준**이다. 서버 오류나 큐 대기 시간은 과금 대상이 아니지만, 생성이 완료된 결과물은 품질과 무관하게 과금되므로 "결과가 마음에 안 들어 다시 생성"하는 재생성도 전부 비용이다.
- API 키는 회사 공용 계정 기준으로 전달받으며, 개발자가 개인 결제수단을 연결하지 않는다.

**단가 확인 필요 항목**: 2026-07-06 결제 운영안 기준으로 이미지 생성이 "GPT Image 2 on fal"로 정리되었다. fal 경유 시 단가가 OpenAI 직접 호출과 다를 수 있으므로, 키 수령 후 첫 호출 전에 fal.ai 모델 페이지에서 실제 단가를 확인하고 이 표를 갱신한다.

## 4. 테스트 시나리오별 예상 비용

### 4.1 이미지 생성 (M001~M006, 2주차~)

| 시나리오 | 물량 | 단가 | 예상 비용 |
| --- | --- | --- | --- |
| 개발·테스트 + 데모 (medium) | 120장 | $0.042 | **$5.04** |
| 3x3 스토리보드 그리드 | 0장 | — | **$0** (B서비스가 생성해 제공, A2팀 생성 없음) |

- 120장 산정 근거: 4주 × 팀 테스트 + 프롬프트 조정 반복 + 데모 후보.
- 데모 후보만 high로 전환할 경우 장당 약 4배 비용이므로, high는 중간점검·최종 발표 직전에만 사용한다.

### 4.2 영상 생성 (A002~A005, 3~4주차 중심)

산정 단위는 **3x3 스토리보드 이미지 1장 + 영상 프롬프트 → 영상 1개(1클립)**다. 과거 문서의 "1세트 = 9클립" 계산은 폐기됐다.

| 시나리오 | 모델 | 물량 | 계산 | 예상 비용 |
| --- | --- | --- | --- | --- |
| 개발 테스트 (연동·디버깅) | Kling 5초 | 8클립 | 8 × 5s × $0.11 | **$4.40** |
| 중간점검 전후 검증 | Seedance 5초 | 10클립 | 10 × 5s × $0.3034 | **$15.17** |
| 3주차 품질 확인 | Seedance 10초 | 6클립 | 6 × 10s × $0.3034 | **$18.20** |
| 4주차 최종 출력 | Seedance 15초 | 7클립 | 7 × 15s × $0.3034 | **$31.86** |
| **영상 소계** | | 31클립 | | **$69.63** |

- 참고: Seedance 15초 1클립 = 약 $4.55. Kling 5초 1클립 = $0.55. **테스트에 Seedance를 쓰면 비용이 8배 이상 커진다.**

### 4.3 프롬프트 보조 (Claude Sonnet)

| 시나리오 | 물량 | 예상 비용 |
| --- | --- | --- |
| 프롬프트 생성 보조 (한국어 → 영문 변환, 역프롬프트 포함) | 350회 | **$5.25** |

2026-07-06 KX_전체비용_통합 시트 기준으로 역프롬프트(S001~S004) 호출은 별도 산정 없이 프롬프트 보조 350회에 포함한다.

### 4.4 전체 요약

| 구분 | 예상 비용 |
| --- | --- |
| 사용량 기반 합계 (이미지 + 영상 + 프롬프트 보조) | 약 **$80** |
| 고정 구독 (Claude Team 4석, Figma 2석, Vercel Pro) | 별도 $180 (회사 결제팀 관리 범위) |
| AWS EC2 예비 예산 (상세는 BE 산정 예정) | 별도 $100 |

사용량 기반 API 비용 자체는 크지 않다. **리스크는 총액이 아니라 수칙 위반**(테스트에 Seedance·high·긴 영상 사용)과 **실패 재생성 반복**에서 나온다.

## 5. 개발자 호출 수칙

| # | 수칙 | 근거 |
| --- | --- | --- |
| 1 | 영상 테스트는 반드시 **Kling · 5초** | Seedance 대비 초당 약 1/3, 짧은 길이로 비용 최소화 |
| 2 | 이미지 테스트는 **medium 고정**, high는 데모 후보만 | 장당 4배 차이 |
| 3 | 10~15초 영상은 최종 발표 후보에만 사용 | 15초 1클립 ≈ $4.55 |
| 4 | fal.ai **Hard Limit $100** 설정을 키 수령 즉시 확인 | 과호출 자동 차단. 설정 전 영상 API 대량 테스트 금지 |
| 5 | 생성 실패율이 높으면 호출 반복 전에 프롬프트·입력 구조부터 조정 | 재생성도 전부 과금 |
| 6 | 매주 금요일 PM이 대시보드 사용량 확인 — 이상 급증 시 즉시 공유 | 주간 비용 점검 규칙 |

## 6. 키 수령 후 개발자가 먼저 확인할 것

1. fal.ai에서 사용할 이미지 모델 ID(GPT Image 2 on fal)와 실제 단가
2. 영상 모델 확정 (Kling / Seedance 2.0 중 테스트·데모 각각) 및 요청 파라미터(길이, 해상도, reference image 필드)
3. `video-status` 상태 조회 호출이 과금 대상인지 여부
4. Hard Limit $100 설정 완료 여부 (Billing > Spend Limit)
5. 크레딧 잔액 확인 방법과 잔액 부족 시 에러 응답 형태

## 7. 후속조치
1. 결제 확정 즉시 이 문서의 예상 물량을 팀 실사용 계획과 대조해 초기 크레딧 충전액 확정 (권장: Hard Limit과 동일한 $100 수준에서 시작)
2. 키 수령 후 6번 확인 항목 결과를 반영해 단가표 갱신
3. 주간 비용 점검 결과를 매주 금요일 이 문서 또는 주간 보고서에 누적 기록
