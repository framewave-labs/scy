# A2팀 개발자 전달사항
> 작성일: 2026-07-06 | 작성자: 손찬용 PM
> 이슬기 PM님 공유용 / 계정·결제 운영안을 개발 착수 기준으로 전환해 개발자에게 전달하기 위한 문서

## 1. 목적
본 문서는 [[2026-07-06_A2팀_API_결제_운영안]], [[2026-07-06_A2팀_배포환경_결제_운영안]]에서 정리한 계정/결제 운영 기준을 바탕으로, 백엔드·프론트엔드 개발자가 실제 착수 시 확인해야 할 권한·환경변수·구현 기준과 개발자에게 먼저 확인할 질문을 정리한 문서다. 계정 생성과 결제는 회사가 담당하고, 개발자는 그 결과로 제공되는 최소 권한과 환경변수만 받아 구현에 들어가는 것을 전제로 한다.

## 2. 공통 안내
- 회사가 계정 생성과 결제를 담당하고, 개발팀은 필요한 최소 권한과 환경변수만 받아 구현한다.
- 개발자는 직접 결제수단을 관리하지 않는다.
- 계정/권한은 회사 측이 제공하는 기준으로 작업한다.

## 3. 백엔드 개발자 전달사항
- AWS 계정 및 최소권한 IAM 역할 필요
- EC2, S3, 환경변수 사용 기준 필요
- `/api/generate-image`, `/api/generate-video`, `/api/video-status`, `/api/file` 구현 기준 필요
- 영상 상태값 `PENDING / RUNNING / SUCCESS / FAILED` 매핑 필요
- 파일 업로드/삭제/다운로드 정책 확정 필요
- 배포용 리전, 서버 URL, 포트, 헬스체크 기준 필요

## 4. 프론트엔드 개발자 전달사항
- Vercel Team 프로젝트 초대 필요
- 배포 브랜치 및 프리뷰/프로덕션 URL 기준 필요
- 환경변수 목록 필요
- 이미지 생성, 저장, 라이브러리, 영상 상태 UI 연동 기준 필요
- 로그인/권한 여부에 따른 라우팅 기준 필요
- 배포 전 사전 빌드 완료 및 접속 확인 필요

## 5. fal.ai 연동 관련
- 이미지 생성 모델: GPT Image 2 on fal
- 영상 생성 모델: fal.ai 영상 모델 연동
- API 키는 회사 공용 계정 기준으로 전달받는다.
- 실패 호출, 재시도, 길이 제한은 비용 리스크 기준과 함께 관리한다.

## 6. 개발자에게 먼저 확인할 질문
- AWS에서 꼭 필요한 권한 범위는 어디까지인지
- Vercel에서 필요한 환경변수는 무엇인지
- fal.ai에서 사용할 모델과 요청 파라미터는 무엇인지
- 파일 저장을 S3로 할지 로컬로 할지
- 영상 상태 조회를 polling으로 할지 다른 방식으로 할지
- 배포 전 사전 빌드와 접속 URL 확인 책임자가 누구인지

## 7. 후속조치
1. AWS/IAM 권한 요청 목록 확정
2. Vercel Team 초대 및 환경변수 목록 확정
3. fal.ai API 키와 모델 범위 확정
4. 파일 저장 및 상태조회 정책 확정
5. 구현 담당자별 작업 분배
