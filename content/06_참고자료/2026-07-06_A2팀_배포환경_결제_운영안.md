# A2팀 배포환경 결제 운영안
> 작성일: 2026-07-06 | 작성자: 손찬용 PM
> 이슬기 PM님 공유용 / AWS와 Vercel 배포환경 결제 및 권한 운영 기준

## 1. 목적
본 문서는 A2팀 KX GenStudio의 프론트엔드 및 백엔드 배포를 위해 AWS와 Vercel의 계정, 결제, 권한을 어떻게 운영할지 정리한 문서다. PRD 기준으로 4주차 최소 완주 목표에는 Vercel FE와 AWS BE 기준 배포 시연이 포함되어 있으므로, 배포환경은 별도 운영 기준이 필요하다. [cite:194]

## 2. 운영 대상
- AWS: 백엔드 배포, 저장소, IAM, 비용 관리 [cite:194][cite:219]
- Vercel: 프론트엔드 배포, Team Billing, Spend Management [cite:130][cite:131]

## 3. AWS 운영 방식
- 계정 소유: 회사 [cite:56][cite:219]
- 결제 담당자: 회사 결제팀 또는 지정 담당자 [cite:56][cite:197]
- 개발 권한자: A2팀 백엔드/배포 담당자 [cite:210][cite:194]
- 권장 방식: 회사 소유 AWS 계정에서 Billing과 IAM 권한을 분리하고, 개발팀에는 EC2/S3 중심의 최소 권한만 부여 [cite:219][cite:220]

> **실제 진행 (2026-07-07 확인)**: 위 권장 방식과 달리, 이번 결제 라운드에서 AWS는 제외되어 **개인 계정으로 우선 세팅**하는 방향으로 진행한다. 이후 회사 결제 확정 시 **AWS Organizations로 개인 계정을 멤버 계정으로 편입**하는 방식으로 전환할 예정이다. 초기 세팅 단계의 임시 조치이며, 회사 소유 계정/권한 분리라는 원래 권장 방식은 Organization 편입 시점에 다시 적용한다.

### AWS 권장 이유
AWS는 결제 접근을 IAM으로 분리할 수 있으므로, 회사가 개발 구조를 모두 이해하지 않아도 결제 담당자와 개발 권한자를 분리해 운영할 수 있다. 초기 비용 통제를 위해서는 필요 시 100달러 Advance Pay를 검토할 수 있지만, 우선순위는 회사 계정 소유와 권한 분리다. [cite:175][cite:210][cite:219]

### AWS 필요한 세팅 목록
- 회사 소유 계정 생성 또는 기존 계정 사용 여부 확정 [cite:56]
- Billing 접근 권한 설정 [cite:219][cite:220]
- IAM 최소권한 역할 생성 [cite:210][cite:219]
- EC2/S3/환경변수 운영 권한 전달 [cite:194]
- 필요 시 Budgets 또는 비용 확인 방식 설정 [cite:58][cite:62]
- 초기 100달러 Advance Pay 검토 여부 결정 [cite:175]

### AWS 후속조치
1. 개인 계정으로 EC2/S3 우선 세팅 (2026-07-07 기준 진행 중)
2. 결제 담당자 지정 및 회사 계정/Organization 편입 시점 확정 [cite:197]
3. AWS Organizations로 개인 계정을 멤버 계정으로 편입 [cite:56]
4. 편입 후 A2팀 개발자에게 최소권한 IAM 역할 전달 [cite:210]
5. EC2, S3, 환경변수, 배포 주소 기준 확정 [cite:194]
6. 예산 알림 및 월 비용 확인 방식 설정 [cite:58][cite:62]

## 4. Vercel 운영 방식
- 계정 소유: 회사 또는 팀 Owner [cite:209][cite:212]
- 결제 담당자: Billing 역할 [cite:130][cite:216]
- 개발 권한자: A2팀 FE 담당자 [cite:209][cite:216]
- 권장 방식: 회사 또는 팀 소유 Team 계정에서 Pro 플랜과 Billing을 관리하고, 개발팀은 Member/Developer로 초대 [cite:130][cite:131][cite:216]

### Vercel 권장 이유
Vercel은 Team 단위 청구와 역할 분리가 자연스러워, 개인 계정보다 회사 Team 계정으로 시작하는 것이 인수인계와 비용 통제에 유리하다. Spend Management를 함께 사용하면 예상치 못한 과금을 줄일 수 있다. [cite:131][cite:135][cite:216]

### Vercel 필요한 세팅 목록
- 회사 또는 팀 소유 Team 생성 [cite:209][cite:212]
- Pro 플랜 적용 (2026-07-06 확정, 1석 $20/월) [cite:130][cite:132]
- Billing 역할 초대 [cite:216][cite:212]
- FE 담당자 Member/Developer 초대 [cite:209][cite:216]
- 프로젝트 생성, 빌드 세팅, 환경변수 입력, 접속 URL 확인 [cite:194]
- Spend Management 설정 [cite:131][cite:135]

### Vercel 후속조치
1. Team 생성 가능 여부 확인 [cite:209]
2. 결제 담당자 Billing 역할 초대 [cite:216][cite:212]
3. FE 담당자 프로젝트 초대 [cite:209]
4. 환경변수와 배포 URL 확정 [cite:194]
5. Spend Management 및 비용 알림 설정 [cite:131]
