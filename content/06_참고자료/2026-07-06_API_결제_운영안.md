# A2팀 API 결제 운영안
> 작성일: 2026-07-06 | 작성자: 손찬용 PM
> 이슬기 PM님 공유용 / fal.ai 중심 API 결제 및 크레딧 운영 기준

## 1. 목적
본 문서는 A2팀의 이미지 생성 및 영상 생성에 사용되는 외부 API 결제를 회사 기준으로 운영하기 위해 작성한다. PRD상 이미지 생성은 GPT Image 2 기준으로, 영상 생성은 fal.ai 연동 흐름 기준으로 관리되므로, API 결제는 외부 플랫폼 계정과 크레딧, API 키를 어떤 방식으로 운영할지 정리하는 것이 목적이다. [cite:194][cite:239][cite:240]

## 2. 운영 대상
- 이미지 생성: GPT Image 2 on fal [cite:239][cite:240]
- 영상 생성: fal.ai 영상 모델 연동 [cite:194][cite:17]

## 3. 권장 운영 방식
- 계정 소유: 회사 공용 계정 또는 회사가 승인한 워크스페이스 [cite:17][cite:157]
- 결제 담당자: 회사 결제팀 또는 PM 승인 담당자 [cite:17]
- 사용 담당자: A2팀 개발자 및 연동 담당자 [cite:194]
- 비용 구조: 선불 크레딧 충전 후 사용량 차감 [cite:17]

## 4. 왜 이 방식이 적절한가
fal.ai는 선불 크레딧 기반으로 사용량에 따라 비용이 차감되므로, 회사 공용 계정에 크레딧을 충전하고 개발팀은 API 키만 받아 쓰는 구조가 가장 단순하다. 또한 PRD상 영상 생성은 실패 호출 및 긴 영상 옵션에서 비용 리스크가 커질 수 있어, 결제 책임과 사용 책임을 분리하는 편이 안전하다. [cite:17][cite:194]

## 5. 필요한 세팅 목록
- 회사 공용 fal.ai 계정 또는 워크스페이스 생성 [cite:157]
- GPT Image 2 사용 여부 확정 [cite:239][cite:240]
- 영상 모델 사용 범위 확정 [cite:194]
- 크레딧 충전 담당자 지정 [cite:17]
- API 키 발급 및 전달 대상 지정 [cite:157]
- 재생성/길이 제한 등 내부 운영 기준 확정 [cite:45][cite:46]

## 6. 후속조치
1. 회사 측에 fal.ai 공용 계정 운영 가능 여부 확인 [cite:17]
2. 충전 담당자 지정 및 초기 크레딧 확보 [cite:17]
3. A2팀 API 연동 담당자에게 키 전달 [cite:157]
4. 이미지/영상 모델별 사용 기준을 내부 문서에 고정 [cite:194]
5. 비용 경고 또는 월 사용 한도 기준을 별도 설정 [cite:45][cite:46]
